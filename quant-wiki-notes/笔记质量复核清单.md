# 笔记质量复核清单

> [!note] 核心问题
> 扩库速度快时，最容易出现：断链、重复入口、示例被当成建议、官网过时、作业数字不一致。本清单供**人工或 Codex 类复核**使用；通过后再合并/交付。

## 一、链接与命名

| 检查 | 通过标准 |
|---|---|
| 新导航文 wikilink | 目标笔记存在；路径型用 `[[路径/笔记|显示名]]`（竖线两侧无反斜杠）；或**唯一 basename** / alias |
| 枢纽 `目录.md` | 有 `title` 与覆盖显示名的 `aliases`（如 `进阶专题总览`） |
| 作业填空 | 优先 `[[作业填空/目录]]` 等路径链接，避免多个「目录」歧义 |
| 外链 | 优先 https 官网；注明「以打开时页面为准」 |

## 二、内容诚实性

| 检查 | 通过标准 |
|---|---|
| 投资建议 | 教学/示例标明**非投资建议** |
| 示例身份 | 虚构「阿 Q」/虚拟研报不得被写成事实荐股 |
| 真实 EXP | 数字来自可复现 JSON；写清引擎简化 |
| 监管 | 日期与链接可核对；不替代法律意见 |

## 三、结构完整（实操导航类）

| 块 | 需要 |
|---|---|
| 核心问题 callout | 有 |
| 学习目标 | 3–5 条 |
| 路径/清单/作业 | 至少一种可执行产出 |
| 相关概念 | 链回作业或总表 |
| 常见误区（推荐） | 有则更好 |
