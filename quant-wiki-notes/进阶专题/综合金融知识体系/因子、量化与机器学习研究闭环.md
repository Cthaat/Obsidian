# 因子、量化与机器学习研究闭环

> [!note] 学习定位
> 量化研究的核心不是复杂模型，而是把可证伪假设转成稳定数据、稳健回测、风险控制和实盘监控。本文仅用于学习、研究与复盘，不构成任何投资建议。

## 一、因子研究闭环

- 提出经济解释。
- 定义可复现口径。
- 处理极值、缺失、行业和市值暴露。
- 检验 IC、分层收益、换手、容量和样本外稳定性。
- 上线后监控衰减和拥挤。

## 二、机器学习闭环

- 按时间切分训练、验证和测试。
- 确保特征在交易时点真实可得。
- 目标函数必须扣成本。
- 用可解释性和风险约束避免黑箱失控。

## 三、研究治理

- 每个结果必须有基准策略。
- 每次参数修改必须留下记录。
- 每个上线模型必须有降级和暂停条件。

## 实战掌握清单

### 研究闭环

因子和机器学习研究都要走同一条主线：假设、数据、特征、验证、组合、交易、监控。因子强调经济解释和可解释暴露，机器学习强调非线性拟合和预测能力，但二者都必须面对数据泄漏、过拟合、交易成本和容量限制。

### 最低研究标准

1. 写清楚信号为什么应该有效：风险补偿、行为偏差、制度限制、信息滞后或流动性补偿。
2. 处理数据口径：复权、停牌、退市、公告时点、缺失值、异常值和可交易性。
3. 使用时间序列验证：样本内开发、样本外测试、滚动窗口和参数扰动。
4. 组合层面看贡献：IC好不等于组合赚钱，必须看换手、暴露、成本和回撤。
5. 实盘后监控衰减：信号分布、成交质量、拥挤度、容量和收益归因。

### 机器学习特别风险

随机切分、未来数据泄漏、标签选择不当、特征工程污染、模型不可解释和实盘分布漂移，是机器学习交易中最常见的失败来源。复杂模型必须先战胜简单基准，否则复杂度没有价值。

## 深度案例与训练

### 小型研究项目

围绕 因子、量化与机器学习研究闭环 设计一个一页纸研究项目：先说明它属于资产定价、收益来源、风险约束、交易执行还是行为纪律，再选一个真实标的或策略做案例。案例中必须写清楚输入数据、判断逻辑、仓位动作和退出条件。

### 全局连接

- 与宏观连接：增长、通胀、利率、信用和流动性如何影响它。
- 与组合连接：它会增加还是降低组合风险，相关性会在何时变化。
- 与执行连接：交易成本、滑点、流动性和规则限制是否改变结论。
- 与复盘连接：哪些证据说明判断正确，哪些证据要求暂停。

### 训练标准

能把 因子、量化与机器学习研究闭环 讲清楚还不够，还要能把它写成一条可执行规则、一张风险清单和一个复盘问题。

## 最小可执行项目

### 一页系统卡

围绕 因子、量化与机器学习研究闭环 写一张系统卡：它属于哪条主线，影响哪类资产，使用哪些数据，改变哪些交易动作，暴露哪些风险。系统卡的目标是把知识从“知道”变成“会用”。

| 模块 | 输出 |
|---|---|
| 收益来源 | 这项知识解释哪类收益 |
| 风险来源 | 最可能亏在哪里 |
| 数据证据 | 需要哪些指标或案例 |
| 组合影响 | 会提高还是降低相关风险 |
| 执行约束 | 成本、流动性和规则限制 |
| 复盘问题 | 哪个证据会推翻原判断 |

### 验收标准

读完 因子、量化与机器学习研究闭环 后，应能写出一个真实资产案例，并说明买入、卖出、仓位或不交易中的一个动作如何改变。
