# 金融数据、回测与模型治理

> [!note] 学习定位
> 数据质量决定研究上限，模型治理决定实盘下限。没有数据治理的回测，很容易把错误包装成收益。本文仅用于学习、研究与复盘，不构成任何投资建议。

## 一、数据治理

- 记录供应商、下载时间、字段解释、复权方式、时区和交易日历。
- 处理退市、停牌、公告日、财报修订、缺失值和异常值。
- 关键字段用多个来源抽样交叉验证。

## 二、回测治理

- 信号生成和交易执行必须有时间间隔。
- 手续费、滑点、冲击成本、涨跌停和容量必须进入测算。
- 样本内结果必须经过样本外、滚动和压力测试。

## 三、模型治理

- 记录每次参数、特征和代码版本。
- 设定上线门槛、暂停条件和降级方案。
- 用实盘数据持续比较预期和现实偏差。

## 实战掌握清单

### 数据治理清单

- 数据血缘：供应商、接口、下载时间、字段定义、时区、交易日历和版本号。
- 时间可得性：财报、公告、指数成分、评级和宏观数据必须按真实可获得时间处理。
- 异常处理：停牌、退市、涨跌停、缺失、极值、复权、合约换月和代码变更。
- 交叉验证：关键字段至少抽样对比两个来源，尤其是成交量、复权因子和财务数据。

### 回测质量门

1. 基准策略是否明确，复杂策略是否真的优于简单规则。
2. 是否包含费用、滑点、冲击成本、容量、最小交易单位和成交失败。
3. 是否做样本外、滚动窗口、参数扰动、市场分段和压力测试。
4. 是否输出失败案例，而不是只展示收益曲线。

### 模型治理

每个模型都要有版本、参数、训练样本、特征清单、上线日期、暂停条件和回滚方案。实盘监控要比较预期收益与实际收益、预测分布与真实分布、信号强度与成交质量。模型失效不是意外，而是治理流程必须预设的常态。

## 深度案例与训练

### 小型研究项目

围绕 金融数据、回测与模型治理 设计一个一页纸研究项目：先说明它属于资产定价、收益来源、风险约束、交易执行还是行为纪律，再选一个真实标的或策略做案例。案例中必须写清楚输入数据、判断逻辑、仓位动作和退出条件。

### 全局连接

- 与宏观连接：增长、通胀、利率、信用和流动性如何影响它。
- 与组合连接：它会增加还是降低组合风险，相关性会在何时变化。
- 与执行连接：交易成本、滑点、流动性和规则限制是否改变结论。
- 与复盘连接：哪些证据说明判断正确，哪些证据要求暂停。

### 训练标准

能把 金融数据、回测与模型治理 讲清楚还不够，还要能把它写成一条可执行规则、一张风险清单和一个复盘问题。

## 最小可执行项目

### 一页系统卡

围绕 金融数据、回测与模型治理 写一张系统卡：它属于哪条主线，影响哪类资产，使用哪些数据，改变哪些交易动作，暴露哪些风险。系统卡的目标是把知识从“知道”变成“会用”。

| 模块 | 输出 |
|---|---|
| 收益来源 | 这项知识解释哪类收益 |
| 风险来源 | 最可能亏在哪里 |
| 数据证据 | 需要哪些指标或案例 |
| 组合影响 | 会提高还是降低相关风险 |
| 执行约束 | 成本、流动性和规则限制 |
| 复盘问题 | 哪个证据会推翻原判断 |

### 验收标准

读完 金融数据、回测与模型治理 后，应能写出一个真实资产案例，并说明买入、卖出、仓位或不交易中的一个动作如何改变。

## 八、实战案例：同一个策略为什么两套回测结果不同

常见原因不是策略逻辑不同，而是数据和执行口径不同：

- 一个使用前复权，一个使用后复权。
- 一个包含退市股票，一个只保留当前仍上市股票。
- 一个按公告日使用财报，一个按报告期提前使用财报。
- 一个假设全额成交，一个考虑涨跌停、停牌和成交量限制。
- 一个只算佣金，一个加入滑点、冲击成本和印花税。

因此，回测报告必须写明数据口径。没有口径的收益曲线无法复盘，也无法比较。

## 九、模型上线门槛

模型从研究到实盘，至少经过四道门：

1. 研究门：样本外有效，且有清晰经济解释。
2. 成本门：扣除交易成本后仍有收益。
3. 风控门：最大回撤、容量和异常场景可承受。
4. 监控门：上线后能跟踪数据漂移、信号衰减和成交质量。

任何一道门过不了，模型都不应直接进入实盘。

## 进一步实战化

### 一页应用练习

围绕 `金融数据、回测与模型治理` 选择一个真实资产或策略，写一页应用卡：

| 项目 | 输出 |
|---|---|
| 它解决什么问题 | 收益来源、风险约束、执行效率或心理纪律 |
| 需要哪些证据 | 数据、财报、宏观、估值、价格或交易记录 |
| 改变哪个动作 | 买入、卖出、仓位、对冲、观察或不交易 |
| 最大误用风险 | 在什么情况下这个框架会误导你 |
| 复盘指标 | 哪个证据说明原判断应更新 |

这张卡的目的，是把综合知识从“概念地图”变成“决策动作”。
