# 交易日志、复盘系统与错误库

> [!note] 学习定位
> 复盘不是事后写感想，而是把决策、执行和结果变成可改进的数据。没有交易日志，错误会重复发生；没有错误库，经验会随着情绪消失。本文仅用于学习、研究与复盘，不构成任何投资建议。

## 一、交易日志要记录什么

| 类别 | 记录内容 | 目的 |
|---|---|---|
| 交易前 | 假设、证据、反方观点、仓位、止损、目标 | 防止临场冲动 |
| 交易中 | 下单价格、订单类型、成交、滑点、情绪 | 评估执行质量 |
| 交易后 | 盈亏、归因、是否遵守规则、改进项 | 建立反馈循环 |
| 周期复盘 | 月度收益、回撤、错误频率、策略表现 | 识别系统性问题 |

```text

```

## 三、错误库分类

错误库要按“可改进原因”分类，而不是按盈亏分类：

- 研究错误：数据错、财报误读、行业判断错、估值假设错。
- 策略错误：信号无效、参数过拟合、容量不足、成本低估。
- 仓位错误：过度集中、杠杆过高、加仓太急、现金不足。
- 执行错误：追价、错单、滑点过大、未按计划止损。
- 心理错误：怕错过、想翻本、锚定成本、盈利过早落袋。
- 规则错误：交易计划本身不清晰，导致无法判断是否违反。

## 四、复盘时不要只看盈亏

一个赚钱交易可能是坏决策，一个亏钱交易也可能是好决策。复盘要拆成：

1. 过程是否正确：是否按计划执行。
2. 假设是否正确：证据是否支持原判断。
3. 风险是否可控：亏损是否在预算内。
4. 结果是否可重复：收益来自能力还是运气。
5. 规则是否要改：是偶发错误还是流程缺陷。

## 五、周度和月度复盘

| 周度复盘 | 月度复盘 |
|---|---|
| 本周是否执行计划 | 本月收益和回撤归因 |
| 最大错误是什么 | 哪类错误重复出现 |
| 是否有冲动交易 | 哪个策略贡献最大 |
| 是否需要暂停 | 风险预算是否需要调整 |
| 下周重点观察 | 下月学习和改进计划 |

## 六、错误转规则

复盘的最终产物必须是规则。例如：

- 错误：利好消息后追高买入。
- 规则：消息驱动交易必须等到价格回到计划区间，或只用试探仓。
- 错误：亏损后加仓摊低成本。
- 规则：只有原假设增强才允许加仓，单纯因为亏损不允许加仓。

## 九、错误库维护规则

错误库要定期整理，否则会变成情绪垃圾桶。建议每月做一次维护：

1. 合并重复错误，把“追高买入”“怕错过买入”归为同一类。
2. 删除无法行动的描述，例如“以后要冷静”，改成具体规则。
3. 标记最高成本错误，优先修复亏损最大或重复最多的错误。
4. 检查新规则是否真的减少错误，而不是制造新的僵化问题。

## 十、优秀复盘的标准

一份好复盘应该能让未来的你更容易做对，而不是让现在的你情绪舒服。最低标准是：

- 能区分好决策坏结果和坏决策好结果。
- 能找到可修改的流程，而不是只归因于运气。
- 能把错误转成下一次交易前的检查项。
- 能记录不交易的正确决策，避免只复盘买卖。

## 进一步实战化

### 一页应用练习

围绕 `交易日志、复盘系统与错误库` 选择一个真实资产或策略，写一页应用卡：

| 项目 | 输出 |
|---|---|
| 它解决什么问题 | 收益来源、风险约束、执行效率或心理纪律 |
| 需要哪些证据 | 数据、财报、宏观、估值、价格或交易记录 |
| 改变哪个动作 | 买入、卖出、仓位、对冲、观察或不交易 |
| 最大误用风险 | 在什么情况下这个框架会误导你 |
| 复盘指标 | 哪个证据说明原判断应更新 |

这张卡的目的，是把综合知识从“概念地图”变成“决策动作”。

## 十一、最终提醒

交易日志最有价值的部分，不是记录某次交易赚了多少钱，而是让你发现自己在哪些情景下会重复犯错。复盘也不是事后证明自己聪明，而是把错误变成下一次下单前能执行的限制条件。

如果一份日志只写行情和盈亏，它只是流水账；如果它能写清楚计划、情绪、证据、动作、偏离和改进规则，它才会逐渐变成交易系统的一部分。长期看，能复盘的人不是没有犯错，而是犯同一种错的次数越来越少。
