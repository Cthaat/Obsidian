# 技术分析

> [!note] 核心定位
> 技术分析 属于价格行为和技术分析框架。它的价值是辅助识别趋势、波动、支撑阻力和入退出场纪律，而不是替代估值、基本面和风险管理。

## 一、技术分析真正解决什么问题

技术分析研究的是价格、成交量、波动和市场行为的可观察结构。它不直接回答“这家公司值多少钱”，而是回答：

- 市场当前是趋势、震荡，还是极端情绪？
- 资金是否正在流入或流出？
- 入场点、止损点、加仓点和退出点在哪里更清晰？
- 这笔交易的盈亏比是否值得做？

技术分析的价值不在于预测未来，而在于把模糊的交易行为规则化。它适合辅助择时、风控和执行，不应替代基本面研究、估值和组合管理。

## 二、技术分析的主要工具

| 工具 | 关注点 | 常见用途 |
|---|---|---|
| 趋势线/均线 | 方向和趋势强弱 | 判断顺势、破位和移动止损 |
| 支撑阻力 | 市场记忆和供需区域 | 设定入场、止损、止盈位置 |
| K线形态 | 单周期或多周期价格行为 | 观察反转、延续和犹豫 |
| 成交量 | 价格变化背后的参与度 | 判断突破真假和资金强度 |
| 动量指标 | 上涨/下跌速度 | 辅助判断强弱和背离 |
| 波动率指标 | 价格波动幅度 | 设定止损、仓位和突破过滤 |
| 多时间框架 | 大小周期一致性 | 避免小周期信号逆大趋势 |

## 三、技术分析的正确使用顺序

1. **先判断市场环境**：趋势市、震荡市、单边下跌、事件驱动行情的规则不同。
2. **再确定交易级别**：日线、周线、分钟线不能混用；入场级别和止损级别要一致。
3. **只选择少数工具**：趋势、量价、支撑阻力、波动率任选几类即可，不要堆满指标。
4. **写清交易规则**：入场、止损、止盈、加仓、减仓都要能被复盘。
5. **用回测或交易日志验证**：看信号长期胜率、盈亏比、回撤和适用环境。

## 四、技术分析的误区

- 把图形当成必然预测，而不是概率线索。
- 只寻找支持自己持仓方向的形态。
- 同时使用太多高度相关指标，制造“确认感”。
- 不区分趋势市和震荡市，导致指标频繁失效。
- 没有止损，亏损后把短线交易改口成长期持有。
- 忽略交易成本、滑点和流动性。

## 五、交易计划模板

```text
交易级别：
市场环境：
入场信号：
确认条件：
止损位置：
目标位置：
单笔风险：
失效条件：
复盘字段：
```

一个成熟交易者不会问“这个形态准不准”，而会问“这个信号在什么环境下有效、错了亏多少、对了赚多少、是否能长期执行”。

## 八、适用环境

`技术分析` 在不同市场环境中的含义不同。趋势行情中，突破和动量信号更有价值；震荡行情中，支撑阻力和均值回归更重要；事件行情中，技术形态可能被消息直接打穿。

| 环境 | 使用方式 |
|---|---|
| 趋势市 | 顺势入场、移动止损、让利润奔跑 |
| 震荡市 | 区间交易、控制频率、减少追涨杀跌 |
| 高波动 | 扩大止损距离、降低仓位 |
| 低流动性 | 谨慎使用形态信号，防止无法退出 |

## 九、训练方法

选取20个历史案例，记录信号出现后的结果：成功、失败、横盘、跳空失效。技术分析只有经过样本化复盘，才会从“看图感觉”变成交易规则。

## 深度案例与训练

### 图表到规则

学习 技术分析 时，把图表语言转成可执行条件：信号出现在哪里、需要什么成交量确认、失效点在哪里、盈亏比是否足够、是否需要大级别趋势配合。

### 训练方法

- 选取趋势、震荡、急跌反弹三种行情，各找十个案例。
- 记录入场点、止损点、目标位、实际成交和是否出现假信号。
- 用相同规则复盘，不在失败后临时修改解释。

### 风险边界

技术信号在低流动性、消息冲击和极端波动中容易失真。它适合辅助执行和风险定位，不应替代基本面、宏观或组合层面的判断。

## 最小可执行项目

### 技术信号统计

围绕 技术分析 选取至少三十个历史样本，按趋势、震荡和急跌反弹分类。每个样本记录信号出现位置、成交量、确认条件、入场点、失效点、目标位和最终盈亏。

| 输出 | 用途 |
|---|---|
| 成功样本 | 找共同条件 |
| 失败样本 | 找假信号来源 |
| 盈亏分布 | 判断是否值得交易 |
| 最大连续亏损 | 设计仓位和止损 |

### 验收标准

能把图形或指标转成可复现规则，并知道在什么行情中应降低使用权重。
