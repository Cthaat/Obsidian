# 风险建模

> [!note] 核心定位
> 风险建模的目标不是预测每一次亏损，而是在交易前回答：亏损会从哪里来、可能有多大、多久恢复、什么情况下必须降仓或停止。一个不能触发动作的风险模型，只是漂亮报表。

## 一、风险建模的目的

风险建模不是为了预测每一次亏损，而是为了在交易前回答：“如果我错了，会以什么方式亏、亏多少、多久恢复、是否会被迫出局？”

一个风险模型至少要覆盖五类风险：

| 风险 | 典型问题 |
|---|---|
| 市场风险 | 价格波动、beta、利率、汇率、商品、波动率变化 |
| 信用风险 | 债务人违约、可转债信用恶化、交易对手风险 |
| 流动性风险 | 想卖时卖不出，或卖出成本远高于预期 |
| 模型风险 | 假设错、参数错、数据错、过拟合 |
| 操作风险 | 系统故障、订单错误、接口异常、权限和流程问题 |

## 二、常见风险模型

| 模型 | 用途 | 局限 |
|---|---|---|
| 波动率模型 | 衡量日常价格波动 | 不能充分描述尾部跳跃 |
| VaR | 给定置信水平下的潜在损失 | 可能低估极端亏损 |
| CVaR/ES | 极端亏损发生后的平均损失 | 依赖尾部样本质量 |
| 因子风险模型 | 拆分市场、行业、风格风险 | 因子选择和协方差估计会错 |
| 压力测试 | 模拟危机场景损失 | 场景设计依赖经验 |
| 流动性模型 | 估计成交冲击和退出时间 | 牛市成交量会高估容量 |

## 三、风险模型输入

风险模型的质量取决于输入，而不是公式看起来多高级。

- 收益率序列：是否有复权、停牌、异常值处理。
- 持仓权重：是否包含现金、杠杆、衍生品和隐性敞口。
- 协方差矩阵：样本长度、滚动窗口、收缩方法是否合理。
- 因子暴露：市场、行业、市值、价值、成长、动量、利率等。
- 流动性数据：成交量、价差、盘口深度、成交量占比。
- 压力场景：历史危机、利率上行、流动性枯竭、政策冲击。

## 四、从模型到动作

风险模型必须能触发动作，否则只是报告装饰。

| 触发器 | 动作 |
|---|---|
| 组合波动率超过预算 | 降低总仓位或对冲 beta |
| 单一因子暴露过高 | 调整持仓，降低集中风险 |
| 最大回撤接近警戒线 | 暂停新开仓，复查策略 |
| 流动性低于门槛 | 降低仓位或延长调仓周期 |
| 压力测试亏损超限 | 降杠杆、提高现金、减少拥挤持仓 |

## 五、风险建模误区

- 把历史最大回撤当成未来最大风险。
- 只看日常波动，不看跳空、停牌、涨跌停和极端流动性。
- 用复杂模型掩盖简单问题：仓位太大、相关性太高。
- 模型没有连接到仓位规则和止损规则。
- 只在亏损后才修改风险参数。

## 七、组合风险建模案例

假设组合包含股票多头、债券ETF、商品ETF和一组量化策略，可以按以下步骤建模：

| 步骤 | 输出 |
|---|---|
| 暴露拆解 | 权益 beta、久期、信用、商品、外汇、行业和因子暴露 |
| 日常风险 | 年化波动、VaR、CVaR、最大回撤、风险贡献 |
| 极端场景 | 股债同跌、信用利差扩大、商品跳涨、流动性枯竭 |
| 流动性评估 | 每个持仓退出所需天数、成交量占比和冲击成本 |
| 动作规则 | 降仓、对冲、暂停新开仓、提高现金比例 |

## 八、压力测试清单

- 历史场景：重放市场大跌、流动性危机、利率快速上行或信用风险暴露时期。
- 假设场景：股票跌 20%、利率上行 100bp、信用利差扩大、主要持仓无法成交。
- 组合场景：相关性从低相关变成同涨同跌，分散化失效。
- 操作场景：行情中断、订单拒单、接口延迟、模型输出异常。

## 九、模型治理

风险模型必须有版本、输入口径、参数、上线日期和负责人。每次模型结果触发动作，都要记录触发原因、执行动作和后续效果。若模型长期不触发任何动作，要检查阈值是否过宽；若频繁触发，要检查风险预算是否与策略波动不匹配。

## 最小可执行项目

### 策略一页纸

围绕 风险建模 写一张策略一页纸：收益来源、适用品种、信号、过滤器、入场、退出、仓位、风控、成本、容量、暂停条件和复盘指标。

| 检验 | 合格标准 |
|---|---|
| 逻辑 | 能解释为什么会有收益 |
| 回测 | 有样本外和参数扰动 |
| 成本 | 扣除手续费、滑点和冲击成本 |
| 风险 | 有最大回撤和连续亏损预案 |
| 实盘 | 有监控、降仓和暂停规则 |

### 常见失败

策略在报告中赚钱，却在实盘中输给交易成本、容量限制、信号衰减和执行错误。
