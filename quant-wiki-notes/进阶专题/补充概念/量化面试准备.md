# 量化面试准备

> [!note] 核心定位
> 量化面试准备不是背题库，而是把金融直觉、数学统计、编程能力、研究流程和实盘风险讲清楚。面试官真正想判断的是：你能不能把一个模糊投资想法变成可验证、可复现、可控风险的策略。

## 一、能力模块

| 模块 | 必备能力 | 典型问题 |
|---|---|---|
| 金融市场 | 资产类别、交易规则、费用、滑点、流动性 | 为什么回测收益会在实盘消失 |
| 统计数学 | 概率、回归、假设检验、时间序列、优化 | IC 和 t 值分别说明什么 |
| 因子研究 | 因子构建、中性化、分层回测、归因 | 如何判断一个因子不是行业暴露 |
| 机器学习 | 标签、特征、交叉验证、防泄漏、过拟合 | 为什么金融数据不能随机切分 |
| 编程工程 | Python、Pandas、NumPy、SQL、日志、版本管理 | 如何处理缺失值和异常行情 |
| 风控执行 | 仓位、回撤、成本、容量、订单状态 | 策略上线前需要哪些保护 |

## 二、项目讲述框架

准备至少两个能讲透的项目：一个研究型项目，一个工程型或实盘型项目。每个项目用同一套结构表达：

1. 背景：要解决什么投资问题，为什么值得做。
2. 数据：数据来源、字段、时间范围、清洗、复权、停牌和退市处理。
3. 方法：信号或模型如何生成，为什么有经济解释。
4. 验证：样本内、样本外、滚动窗口、参数扰动和基准对照。
5. 结果：收益、回撤、换手、成本、容量和失败案例。
6. 风控：仓位、止损、熔断、监控和暂停条件。
7. 反思：最大缺陷是什么，下一版会如何改。

## 三、常考知识清单

- 因子：去极值、标准化、中性化、IC、Rank IC、分层收益、换手、拥挤和衰减。
- 回测：前视偏差、幸存者偏差、交易成本、滑点、容量、样本外和过拟合。
- 组合：均值方差、风险平价、因子暴露、风险预算、相关性和再平衡。
- 衍生品：期货保证金、期权希腊字母、波动率、卖方尾部风险。
- 统计：p 值、置信区间、协方差、正则化、交叉验证、时间序列自相关。
- 工程：数据管道、任务调度、异常告警、日志追踪、模拟盘和实盘隔离。

## 四、代码题准备

代码题不只考语法，更考边界处理。练习时优先覆盖：

- 用 Pandas 计算日收益、滚动波动率、最大回撤和换手率。
- 实现一个简单的单因子分层回测。
- 对时间序列做训练集、验证集、测试集切分。
- 写函数处理缺失值、极端值、复权和停牌。
- 用清晰变量名和注释说明假设，不为了炫技写难读代码。

## 五、回答问题的原则

- 先给结论，再解释假设和限制。
- 不确定时说明不确定在哪里，不要编造精确答案。
- 讲策略时必须同时讲风险和失败场景。
- 讲模型时必须说明为什么不会数据泄漏。
- 讲收益时必须说明扣费后、样本外和容量约束。

## 最小可执行项目

### 模拟面试包

围绕 量化面试准备 准备一套可反复练习的模拟面试包：一页项目介绍、一个代码题、一个因子研究题、一个风控题和一个失败复盘题。每个回答都要先讲结论，再讲假设、限制和风险。

| 题型 | 练习任务 | 验收标准 |
|---|---|---|
| 项目题 | 用 3 分钟讲清一个策略项目 | 数据、方法、结果、风控都完整 |
| 代码题 | 手写最大回撤或分层回测 | 边界条件、缺失值和可读性合格 |
| 研究题 | 解释一个因子为何有效 | 有经济逻辑和样本外验证 |
| 风控题 | 说明实盘上线前检查 | 包含仓位、成本、容量和异常处理 |
| 反思题 | 讲一个项目失败点 | 能承认不足并提出改进 |

### 常见扣分点

只讲收益不讲回撤，只讲模型不讲数据可得性，只讲回测不讲实盘成本，只讲技术细节不讲投资逻辑，都会让回答显得不成熟。


## 进一步掌握

学习 `量化面试准备` 时，要把它当作研究基础设施，而不是单独的工具说明。真正重要的是数据能否复现、字段口径是否稳定、异常能否被发现，以及研究结果能否安全地走到实盘或组合复盘。一个工具只要影响数据、信号、订单或日志，就已经进入风险管理范围。

| 检查维度 | 应用方法 | 常见错误 |
|---|---|---|
| 数据契约 | 记录字段、频率、复权、时区、更新时间 | 只看能否下载，不看口径变化 |
| 复现能力 | 保存参数、版本、原始数据和处理脚本 | 回测结果无法重跑 |
| 生产边界 | 区分研究、仿真、小资金和正式账户 | 研究脚本误触实盘权限 |

### 训练任务

围绕 `量化面试准备` 做一次小型验收：选一个真实标的或数据表，记录获取路径、字段解释、缺失值、异常值、更新频率和与第二数据源的差异。最后写出“什么时候必须停止使用这份数据”的条件。掌握工具的标志不是会调用接口，而是能解释数据错误会怎样传导成投资错误。
