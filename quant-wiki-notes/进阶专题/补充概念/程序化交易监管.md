# 程序化交易监管

> [!note] 核心定位
> 程序化交易监管 属于量化研究和实盘交易的工具/接口/工程问题。它的重点是数据口径、稳定性、可追踪性、异常处理和从研究到实盘的安全边界。

## 一、程序化交易监管关注什么

程序化交易监管的核心不是反对量化，而是要求交易行为可识别、可控制、可追溯，防止系统性异常、市场冲击和不公平交易。对投资者和开发者来说，它意味着策略上线不能只看收益，还要满足合规、风控和运维要求。

监管通常关注：

- 策略是否备案或按规则报告。
- 订单频率、撤单率、成交比是否异常。
- 是否存在自成交、虚假申报、拉抬打压等行为。
- 系统是否有异常监控、风控阈值和紧急停止机制。
- 交易日志、参数版本和决策链路是否可追溯。
- 高频或大规模交易是否冲击市场稳定。

## 二、交易系统必须留下什么记录

| 记录 | 作用 |
|---|---|
| 策略版本 | 追踪哪一版代码和参数产生了订单 |
| 信号日志 | 说明为什么在某个时点产生买卖信号 |
| 风控日志 | 记录仓位、限额、撤单、拒单和异常触发 |
| 订单日志 | 保存报单、撤单、成交、失败和回报 |
| 数据日志 | 记录行情、财报、因子和接口时间戳 |
| 人工干预日志 | 记录暂停、恢复、参数调整和紧急操作 |

没有日志的程序化交易，本质上无法复盘，也无法向监管、风控或自己解释。

## 三、上线前检查清单

1. 策略逻辑是否有书面说明，收益来源和风险来源是否清楚。
2. 数据源异常、延迟、缺失、重复时是否会停止交易。
3. 单笔、单日、单策略、单账户风险限额是否设置。
4. 撤单率、报单频率、成交量占比是否可控。
5. 是否有模拟盘、小资金、灰度上线流程。
6. 是否有一键暂停、断线重连、失败回滚和人工接管。
7. 是否保存完整日志，能还原每一笔订单。

## 四、常见合规与运营风险

- 回测策略直接实盘运行，没有经过仿真和灰度。
- 系统异常时继续下单，造成重复报单或反向持仓。
- 参数临时调整无记录，事后无法解释盈亏来源。
- 订单频率和撤单率超出预期，引发交易所风控。
- 使用未经确认的数据源，导致错误信号。
- 未区分研究环境、仿真环境和生产环境。

## 五、程序化交易者的底线

程序化交易的优势是纪律和速度，风险也是纪律和速度：一旦规则错，系统会比人更快地放大错误。因此优秀交易者必须把监管、风控和运维当作策略的一部分，而不是上线后的附属工作。

## 八、异常情景演练

`程序化交易监管` 必须做异常演练。实盘系统最容易在少见但高压的时刻出错，例如行情延迟、接口断线、订单回报重复、合约切换、权限变更或数据字段异常。

| 异常 | 应对要求 |
|---|---|
| 行情中断 | 停止开新仓，保留当前状态 |
| 下单失败 | 不自动无限重试，记录失败原因 |
| 重复回报 | 订单状态去重，避免重复持仓 |
| 参数错误 | 版本回滚，禁止盘中无记录修改 |
| 风控触发 | 先停交易，再排查原因 |

## 深度案例与训练

### 概念转应用

学习 程序化交易监管 时，把它放到一个真实投资问题里：它影响收益判断、风险判断、估值判断、仓位判断还是执行判断。只有能改变行动的概念，才值得进入交易清单。

### 三步练习

1. 写一句话定义，不超过三十字。
2. 找一个成功案例和一个失败案例。
3. 把概念转成一条交易前检查项和一条交易后复盘项。

### 失效边界

概念越抽象，越容易被滥用。遇到不能量化、不能验证、不能复盘的结论，要降低仓位和信心等级。

## 最小可执行项目

### 概念应用卡

围绕 程序化交易监管 写一张概念应用卡：定义、适用场景、反例、可观察数据、交易动作和复盘问题。概念必须连接到真实行动。

| 问题 | 输出 |
|---|---|
| 它解释什么 | 收益、风险、成本、流动性或行为 |
| 它依赖什么 | 数据、市场环境和制度前提 |
| 它改变什么 | 买入、卖出、仓位、对冲或不交易 |
| 它何时失效 | 反例和退出条件 |

### 验收标准

能用一个真实案例说明它如何改变决策。


## 进一步掌握

学习 `程序化交易监管` 时，要把监管要求放在策略设计之前，而不是上线之后再补。程序化交易的风险不仅来自亏损，还来自异常交易、频繁撤单、穿透式管理、账户权限、日志留存和交易所规则变化。合规失败会让本来有效的策略失去执行资格。

| 检查维度 | 应用方法 | 常见错误 |
|---|---|---|
| 交易行为 | 监控频率、撤单率、报单量和异常价格 | 只控制收益风险，不控制交易行为 |
| 账户权限 | 明确谁能改参数、启动和暂停系统 | 盘中临时修改无记录 |
| 留痕审计 | 保存信号、订单、回报、风控和人工操作 | 出问题后无法还原经过 |

### 训练任务

围绕 `程序化交易监管` 写一张上线合规清单：策略说明、账户范围、频率限制、异常处理、日志保存、人工接管和复盘机制。优秀交易者不仅要知道怎样赚钱，还要知道哪些行为不能做、哪些边界不能碰。
