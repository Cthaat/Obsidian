# 量化回测

> [!note] 核心定位
> 量化回测 属于策略研究和组合落地主题。重点是先定义收益来源，再验证数据、成本、风险、容量和失效条件，最后写成可监控的交易流程。

## 一、回测的本质

量化回测不是证明策略一定赚钱，而是在历史数据中检验一条投资假设是否经得起成本、风险和样本外变化的挑战。它回答的是：

- 这条规则是否有可重复收益来源？
- 收益来自 alpha、beta、风格暴露、流动性补偿，还是数据错误？
- 最大回撤、最长亏损期和尾部损失是否可承受？
- 扣除交易成本后是否仍有意义？
- 实盘执行是否可能接近回测结果？

回测结果越漂亮，越应该先怀疑，而不是先兴奋。

## 二、回测流程

| 步骤 | 关键问题 |
|---|---|
| 假设定义 | 策略赚的是什么钱，为什么市场会给这笔钱 |
| 数据准备 | 是否有复权、停牌、退市、公告日、缺失值和异常值处理 |
| 规则实现 | 信号、下单、成交、持仓、调仓是否符合真实交易时序 |
| 成本建模 | 佣金、印花税、滑点、冲击成本、借券/融资成本是否计入 |
| 风险评估 | 年化收益、波动率、夏普、最大回撤、Calmar、尾部风险 |
| 稳健性检验 | 参数敏感性、滚动窗口、样本外、不同市场环境 |
| 实盘映射 | 容量、流动性、订单执行和监控机制是否可行 |

## 三、最危险的回测偏差

| 偏差 | 表现 | 修正方法 |
|---|---|---|
| 前视偏差 | 使用了交易时点之后才知道的信息 | 严格按时间戳和公告日取数 |
| 幸存者偏差 | 只回测仍然存在的股票 | 纳入退市、ST、停牌和历史成分变化 |
| 数据挖掘 | 反复调参直到曲线好看 | 保留样本外和参数稳定性检查 |
| 成本低估 | 回测成交完美，实盘滑点巨大 | 加入价差、成交量占比、冲击成本 |
| 容量错觉 | 小资金有效，大资金无法成交 | 估算每日成交量占比和清仓天数 |
| 基准错配 | 策略只是承担了市场 beta | 与合适基准和因子暴露比较 |

## 五、如何读回测结果

不要只看累计收益曲线。更重要的是看收益来自哪里：

- 如果收益主要来自少数几天，策略可能依赖极端事件。
- 如果回撤长期不恢复，可能不适合真实资金。
- 如果换手很高，成本小幅上升就可能摧毁收益。
- 如果参数稍变就失效，可能是过拟合。
- 如果所有策略都在同一阶段赚钱，可能只是市场 beta。

好的回测不是“无敌曲线”，而是清楚暴露自己的弱点，让你知道什么情况下不能用它。

## 八、策略上线前的最后一问

`量化回测` 上线前，要从“能赚钱”转向“亏钱时怎么办”。策略在历史上赚钱并不稀奇，能解释亏损、限制亏损、复盘亏损，才是真正可用。

| 问题 | 最低答案 |
|---|---|
| 为什么赚钱 | 清晰收益来源，不依赖玄学参数 |
| 为什么会亏 | 明确失效环境和风险来源 |
| 亏多少停 | 有回撤、波动、连续亏损阈值 |
| 如何降级 | 降仓、暂停、回到基准策略 |
| 如何复盘 | 数据、信号、执行、组合、心理分开记录 |

## 深度案例与训练

### 策略拆解

学习 量化回测 时，写清楚策略的六个零件：信号、过滤器、入场、退出、仓位和风控。任何说不清这六项的策略，都还只是想法。

### 验证路径

1. 用最简单规则建立基准收益。
2. 加入手续费、滑点、冲击成本和容量约束。
3. 做样本外、滚动窗口和参数扰动。
4. 统计收益来自哪些时期、哪些品种和哪些市场状态。

### 实盘监控

上线后关注信号命中率、成交质量、回撤、换手、资金占用和异常订单。策略失效时优先降仓或暂停，而不是继续调参寻找安慰。

## 最小可执行项目

### 策略一页纸

围绕 量化回测 写一张策略一页纸：收益来源、适用品种、信号、过滤器、入场、退出、仓位、风控、成本、容量、暂停条件和复盘指标。

| 检验 | 合格标准 |
|---|---|
| 逻辑 | 能解释为什么会有收益 |
| 回测 | 有样本外和参数扰动 |
| 成本 | 扣除手续费、滑点和冲击成本 |
| 风险 | 有最大回撤和连续亏损预案 |
| 实盘 | 有监控、降仓和暂停规则 |

### 常见失败

策略在报告中赚钱，却在实盘中输给交易成本、容量限制、信号衰减和执行错误。
