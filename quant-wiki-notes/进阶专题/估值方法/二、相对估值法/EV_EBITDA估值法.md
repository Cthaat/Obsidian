# EV/EBITDA估值法

> [!note] 💡 概念解析
> EV/EBITDA是企业价值倍数估值法，排除融资决策（利息、负债）和会计政策（折旧、摊销、税率）的影响，纯粹从运营效率角度衡量公司价值，是跨行业比较的"净化器"。

## 一、计算公式

$$EV/EBITDA = \frac{\text{企业价值(EV)}}{\text{息税折旧摊销前利润(EBITDA)}}$$

### 企业价值（EV）

$$EV = \text{市值} + \text{总负债} - \text{现金及现金等价物}$$

EV代表收购整个公司需要支付的总价格，比市值更全面地反映公司价值。

### EBITDA

$$EBITDA = \text{净利润} + \text{利息} + \text{所得税} + \text{折旧} + \text{摊销}$$

EBITDA大致代表主营业务的现金流，剔除了资本结构和折旧政策的影响。

## 二、与PE的对比

| 比较维度 | PE | EV/EBITDA |
|---------|-----|-----------|
| 分子 | 股权价值（市值） | 企业价值（EV） |
| 分母 | 净利润 | EBITDA |
| 资本结构影响 | 受影响 | 不受影响 |
| 折旧政策影响 | 受影响 | 不受影响 |
| 税率影响 | 受影响 | 不受影响 |
| 跨国可比性 | 较差 | 较好 |

## 三、适用场景

### 最适合
- **资本密集型行业**：航空、电信、能源、制造业
- **高负债行业**：剔除了资本结构差异
- **折旧摊销重的行业**：核电、酒店、物业出租
- **跨行业/跨国比较**：剔除了税率、资本结构等干扰因素
- **并购定价**：最早用作收购兼并的定价标准

### 不适用
- 轻资产、低资本支出的公司
- 需要大量资本开支维持运营的公司（可能过于"美化"盈利）

> [!warning] 注意事项
> - EBITDA**不等于**真正的现金流，忽略了营运资本变动
> - 计算相对复杂
> - 与PE一样不能体现企业未来的成长性
> - 需结合其他估值法使用

## 四、操作方法

$$\text{公司股权价值} = \text{可比公司EV/EBITDA} \times \text{公司EBITDA} - \text{债务价值}$$

> [!example] 计算示例
> 公司D：市值2000万，净债务500万，现金100万，EBITDA 500万
> - EV = 2000 + 500 - 100 = 2400万
> - EV/EBITDA = 2400/500 = 4.8
> - 若同行业平均EV/EBITDA为8，则公司D可能被低估

## 实战掌握清单

> [!tip] 交易者视角
> EV/EBITDA估值法 的学习重点不是记住术语，而是把它放进研究、组合、执行和复盘的闭环。估值不是算出一个精确数字，而是在现金流、增长、折现率、资本结构和市场情绪之间建立合理区间。

### 关键判断

- 确认企业处于成长期、成熟期、周期底部还是困境修复。
- 选择与商业模式匹配的估值方法，而不是机械套PE或DCF。
- 拆分利润质量、现金流质量、资本开支和负债约束。

### 落地动作

1. 建立保守、基准、乐观三种情景，并做敏感性分析。
2. 把绝对估值与相对估值交叉验证。
3. 估值结论必须连接到仓位、买入区间、退出区间和安全边际。

### 失效边界

- 用高增长假设掩盖商业模式问题。
- 忽略周期顶部的低估值陷阱。
- 把会计利润当作可分配现金流。

### 复盘问题

- 这项知识改变了哪一个具体决策：标的、方向、仓位、退出、对冲还是不交易？
- 如果判断相反，最大亏损、最长恢复期和退出触发条件是什么？
- 有没有一个更简单的基准方法可以取得相近结果？

## 深度案例与训练

### 估值练习

围绕 EV/EBITDA估值法 选择三家公司：高成长、成熟现金流和周期行业。分别计算估值区间，并写出增长率、利润率、折现率、资本开支和负债对结果的影响。

### 交叉验证

- 相对估值要看同业差异和周期位置。
- 绝对估值要做情景分析和敏感性分析。
- 估值结论必须连接安全边际、仓位和退出规则。

### 常见错误

用单一指标判断贵贱、把历史高增长外推太远、在周期顶部相信低PE。

## 最小可执行项目

### 估值模型卡

围绕 EV/EBITDA估值法 建一张估值模型卡：适用企业、核心变量、估值公式、敏感性分析、相对估值对照、安全边际和退出规则。

| 检查点 | 问题 |
|---|---|
| 现金流 | 利润是否真实可分配 |
| 增长 | 增长是否可持续 |
| 折现率 | 风险是否被低估 |
| 同业对比 | 是否存在商业模式差异 |
| 周期位置 | 当前利润是否在顶部或底部 |

### 验收标准

估值输出必须是区间和情景，而不是单点价格。
