# AI高频交易前沿

> [!note] 概述
> 高频交易（HFT）是机器学习最具挑战性的应用领域之一。微秒级的决策要求、海量数据流、极低信噪比，使得传统ML方法需要特殊适配。

## HFT对ML的特殊要求

| 要求 | 传统ML | HFT ML |
|------|--------|--------|
| 延迟 | 毫秒-秒级 | 微秒级 |
| 数据量 | GB级/天 | TB级/天 |
| 特征工程 | 离线预计算 | 在线流式计算 |
| 模型推理 | CPU/GPU | FPGA/定制硬件 |
| 更新频率 | 周/月 | 实时在线 |

## HFT中的核心ML应用

### 1. 订单流预测

```
订单簿（LOB）数据：
├── 买卖挂单量（5档/10档）
├── 成交记录（逐笔）
├── 撤单信息
└── 价差变化

→ LSTM/CNN → 预测短期价格方向（毫秒-秒级）
```

### 2. 做市商策略优化

- 预测最优买卖报价
- 库存风险管理
- 对手方行为建模

### 3. 异常检测

- 市场操纵识别（刷单、幌骗）
- 流动性枯竭预警
- 闪崩风险预测

### 4. 执行算法

- VWAP/TWAP执行的ML增强
- 智能订单路由
- 流动性预测

## 基础设施要求

| 层级 | 需求 | 实现方案 |
|------|------|---------|
| 网络 | <1μs延迟 | 交易所托管机房 |
| 数据 | 实时流处理 | Kafka/Redpanda |

| 推理 | <10μs | FPGA/GPU kernel |
| 存储 | 时序DB | ClickHouse/Kdb+ |

## 关键挑战

1. **信噪比极低**：Tick级别数据99%是噪声
2. **分布漂移快**：市场微观结构持续变化
3. **竞争对手也在用ML**：不断军备竞赛
4. **成本极高**：机房、数据、硬件投入百万起
5. **监管风险**：交易所/监管对HFT审查趋严

## 个人/HFT入门建议

1. 从比特币交易所开始（门槛低、数据公开）
2. 使用L2（订单簿）数据而非Tick
3. 从分钟/秒级做起，再逐步到毫秒级
4. 关注做市策略（稳定性优于投机性策略）
5. 成本建模比信号生成更重要

## 实战掌握清单

> [!tip] 交易者视角
> AI高频交易前沿 的学习重点不是记住术语，而是把它放进研究、组合、执行和复盘的闭环。机器学习在交易中的价值是处理非线性和高维特征，但最大的敌人是泄漏、过拟合和不可交易。

### 关键判断

- 明确标签定义、预测周期、样本切分和特征可得时间。
- 用时间序列交叉验证而不是随机打乱。
- 比较简单线性模型、树模型和深度模型的真实增益。

### 落地动作

1. 先建立可解释基准，再增加复杂度。
2. 检查特征重要性、模型稳定性、换手和容量。
3. 上线后监控数据漂移、预测分布漂移和收益衰减。

### 失效边界

- 标签泄漏。
- 样本外表现坍塌。
- 模型信号无法低成本成交。

### 复盘问题

- 这项知识改变了哪一个具体决策：标的、方向、仓位、退出、对冲还是不交易？
- 如果判断相反，最大亏损、最长恢复期和退出触发条件是什么？
- 有没有一个更简单的基准方法可以取得相近结果？

## 深度案例与训练

### ML研究卡

围绕 AI高频交易前沿 写清楚标签、特征、时间切分、模型、基准、评价指标和上线监控。金融机器学习的关键不是拟合能力，而是防泄漏和样本外稳定。

### 实验顺序

1. 先用简单模型建立基准。
2. 再加入复杂特征或复杂模型。
3. 用时间序列验证和滚动窗口评估。
4. 将预测转成组合，纳入成本、换手和容量。

### 失效监控

关注数据漂移、预测分布漂移、成交质量下降和收益归因变化。

## 最小可执行项目

### 样本外挑战

围绕 AI高频交易前沿 做一次样本外挑战：固定模型后，在未见过的时间区间检验预测、换手、成本和回撤，不允许根据结果反复修改参数。

| 检查点 | 目标 |
|---|---|
| 基准 | 简单模型可比较 |
| 防泄漏 | 特征时点正确 |
| 稳定性 | 滚动窗口不过度波动 |
| 可交易 | 成本后仍有效 |
| 监控 | 上线后能发现漂移 |

### 验收标准

模型复杂度必须换来真实的样本外收益。
