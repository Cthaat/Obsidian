# AI高频交易前沿

> [!note] 概述
> 高频交易（HFT）是机器学习最具挑战性的应用领域之一。微秒级的决策要求、海量数据流、极低信噪比，使得传统ML方法需要特殊适配。

## HFT对ML的特殊要求

| 要求 | 传统ML | HFT ML |
|------|--------|--------|
| 延迟 | 毫秒-秒级 | 微秒级 |
| 数据量 | GB级/天 | TB级/天 |
| 特征工程 | 离线预计算 | 在线流式计算 |
| 模型推理 | CPU/GPU | FPGA/定制硬件 |
| 更新频率 | 周/月 | 实时在线 |

## HFT中的核心ML应用

### 1. 订单流预测

```
订单簿（LOB）数据：
├── 买卖挂单量（5档/10档）
├── 成交记录（逐笔）
├── 撤单信息
└── 价差变化

→ LSTM/CNN → 预测短期价格方向（毫秒-秒级）
```

### 2. 做市商策略优化

- 预测最优买卖报价
- 库存风险管理
- 对手方行为建模

### 3. 异常检测

- 市场操纵识别（刷单、幌骗）
- 流动性枯竭预警
- 闪崩风险预测

### 4. 执行算法

- VWAP/TWAP执行的ML增强
- 智能订单路由
- 流动性预测

## 基础设施要求

| 层级 | 需求 | 实现方案 |
|------|------|---------|
| 网络 | <1μs延迟 | 交易所托管机房 |
| 数据 | 实时流处理 | Kafka/Redpanda |
| 特征 | 在线计算 | Flink/Custom C++ |
| 推理 | <10μs | FPGA/GPU kernel |
| 存储 | 时序DB | ClickHouse/Kdb+ |

## 关键挑战

1. **信噪比极低**：Tick级别数据99%是噪声
2. **分布漂移快**：市场微观结构持续变化
3. **竞争对手也在用ML**：不断军备竞赛
4. **成本极高**：机房、数据、硬件投入百万起
5. **监管风险**：交易所/监管对HFT审查趋严

## 个人/HFT入门建议

1. 从比特币交易所开始（门槛低、数据公开）
2. 使用L2（订单簿）数据而非Tick
3. 从分钟/秒级做起，再逐步到毫秒级
4. 关注做市策略（稳定性优于投机性策略）
5. 成本建模比信号生成更重要
