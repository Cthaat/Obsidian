# 机器学习算法全景

> [!note] 概述
> 系统介绍量化交易中常用的机器学习算法，从传统ML到深度学习，覆盖每类算法的原理、适用场景和交易应用。

## 一、传统机器学习

### 支持向量机（SVM）

**原理**：寻找最优超平面将不同类别分开。

**量化应用**：
- 股票涨跌二分类预测
- 异常市场状态识别
- 结合多个因子做信号生成

### 随机森林（Random Forest）

**原理**：多棵决策树的集成，通过投票/平均提高稳定性。

**量化应用**：
- 多因子选股（非线性关系捕捉）
- 特征重要性分析
- 抵抗过拟合（集成天然正则化）

> 适合处理因子之间的非线性交互关系，比线性回归更具表达力。

### XGBoost / LightGBM

**原理**：梯度提升树，逐棵添加弱学习器。

**量化应用**：
- 因子挖掘竞赛的主力模型
- 中低频选股策略
- 特征工程自动化

## 二、深度学习模型

### LSTM（长短期记忆网络）

```python
# LSTM核心：三个门控机制
# 遗忘门 → 决定丢弃哪些信息
# 输入门 → 决定存储哪些新信息
# 输出门 → 决定输出什么
```

**量化应用**：
- 时序价格预测（日线/分钟级）
- 技术指标模式识别
- 订单流异常检测

**关键考虑**：
- 时间序列必须按时间顺序划分训练/测试集（不可随机shuffle）
- 需要大量数据，避免参数过多
- 金融数据信噪比低，需合理设计标签

### GRU（门控循环单元）

LSTM的简化版，减少一个门控，参数更少。

| 对比 | LSTM | GRU |
|------|------|-----|
| 参数量 | 较多 | 较少（约少25%） |
| 训练速度 | 较慢 | 较快 |
| 长期记忆 | 更强 | 略弱 |
| 适合场景 | 长序列 | 中等序列 |

### Transformer

**原理**：基于自注意力机制，捕捉全局依赖关系。

**量化优势**：
- 并行计算，训练效率高
- 多头注意力捕捉多时间尺度关系
- 对长序列表现优异
- 适合多资产联合建模

### CNN在金融的应用

- 将价量数据转化为"图像"进行模式识别
- K线形态识别
- 多时间维度卷积

## 三、生成模型

### GAN（生成对抗网络）

**量化应用**：
- **数据增强**：生成合成市场数据训练策略
- **特征生成**：GAN生成新时序特征增强选股
- **风险模拟**：生成极端市场场景压力测试

> 西南证券研报：GAN_GRU因子在全A市场RankIC达7.03%，多空年化46.64%

### AutoEncoder（自编码器）

**量化应用**：
- 因子降维：将高维因子压缩到低维
- 去噪：去除市场数据中的噪声
- 特征提取：从原始数据学习有效表征

## 四、强化学习（RL）

| 算法 | 特点 | 交易应用 |
|------|------|---------|
| DQN | 离散动作空间 | 买/卖/持有决策 |
| PPO | 连续动作空间 | 仓位比例控制 |
| A2C/A3C | Actor-Critic架构 | 多目标优化 |

**核心优势**：直接学习"怎么做"而非"预测什么"，适合交易决策。

## 五、模型训练注意事项

### 时间序列的特殊性

```
❌ 错误：随机划分训练/测试集
   (用未来数据训练 → 预测过去 → 泄漏)

✅ 正确：保持时间顺序
   训练集: [2015-2019]
   验证集: [2020-2021]
   测试集: [2022-2023]
```

### 防止过拟合

- 使用时间序交叉验证（非随机K折）
- 加入正则化（L1/L2/Dropout）
- 早停（Early Stopping）
- 简化模型（参数量适配数据量）

### 标签设计

| 回归标签 | 分类标签 |
|---------|---------|
| 未来N天收益率 | 上涨/下跌 |
| 超额收益（vs指数） | 排名分位数 |
| 实现波动率 | 趋势延续/反转 |

## 课程化学习补充

> [!important] 学习定位
> 机器学习在交易中是预测和排序工具，不是收益机器；关键在数据切分、特征稳定性、可解释性和上线监控。本文仅用于学习、研究与复盘，不构成任何投资建议。

### 必须掌握的问题

- 训练/验证/测试是否按时间切分
- 特征是否可在交易时点获得
- 收益标签是否扣成本
- 模型漂移是否有监控

### 实战应用流程

1. 先写清楚你的投资假设：为什么这个信号、资产或方法应该产生收益。
2. 明确数据口径：样本范围、更新时间、复权/分红/停牌处理和交易日历。
3. 做最小可行验证：先用简单规则验证方向，再逐步加入复杂模型。
4. 把成本和约束前置：手续费、滑点、冲击成本、保证金、流动性和容量都要进入测算。
5. 上线后持续复盘：记录信号、下单、成交、持仓、回撤和失效原因。

### 风险与失效条件

- 随机切分造成泄漏
- 高维特征过拟合
- 黑箱模型难以风控
- 训练收益无法覆盖交易成本

### 复盘问题

- 这笔交易或这套模型赚的是什么钱：风险补偿、行为偏差、流动性溢价，还是偶然噪音？
- 如果市场环境反过来，最大亏损和最长恢复期会是多少？
- 当前结论是否依赖某个不可持续假设，例如低利率、低波动、充裕流动性或监管套利？
- 有没有一个更简单的基准策略能取得接近效果？
