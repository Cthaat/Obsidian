# 机器学习交易实操导航

> [!note] 核心问题
> 金融里的 ML 最容易变成「高维过拟合器」。本篇在进阶「机器学习交易」与阶段五 [[机器学习与AI在量化中的应用]] 之间搭桥：先校准期待，再给 **防泄露清单、工具路径（Qlib/简单基线）、2 周抽样计划**，并明确 **何时不该上 ML**。

## 学习目标

1. 口述 ML 在量化中擅长/不擅长之事。  
2. 用清单检查标签、特征、时间切分是否泄露。  
3. 在「线性/树模型/深度学习」间做出有依据的先后顺序。  
4. 知道 Qlib 与教学价量脚本的层级差。  
5. 完成一页「ML 实验卡」，而不是先训 LSTM。  

## 何时不该上 ML（先过这关）

| 条件 | 建议 |
|---|---|
| 阶段零双均线/研究包未闭环 | 先 [[阶段零完成验收]] |
| 不会解释自己的标签定义 | 先写清预测目标 |
| 只有小样本短区间 | 用简单规则 + 强样本外 |
| 想「自动挖 alpha」无经济逻辑 | 停；回 [[因子投资实操导航]] |
| 无法说明 purge/embargo | 禁止随机 K 折当结论 |

## 防泄露与验证清单（必做）

| # | 检查 | 合格标准 |
|---:|---|---|
| 1 | 标签时点 | 标签只用未来收益时，特征截止于信号日之前 |
| 2 | 特征时点 | 财务/分析师类必须 PIT/公告日 |
| 3 | 标准化 | 截面 z-score 不得用全样本统计量偷窥 |
| 4 | 切分 | 按时间前向切分，禁止打乱时间的随机 CV 当主证据 |
| 5 | Purge / Embargo | 训练/测试边界清洗重叠标签窗口 |
| 6 | 多重试错 | 搜了很多结构后要打折「显著性」 |
| 7 | 成本 | 换手进入回测；否则纸面 alpha |
| 8 | 基准 | 相对买入持有/简单因子，而非只报准确率 |
| 9 | 可交易性 | 涨跌停、停牌、最小单位 |
| 10 | 漂移监控 | 若谈上线：分布漂移与暂停规则 |

详见课程文中的时序验证与 [[过拟合识别与防御]]、[[回测方法论]]。

## 模型选择阶梯（实操默认）

```text
1. 简单规则 / 线性（可解释基线）
2. 树模型（非线性、仍要严格时间切分）
3. 序列/深度模型（最后；样本与工程成本高）
```

| 优先读 | 原因 |
|---|---|
| 课程 [[机器学习与AI在量化中的应用]] | 校准期待与 CV |
| 本专题「综合指南」抽样 | 体系化笔记 |
| 「从线性因子到端到端」 | 理解演进，勿跳步 |
| 算法全景 / LSTM 篇 | 有基线后再读 |
| 论文精选 | 当扩展，不当第一周 |

完整树：[[机器学习交易/目录]]。

## 常见误区

| 误区 | 更好的理解 |
|---|---|
| 准确率高=能赚钱 | 分类阈值、成本、非平稳 |
| LSTM 一定强于线性 | 常更易过拟合 |
| 随机 CV 显得严谨 | 金融时间泄露 |
| 特征越多越好 | 噪声维度爆炸 |
| Qlib 一跑就有产品 | 仍是研究平台 |
