# 多因子Alpha挖掘实战

> [!note] 核心要点
> 多因子模型是量化投资的核心工具。本文走的是**实战路线**：不重复讲"因子为何有效"（那是 [[因子投资入门]] 的事），也不重复研究流程总图（那是 [[Alpha因子研究指南]] 的事），而是**手把手把数据处理到因子合成的每一步写成可运行的代码骨架**——去极值、标准化、中性化、IC筛选、合成，一行行落地。

## 第零章：量化投资，本质是场关于"残差"的狩猎

量化投资的本质，是寻找无法被已知因子解释的超额收益（残差），即真正的 Alpha。换句话说：

$$
r_{i} = \underbrace{\alpha_i}_{\text{我们要找的}} + \underbrace{\sum_k \beta_{i,k} F_k}_{\text{已知风格/风险}} + \underbrace{\epsilon_i}_{\text{噪声}}
$$

整套实战的目标，就是把右边的已知部分（市场、行业、规模等）**剥干净**，让 $\alpha_i$ 露出来。这也解释了为什么"中性化"是实战里最关键的一步。

## 第一章：挖掘流程总览

```mermaid
graph LR
    A[原始数据] --> B[特征工程]
    B --> C[去极值]
    C --> D[标准化]
    D --> E[中性化]
    E --> F[单因子IC检验]
    F --> G{合格?}
    G -- 否 --> A
    G -- 是 --> H[多因子合成]
    H --> I[组合优化与回测]
```

| 环节 | 解决什么问题 | 实战难点 |
|------|--------------|----------|
| 特征工程 | 把原始数据变成因子 | 比例化、对数化、避免量纲混乱 |
| 去极值 | 防止异常值主导结果 | 阈值选取 |
| 标准化 | 让不同因子可加 | 截面 vs 时序 |
| 中性化 | 剥掉行业/市值暴露 | 回归残差的正确用法 |
| IC检验 | 筛掉无效因子 | 区分IC高与可交易 |
| 合成 | 把多因子拼成一个分 | 权重、去相关 |

## 第二章：特征工程——把原始数据变成因子

原始数据不能直接当因子。最常见的处理是**比例化**（消除规模影响）和**对数化**（压缩长尾）。

```python
import numpy as np
import pandas as pd

# 示例：构建几个基础因子（df 含每只股票每期的原始字段）
df["value_pb"]   = 1.0 / df["pb"]                 # 价值：取倒数，越便宜越大
df["quality_roe"] = df["net_profit"] / df["equity"]
df["mom_12m"]    = df["close"] / df["close_12m_ago"] - 1   # 12月动量
df["log_mktcap"] = np.log(df["mktcap"])           # 市值取对数，用于后续中性化
```

> [!tip] 实战小细节
> 估值类因子常**取倒数**（用 1/PE、1/PB 而非 PE、PB），这样"越便宜分越高"，方向统一、也避免负值/极端值带来的麻烦。

## 第三章：去极值——别让一只妖股毁掉整个因子

> [!warning] 极端值的破坏力
> 一只因子值离谱的股票（如刚扭亏导致 ROE 爆表），在标准化时会把整个分布拉偏，让其他股票的得分全部失真。**去极值是标准化前的必修课。**

常用 **MAD（中位数绝对偏差）** 法，比简单按分位数截断更稳健：

```python
def winsorize_mad(s, n=3):
    med = s.median()
    mad = (s - med).abs().median()
    upper, lower = med + n * 1.4826 * mad, med - n * 1.4826 * mad
    return s.clip(lower, upper)

# 按每个截面（每期）分别去极值
df["value_pb"] = df.groupby("date")["value_pb"].transform(winsorize_mad)
```

| 去极值方法 | 思路 | 优缺点 |
|------------|------|--------|
| 分位数截断 | 截到1%/99%分位 | 简单，但受样本分布影响 |
| 3σ法 | 截到均值±3标准差 | 对极端值本身敏感 |
| MAD法 | 用中位数，抗极端值 | **更稳健，推荐** |

## 第四章：标准化——让不同因子能相加

去极值后做 **Z-Score 标准化**，把每个因子在每个截面上变成均值0、标准差1，从而不同因子可比、可加：

$$
z_{i} = \frac{f_i - \mu_{\text{cross}}}{\sigma_{\text{cross}}}
$$

```python
def zscore(s):
    return (s - s.mean()) / (s.std() + 1e-9)

for col in ["value_pb", "quality_roe", "mom_12m"]:
    df[col] = df.groupby("date")[col].transform(zscore)
```

> [!note] 截面标准化，不是时序标准化
> 一定是**在同一天的所有股票之间**做标准化（按 date 分组），而不是对单只股票的历史做。我们比的是"今天谁更便宜/更强"，是横截面比较。

## 第五章：中性化——这是实战的灵魂

> [!important] 不中性化，你押的可能是行业而非因子
> 直接用 ROE 选股，可能选出一堆银行股——不是因为 ROE 这个因子有效，而是因为银行业整体 ROE 高。**中性化用回归剥掉行业和市值的影响，只留因子自己的部分（残差）。**

做法：把因子对"行业哑变量 + 对数市值"做截面回归，**取残差**作为中性化后的因子。

```python
import statsmodels.api as sm

def neutralize(group):
    y = group["value_pb"]
    # 行业哑变量 + 对数市值 作为自变量
    X = pd.get_dummies(group["industry"], drop_first=True)
    X["log_mktcap"] = group["log_mktcap"]
    X = sm.add_constant(X).astype(float)
    resid = sm.OLS(y, X).fit().resid       # 残差就是中性化后的纯因子
    return resid

df["value_pb_neu"] = df.groupby("date", group_keys=False).apply(neutralize)
```

中性化前后对比：

| | 中性化前 | 中性化后 |
|---|----------|----------|
| 因子含义 | 混杂行业/规模暴露 | 纯粹的因子超额 |
| 选股结果 | 易扎堆某行业/小盘 | 行业、规模均衡 |
| IC解读 | 可能是风格的IC | 才是因子真实的IC |

## 第六章：单因子IC检验——留谁、汰谁

对每个中性化后的因子，算 **RankIC 时间序列**，再看均值、ICIR 和正IC比例。完整指标定义见 [[因子检验与评价]]。

```python
from scipy.stats import spearmanr

def rank_ic(df, factor, fwd_ret="ret_fwd"):
    ic = df.groupby("date").apply(
        lambda g: spearmanr(g[factor], g[fwd_ret]).correlation
    )
    return ic

ic = rank_ic(df, "value_pb_neu")
print("IC均值:", ic.mean(), " ICIR:", ic.mean() / ic.std(),
      " 正IC占比:", (ic > 0).mean())
```

> [!warning] 三个数一起看，别只盯IC均值
> - **IC均值**：方向和强度（绝对值>0.03算可用，示例）。
> - **ICIR**：稳定性（>0.3~0.5更可靠，示例）。
> - **正IC占比**：方向一致性（持续偏向一边才靠谱）。
> 一个IC均值高但ICIR很低的因子，往往是"几次大赢+经常小输"，实盘体验极差。

## 第七章：多因子合成——拼成一个综合分

合格因子要合成。**合成前务必去相关**（见 [[因子分类体系]] 的相关性矩阵），否则高相关因子叠加只是重复下注。常用合成方式：

```python
# 方式一：等权合成（最稳健的起点）
factors = ["value_pb_neu", "quality_roe_neu", "mom_12m_neu"]
df["score_eq"] = df[factors].mean(axis=1)

# 方式二：ICIR加权（让更稳的因子权重更高）
icir = {f: (rank_ic(df, f).mean() / rank_ic(df, f).std()) for f in factors}
w = pd.Series(icir); w = w / w.abs().sum()
df["score_icir"] = sum(df[f] * w[f] for f in factors)
```

| 合成方式 | 适用 | 风险 |
|----------|------|------|
| 等权 | 因子少、相关低 | 忽略了因子强弱差异 |
| IC/ICIR加权 | 想突出有效因子 | 易过拟合历史IC |
| 最优化加权 | 有明确风险约束 | 最易过拟合，需正则化 |

> [!tip] 从等权起步
> 实战经验：**先用等权跑通整条链路**，确认逻辑和数据没问题，再考虑更复杂的加权。复杂加权带来的提升，常常抵不过它引入的过拟合风险。

## 第八章：组合优化与回测

综合分排序后取头部建仓，叠加换手率、个股/行业权重上限、交易成本约束。回测规范见 [[回测方法论]]，组合风控见 [[风险管理框架]]。机器学习式的非线性合成是另一条路，见 [[AI多因子选股策略]]。

## 第九章：常见陷阱与风险

> [!warning] 实战中的坑（每一条都让人亏过钱）
> - **过拟合**：在历史数据上表现完美，实盘失效——参数越多越危险。
> - **前视偏差**：用了当时还未公开的信息（财报时点、停牌处理）。
> - **生存偏差**：股票池漏掉已退市股，结论天然偏乐观。
> - **数据窥探**：在同一份数据上反复试，总能"试出"好因子。
> - **因子拥挤**：太多人用同一因子，超额被压缩甚至反转（见 [[Alpha衰减与因子生命周期]]）。
> - **顺序搞反**：去极值/标准化/中性化的顺序错了，结果全错。
> - **用时序标准化代替截面**：把横截面比较做成了时间序列比较，逻辑崩塌。

> [!tip] 一句话总结
> 实战的胜负，往往不在"用了多炫的模型"，而在**去极值、中性化、防过拟合这些不起眼的细节有没有做扎实。**

## 实战掌握清单

> [!tip] 交易者视角
> 多因子Alpha挖掘实战 的学习重点不是记住术语，而是把它放进研究、组合、执行和复盘的闭环。量化策略必须从清晰假设出发，经过数据验证、成本测算、风险控制和实盘监控，才可能成为可持续系统。

### 关键判断

- 写清楚收益来自动量、反转、价值、套利、波动率、流动性还是行为偏差。
- 确认信号、过滤器、入场、退出、仓位和风控。
- 看收益是否集中在少数时期、少数品种或少数参数。

### 落地动作

1. 做样本外、滚动窗口和参数扰动测试。
2. 把手续费、滑点、冲击成本、容量和失败交易纳入报告。
3. 上线后监控成交质量、信号衰减、回撤和异常订单。

### 失效边界

- 过拟合。
- 策略容量不足。
- 市场结构变化后没有停止机制。

### 复盘问题

- 这项知识改变了哪一个具体决策：标的、方向、仓位、退出、对冲还是不交易？
- 如果判断相反，最大亏损、最长恢复期和退出触发条件是什么？
- 有没有一个更简单的基准方法可以取得相近结果？
