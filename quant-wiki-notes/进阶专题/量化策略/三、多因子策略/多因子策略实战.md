# 多因子策略实战

> [!note] 实战应用
> 前面几篇讲原理、讲方法。这篇**只做一件事**：从头到尾走一个**可落地的 A 股多因子策略示例**——选定 5 个因子、合成打分、分层、回测、归因，并给出 Python 骨架。所有数字均为**示例/假设**，目的是把流程跑通，不是给出真实业绩承诺。

## 一、策略一句话设定

> **在中证 800 里，按"价值+质量+动量+低波+成长"五因子综合打分，每月选得分最高的 50 只等权持有，行业中性，扣交易成本。**

```mermaid

    A[中证800股票池] --> B[5因子打分]
    B --> C[分10层验证单调性]

    D --> E[月度调仓+扣成本]
    E --> F[业绩归因]
```

## 二、第 1 步：选定因子（含方向）

挑因子的标准是**逻辑清晰 + A 股有效 + 彼此不太重复**。承接 [[多因子策略核心原理]] 的"低相关"思想，特意跨了 5 个不同大类：

| 因子 | 代表指标 | 经济逻辑 | 方向（调正后） |
|------|------|------|:---:|
| 价值 | $1/PB$ | 买得便宜，估值修复 | 越大越好 |
| 质量 | ROE | 好公司长期溢价 | 越大越好 |
| 动量 | 过去 12 月剔除近 1 月收益 | 趋势延续 | 越大越好 |
| 低波 | 过去 60 日波动率 × $(-1)$ | 低波异象 | 越大越好 |
| 成长 | 净利润同比增速 | 公司基本面改善 | 越大越好 |

> [!tip] A 股特色的取舍
> A 股**小盘溢价、短期反转、低波、低换手**等效应历史上较显著。本例用"过去12月剔除近1月"的动量，正是为了**避开 A 股强烈的短期反转**——直接用近 1 月动量在 A 股常常是反指。

> [!warning] A 股的环境约束（务必牢记）
> - **政策影响大**：行业政策可瞬间改变风格，需行业中性 + 监控。
> - **散户比例高**：情绪驱动强，反转/波动类因子更有效。
> - **流动性分化**：小票容量有限，策略容量受限。
> - **T+1 + 涨跌停**：当天买不能卖、停板无法成交，回测必须如实模拟。

## 三、第 2 步：因子预处理与打分

严格执行 [[多因子量化指南]] 的流水线：**去极值 → 标准化 → 行业/市值中性化 → 方向对齐 → 合成**。

本例合成用 **IC_IR 加权**（兼顾有效与稳定，详见 [[多因子策略深度解析]]）：

$$
\text{Score}_i = \sum_{k=1}^{5} w_k\, \tilde z_{i,k}, \qquad w_k = \frac{ICIR_k}{\sum_j |ICIR_j|}
$$

> [!example] 权重示例（数字为假设）
> 用过去 24 个月 IC 算出的 ICIR 假设为：价值 1.2、质量 1.5、动量 0.9、低波 1.1、成长 0.6。
> 归一化后权重约为：质量 0.28、价值 0.22、低波 0.20、动量 0.17、成长 0.13。
> 解读：质量最稳故权重最高，成长 ICIR 最低故权重最低——**稳定性主导了配权**。

## 四、第 3 步：分层回测（验证单调性）

合成得分好不好，先看**分层**：把股票按综合得分从高到低分成 10 层，看各层下期收益是否**单调递减**。

| 层（按得分） | 假设年化收益 | 期望形态 |
|------|------|------|
| 第 1 层（最高分） | 18% | 最高 |
| 第 2 层 | 14% | ↓ |
| 第 5 层 | 9% | 居中 |
| 第 9 层 | 4% | ↓ |
| 第 10 层（最低分） | 1% | 最低 |
| **多空（L1−L10）** | **约 17%** | 单调差越大越好 |

> [!important] 分层比单点回测更可信
> 如果只有第 1 层好、中间乱跳，说明信号不稳，可能是**少数极端股票的运气**。**层层单调递减**才说明综合得分真正在"区分好坏"。先看到漂亮的分层，再谈组合回测。

## 五、第 4 步：组合回测（含成本）

取**第 1 层 / Top 50** 构建组合，月度调仓，行业中性，**如实扣成本**。

| 参数 | 设定（示例） |
|------|------|
| 股票池 | 中证 800（point-in-time） |
| 持仓数 | Top 50，等权 |
| 调仓频率 | 月度 |
| 行业约束 | 行业中性（贴近基准） |
| 交易成本 | 佣金双边 0.03% + 印花税卖出 0.05% + 滑点 0.1% |
| 缓冲带 | 掉出 Top50 但在 Top75 内不卖出 |

> [!warning] 成本与约束决定真假
> **同一套信号，扣不扣成本、加不加 T+1，回测曲线可能天差地别。** 纸面年化 20% 的策略，扣掉换手成本后可能只剩个位数。本例特意加缓冲带，就是为了把换手压下来、让收益扛得住成本。

评价指标（看这几个就够）：

| 指标 | 含义 | 关注点 |
|------|------|------|
| 年化超额收益 | 相对基准的 Alpha | 是否持续为正 |
| 信息比率 IR | 超额收益 / 跟踪误差 | **核心指标**，越高越好 |
| 最大回撤 | 最差的一段跌幅 | 能不能扛得住 |
| 月度胜率 | 跑赢基准的月份占比 | 稳定性 |
| 换手率 | 调仓频繁程度 | 直接关系成本 |

## 六、第 5 步：业绩归因（钱从哪来）

回测有超额收益还不够，要知道**这些 Alpha 到底来自哪个因子**。承接 [[多因子模型详解]] 的截面回归，把组合的超额收益拆回各因子暴露：

$$
R_{p,t}^{\text{超额}} = \sum_{k} \beta_{p,k}\, f_{k,t} + \varepsilon_t
$$

| 收益来源 | 假设贡献占比 | 解读 |
|------|------|------|
| 质量因子 | 35% | 主力贡献 |
| 价值因子 | 25% | 第二来源 |
| 低波因子 | 20% | 稳定贡献 |
| 动量因子 | 12% | 锦上添花 |
| 成长因子 | 5% | 贡献有限 |
| 残差（选股 α） | 3% | 因子之外的部分 |

> [!tip] 归因的两大用途
> 1. **验证逻辑**：如果超额收益主要来自一个**你没打算下注**的暴露（比如悄悄押了小盘），说明中性化没做好。
> 2. **指导迭代**：贡献长期接近 0 的因子（如本例成长仅 5%）可考虑剔除或替换，给系统瘦身。

## 七、可运行的 Python 骨架

> [!note] 说明
> 下面是**教学骨架**，省略了数据接口与异常处理，意在展示"流程怎么串"。真实落地需接入行情/财务数据源（如本地数据库或 Tushare 类接口）并补全细节。

```python
import numpy as np
import pandas as pd

FACTORS = ["value", "quality", "momentum", "low_vol", "growth"]

# ---------- 1. 预处理 ----------
def winsorize(s, n=3):
    med = s.median()
    mad = (s - med).abs().median() * 1.4826
    return s.clip(med - n * mad, med + n * mad)

def zscore(s):
    return (s - s.mean()) / (s.std() + 1e-9)

def neutralize(df, factor, by=["industry", "log_mktcap"]):
    # 对行业哑变量 + 市值回归，取残差（行业/市值中性化）
    X = pd.get_dummies(df[by[0]])
    X["log_mktcap"] = df[by[1]]
    X = np.column_stack([np.ones(len(df)), X.values])
    y = df[factor].values
    beta, *_ = np.linalg.lstsq(X, y, rcond=None)
    return y - X @ beta

# ---------- 2. 打分（IC_IR 加权）----------
def composite_score(panel, weights):
    score = pd.Series(0.0, index=panel.index)
    for f in FACTORS:
        z = zscore(winsorize(panel[f]))
        z = pd.Series(neutralize(panel.assign(**{f: z}), f), index=panel.index)
        score += weights[f] * z          # 方向须事先调正
    return score

# ---------- 3. 分层验证 ----------
def quantile_returns(score, fwd_ret, n=10):
    grp = pd.qcut(score, n, labels=False)
    return fwd_ret.groupby(grp).mean()   # 期望随分位单调递减

# ---------- 4. 组合 + 缓冲带 ----------
def build_portfolio(score, prev_holdings, top=50, buffer=75):
    ranked = score.sort_values(ascending=False)
    keep = [s for s in prev_holdings if s in ranked.index[:buffer]]  # 缓冲带
    new = [s for s in ranked.index[:top] if s not in keep]
    target = (keep + new)[:top]
    w = pd.Series(1.0 / len(target), index=target)  # 等权
    return w

# ---------- 5. 回测主循环（示意）----------
def backtest(dates, weights):
    holdings, curve = [], []
    for d in dates:
        panel = load_panel(d)                 # 取当期因子面板（point-in-time）
        score = composite_score(panel, weights)
        w = build_portfolio(score, holdings)
        ret = next_period_return(w, d)         # 已扣佣金/印花税/滑点
        curve.append(ret)
        holdings = list(w.index)
    return pd.Series(curve, index=dates)

# ---------- 6. 业绩归因（截面回归，示意）----------
def attribution(port_excess_ret, factor_returns):
    X = np.column_stack([np.ones(len(factor_returns))] + 
                        [factor_returns[f] for f in FACTORS])
    beta, *_ = np.linalg.lstsq(X, port_excess_ret, rcond=None)
    return dict(zip(["alpha"] + FACTORS, beta))
```

> [!warning] 代码里最容易藏的"未来函数"
> `load_panel(d)` 必须只返回 **d 时点收盘前真实可得** 的数据：财报要按**披露日**而非**报告期**对齐，指数成分要按当时真实成分。否则归因再漂亮也是自欺。

## 八、从回测到实盘的检查清单

| 检查项 | 通过标准 |
|------|------|
| 无未来函数 | 财报按披露日、成分按当时点 |
| 扣全成本 | 佣金+印花税+滑点+冲击 |
| T+1 / 涨跌停 | 当日买入不可卖、停板不可成交 |
| 分层单调 | 10 层收益基本单调递减 |
| 样本外稳健 | 留出样本外区间仍有正超额 |
| 换手可控 | 换手率在成本可承受范围 |
| 归因清晰 | Alpha 来源与策略意图一致 |

> [!important] 一句话收尾
> 一个能落地的多因子策略，**不是因子有多神，而是每个环节都经得起推敲**：因子有逻辑、分层单调、成本扣足、无未来函数、归因清楚。把这张清单逐项打勾，比追求回测曲线的好看重要得多。

## 风险管理

- 行业中性化（贴近基准、剥离行业押注）
- 市值中性化（避免无意中押小盘）
- 风险因子暴露控制（单票/单行业上限）
- 最大回撤限制（触发阈值降仓）

> [!warning] 实战常见误区
> 1. **回测惊艳、实盘平庸**：多半源于未来函数、未扣成本或过拟合权重。
> 2. **只看收益不看 IR/回撤**：高波动换来的收益不可持续。
> 3. **直接用近 1 月动量**：A 股短期反转强，容易做成反指。
> 4. **中性化偷工减料**：超额收益其实来自悄悄押注的小盘/单一行业。
> 5. **换手失控**：忽略缓冲带，收益被交易成本吃光。
