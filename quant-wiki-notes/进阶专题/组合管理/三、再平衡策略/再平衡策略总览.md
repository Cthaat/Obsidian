# 再平衡策略总览

> [!note] 核心要点
> 投资组合再平衡是维持目标风险特征的最简单杠杆。若不管理，60/40配置在长期牛市中可能偏离至80/20+的股票权重，增加未预期的下行风险。再平衡的主要目标是**风险控制**，而非通过市场时机来打败市场。

## 一、再平衡的本质

### 1.1 为什么需要再平衡

- 各类资产表现差异导致配置**漂移**
- 漂移可能使组合风险超出投资者承受能力
- 再平衡恢复预期风险暴露，实现"高抛低吸"

### 1.2 再平衡溢价

> [!tip] 重要认知
> 所谓"再平衡溢价"（买入表现不佳者、卖出赢家）的可衡量收益大多来自**纪律性**（降低波动性和避免行为错误），而非持续alpha。纪律性再平衡带来**行为性alpha**。

### 1.3 实例

| 阶段 | 股票 | 债券 | 总计 | 股票占比 |
|------|------|------|------|----------|
| 初始 | $6,000,000 | $4,000,000 | $10,000,000 | 60% |
| 股票涨25% | $7,500,000 | $4,000,000 | $11,500,000 | 65.2% |
| 偏离 | — | — | — | +5.2% |
| 再平衡 | $6,900,000 | $4,600,000 | $11,500,000 | 60% |

## 二、三种再平衡触发方式

### 2.1 日历型（Calendar Rebalancing）

在设定的时间间隔进行再平衡。

| 维度 | 描述 |
|------|------|
| 典型设定 | 按季度或年度 |
| 优点 | 简单、可预测、易于审计 |
| 缺点 | 漂移小时引发不必要交易 |
| 适用 | 小型团队、简单账户 |

### 2.2 阈值型（Threshold Rebalancing）

当资产权重偏离预设带宽时再平衡。

| 维度 | 描述 |
|------|------|
| 典型设定 | ±2%、±5%、±10% |
| 优点 | 对实际漂移灵敏，总体交易较低 |
| 缺点 | 需要持续监控，波动期集中交易 |
| 适用 | 税收优惠账户、顾问模型 |

### 2.3 混合型（Calendar + Threshold）

固定节奏检查，只有漂移超过阈值才再平衡。

| 维度 | 描述 |
|------|------|
| 典型设定 | 季度检查 + 5%区间 |
| 优点 | 可预测性与敏感性结合 |
| 缺点 | 需要系统支持 |
| 适用 | 机构模型组合、智能投顾 |

## 三、对比分析

| 方法 | 典型设定 | 主要好处 | 成本结构 | 使用场景 |
|------|----------|----------|----------|----------|
| 日历型 | 季度/年度 | 易于实施 | 频繁时交易成本高 | 简单账户 |
| 阈值型 | ±5% | 限制漂移 | 波动期集中交易 | 税优账户 |
| 混合型 | 季度+5% | 最佳平衡 | 可预测且高效 | 机构/智能投顾 |

> [!tip] 经验法则
> Vanguard实证：对于多元化股债组合，不同再平衡频率下风险调整后回报无显著差异——但再平衡事件数量（成本）差异很大。年度/半年度+5%区间是务实低成本折中。

## 四、再平衡成本

### 4.1 交易摩擦

- 显性佣金
- 买卖价差（点差）
- 市场冲击

### 4.2 税务摩擦

- 已实现收益的税负
- 洗售规则（前后30天限制）
- 税损收割机会

### 4.3 成本衡量框架

1. 估计预期换手率（回测或蒙特卡洛）
2. 应用交易成本假设（点差+佣金+冲击）
3. 应用税率假设
4. 比较：再平衡成本 vs 风险控制价值

## 五、大资金池特殊考虑

- 可预测再平衡时间表产生**订单流可预测性**
- 大型养老金/目标日期基金的再平衡成本可能是实质性的
- 使用**衍生品叠加**（期货、掉期）避免现货市场冲击
- 执行算法：VWAP/TWAP、Almgren-Chriss框架

## 六、运营设计要点

| 环节 | 要求 |
|------|------|
| 策略语言 | 明确再平衡规则和阈值 |
| 执行控制 | 订单路由、滑点监控 |
| 税务管理 | 税务批次元数据、跨账户监控 |
| 自动化 | 系统化触发和执行 |
| 审计追踪 | 记录每次再平衡详情 |
