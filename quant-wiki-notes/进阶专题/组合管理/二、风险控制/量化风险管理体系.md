# 量化风险管理体系

> [!note] 核心要点
> 量化风险管理是决定量化系统能否长期存活的"核心竞争力"。本文系统剖析风险的本质分类、量化风险指标的实战应用，以及如何构建科学的资金管理体系，打造坚不可摧的量化防御体系。

## 一、风险分类

### 1.1 市场风险

| 子类型 | 描述 |
|--------|------|
| 价格波动风险 | 资产价格不利变动导致亏损 |
| 系统性风险 | 宏观因素导致整体市场下跌，无法分散消除 |

### 1.2 流动性风险

| 子类型 | 描述 |
|--------|------|
| 交易流动性风险 | 无法在合理价格平仓 |
| 资金流动性风险 | 无法及时获取资金 |

### 1.3 模型风险

| 子类型 | 描述 |
|--------|------|
| 模型失效 | 市场结构变化导致前提不成立 |
| 过拟合 | 历史表现好但未来表现差 |
| 参数漂移 | 最优参数随时间变化 |
| 数据质量风险 | 错误数据导致决策失误 |

### 1.4 技术风险

- 系统故障、网络中断
- 数据传输延迟/错误
- 账户安全风险

### 1.5 操作风险

- 人为失误（配置错误、误操作）
- 流程缺陷（风控不完善、预案缺失）

### 1.6 黑天鹅事件

极低概率、极高影响事件，超出历史数据统计范围。详见 [[黑天鹅风险管理]]。

## 三、资金管理

### 3.1 仓位控制

根据历史最大回撤反推开仓名义资金：

$$\text{名义资金} = \frac{\text{历史最大回撤}}{\text{可承受回撤比例}}$$

> [!tip] 实战经验
> 预期最大回撤 ≈ 历史最大回撤 × 1.5。长期保证金占用率应控制在20%~40%。

### 3.2 止损机制

- **固定止损**：预设固定比例
- **跟踪止损**：跟随盈利动态调整
- **波动率止损**：基于ATR等波动指标

### 3.3 风险预算

将总风险预算分配到各策略/资产：
- 单策略风险占比上限
- 单资产风险占比上限
- 行业/风格风险占比上限

## 四、量化风控系统架构

```
风控系统
├── 交易前
│   ├── 策略回测验证
│   ├── 风险参数设定
│   └── 资金分配
├── 交易中
│   ├── 实时风险监控
│   ├── 异常预警
│   ├── 自动止损
│   └── 仓位限制
└── 交易后
    ├── 绩效归因
    ├── 风险复盘
    └── 参数调整
```

## 五、为什么量化交易尤其需要精细化风险管理

1. **杠杆放大风险**：杠杆交易收益和风险等比放大
2. **模型黑箱**：策略复杂，潜在风险不易察觉
3. **高频快速决策**：毫秒级决策，亏损瞬间累积
4. **系统性传染**：流动性枯竭时风险蔓延

## 课程化学习补充

> [!important] 学习定位
> 组合管理把单个机会变成可长期复利的资金曲线，核心是相关性、风险预算、再平衡和极端情景。本文仅用于学习、研究与复盘，不构成任何投资建议。

### 必须掌握的问题

- 收益来源是否足够分散
- 相关性在压力期如何变化
- 再平衡规则是否机械执行
- 最大回撤是否在承受范围内

### 实战应用流程

1. 先写清楚你的投资假设：为什么这个信号、资产或方法应该产生收益。
2. 明确数据口径：样本范围、更新时间、复权/分红/停牌处理和交易日历。
3. 做最小可行验证：先用简单规则验证方向，再逐步加入复杂模型。
4. 把成本和约束前置：手续费、滑点、冲击成本、保证金、流动性和容量都要进入测算。
5. 上线后持续复盘：记录信号、下单、成交、持仓、回撤和失效原因。

### 风险与失效条件

- 表面分散实则同一风险
- 低估相关性突变
- 杠杆叠加
- 回撤期停止执行纪律

### 复盘问题

- 这笔交易或这套模型赚的是什么钱：风险补偿、行为偏差、流动性溢价，还是偶然噪音？
- 如果市场环境反过来，最大亏损和最长恢复期会是多少？
- 当前结论是否依赖某个不可持续假设，例如低利率、低波动、充裕流动性或监管套利？
- 有没有一个更简单的基准策略能取得接近效果？
