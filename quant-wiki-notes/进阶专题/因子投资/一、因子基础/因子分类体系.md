# 🗂️ 因子分类体系

> [!note] 概述
> 本笔记基于 factors.directory 收录的 **474个量化因子**，将其按数据来源和构建逻辑归入 **十大类别**。这一分类兼顾学术传统与实战需求。

## 一、分类总览

### 1.1 分类维度

| 维度 | 说明 |
|-----|------|
| **数据来源** | 财务数据（基本面）/ 市场行情（量价）/ 另类数据（舆情、产业链） |
| **经济学逻辑** | 风险溢价 / 行为偏差 / 统计套利 |
| **持有周期** | 高频（日内-周）/ 中频（月-季）/ 低频（年+） |
| **因子层级** | 大类因子（Style Factor）/ 子因子（Signal） |

### 1.2 十大因子类别

| # | 类别 | 英文 | 因子数 | 数据来源 | 代表因子 |
|---|-----|------|-------|---------|---------|
| 1 | 价值因子 | Value | 6 | 财务数据 | B/P, E/P, D/P |
| 2 | 动量因子 | Momentum | 9 | 市场行情 | 累计收益、加速动量 |
| 3 | 质量因子 | Quality | 6 | 财务数据 | ROIC、应计比率 |
| 4 | 成长因子 | Growth | 14 | 财务数据 | PEG、营收增长 |
| 5 | 波动率因子 | Volatility | 14 | 市场行情 | 特质波动率、Beta |
| 6 | 流动性因子 | Liquidity | 7 | 市场行情 | Amihud、换手率 |
| 7 | 规模因子 | Size | 3 | 财务+行情 | 总市值、流通市值 |
| 8 | 技术因子 | Technical | 100 | 市场行情 | RSI、MACD、布林带 |
| 9 | 情绪因子 | Emotion | 25 | 另类数据 | 分析师覆盖、新闻情绪 |
| 10 | 基本面因子 | Fundamental | 180+ | 财务数据 | 100+财务指标衍生 |

## 二、各大类因子详解

### 2.1 价值因子 (Value Factors)

| 子因子名称 | 构建方法 | 逻辑 |
|-----------|---------|------|
| 账面市值比 (B/P) | 账面价值 / 市值 | 低估值 = 便宜 |
| 盈利收益率 (E/P) | 盈利 / 市值 | 高盈利收益率 = 便宜 |
| 股息率 (D/P) | 股息 / 价格 | 高股息 = 便宜 + 现金回报 |
| 现金流市值比 | 经营现金流 / 市值 | 现金流比利润更可靠 |
| 非线性市值 | 市值非线性变换 | 剔除极致值干扰 |

> 详见：[[价值因子]]

### 2.2 动量因子 (Momentum Factors)

| 子因子名称 | 构建方法 | 逻辑 |
|-----------|---------|------|
| 价格动量 (K月) | 过去K个月累计收益 | 趋势延续 |
| 时间序列动量 | 自身历史收益vs阈值 | 绝对趋势信号 |
| 加速度动量 | 动量的变化率 | 趋势是否在加速 |
| 52周高点 | 当前价/52周最高价 | 锚定效应 |
| 行业动量 | 行业指数动量 | 行业beta动量 |
| 短期反转 | 短期（周/日）反转 | 流动性冲击后的反弹 |

> 详见：[[动量因子]]

### 2.3 质量因子 (Quality Factors)

| 子因子名称 | 构建方法 | 逻辑 |
|-----------|---------|------|
| ROIC | 投入资本回报率 | 高盈利效率 |
| ROE | 净资产收益率 | 股东回报引擎 |
| 应计比率 | (净利-经营现金流)/总资产 | 应计越低质量越好 |
| 债务权益比 | 总负债/股东权益 | 低杠杆更安全 |
| 总应计盈余 | 资产负债表法应计 | 盈余质量 |

> 详见：[[质量因子]]

### 2.4 成长因子 (Growth Factors)

| 子因子名称 | 构建方法 | 逻辑 |
|-----------|---------|------|
| 盈利增长 | 净利润同比增速 | 高增长溢价 |
| 营收增长 | 营收同比增速 | 业务扩张 |
| PEG | PE / 盈利增速 | 成长性价比 |
| 研究支出占比 | 研发支出/营收 | 创新能力 |
| EPS增长 | 每股收益同比 | 股东价值增长 |

> 详见：[[成长因子]]

### 2.5 波动率因子 (Volatility Factors)

| 子因子名称 | 构建方法 | 逻辑 |
|-----------|---------|------|
| 特质波动率 | CAPM/FF3残差标准差 | 低异质波动异常 |
| 总波动率 | 收益率标准差 | 低波动率异象 |
| 市场Beta | 对市场收益的敏感度 | 低Beta异常 |
| 下行Beta | 市场下跌时的敏感度 | 下侧风险 |
| VaR / CVaR | 在险价值 | 极端风险 |

> 详见：[[波动率因子]]

### 2.6 流动性因子 (Liquidity Factors)

| 子因子名称 | 构建方法 | 逻辑 |
|-----------|---------|------|
| Amihud非流动性 | 日收益绝对值/成交额 | 价格冲击 |
| 换手率 | 成交量/流通股数 | 交易活跃度 |
| 成交量 | 日均/月均成交量 | 流动性代理 |
| 市值调整换手率 | 换手率 × 市值调整 | 规模中性流动性 |

> 详见：[[流动性因子]]

### 2.7 技术因子 (Technical Factors)

| 子类 | 数量 | 代表因子 |
|-----|------|---------|

| 动量振荡器 | ~20 | RSI、随机指标、MACD |
| 成交量指标 | ~15 | 量比、OBV、MFI |
| 波动率指标 | ~10 | ATR、历史波动率、波幅 |
| 资金流向 | ~10 | 主力资金、大单净买卖 |
| 乖离与反转 | ~10 | 偏差率、超买超卖 |
| 其他 | ~10 | 协偏度、峰度等分布特征 |

| 分析师盈利预测修正 | 预测调整 | 预期变化动量 |
| 新闻量/情绪 | 文本分析 | 信息传播效应 |
| 股东人数变化 | 股东数据 | 筹码集中度 |

### 2.9 基本面因子 (Fundamental Factors)

涵盖 **180+** 个从财务报表衍生的因子：

| 子类 | 说明 | 数量 |
|-----|------|------|
| 盈利能力 | ROE、ROA、毛利率、净利率等 | ~30 |
| 财务杠杆 | 资产负债率、权益乘数等 | ~20 |
| 营运效率 | 存货周转率、应收账款周转率等 | ~20 |
| 现金流 | 经营现金流/市值、自由现金流等 | ~15 |
| 盈余质量 | 应计、盈余持续性、盈余可预测性等 | ~15 |
| 公司治理 | 股权集中度、独立董事等 | ~10 |
| 估值衍生 | 企业价值倍数、去杠杆化指标等 | ~20 |
| 增长 | 各项财务指标增长率 | ~20 |
| 其他 | 无形资产、人均持股等 | ~10 |

> 详见：[[基本面因子]]

### 2.10 规模因子 (Size Factors)

| 子因子名称 | 构建方法 |
|-----------|---------|
| 总市值 | 股价 × 总股本 |
| 自由流通市值 | 股价 × 自由流通股 |
| 对数市值 | ln(总市值) |

## 三、因子分类的交叉性

很多因子**不属于单一类别**，而是横跨多个维度。例如：

- **残差动量** 结合了动量和技术因子
- **流动性调整波动率** 结合了流动性和波动率
- **PEG** 结合了价值和成长

> [!tip] 实战建议
> 分类是为了理解，不应成为束缚。实际构建多因子模型时，关键是考虑因子间的**相关性**与**信息增量**，而非其分类标签。

## 课程化学习补充

> [!important] 学习定位
> 因子投资把可解释的收益来源系统化，关键是因子定义、数据口径、检验方法、组合构建和衰减监控。本文仅用于学习、研究与复盘，不构成任何投资建议。

### 必须掌握的问题

- 因子方向是否有经济解释
- 是否做去极值/中性化/标准化
- IC 和分层收益是否稳定
- 换手和容量是否可交易

### 实战应用流程

1. 先写清楚你的投资假设：为什么这个信号、资产或方法应该产生收益。
2. 明确数据口径：样本范围、更新时间、复权/分红/停牌处理和交易日历。
3. 做最小可行验证：先用简单规则验证方向，再逐步加入复杂模型。
4. 把成本和约束前置：手续费、滑点、冲击成本、保证金、流动性和容量都要进入测算。
5. 上线后持续复盘：记录信号、下单、成交、持仓、回撤和失效原因。

### 风险与失效条件

- 因子动物园
- 样本内挖掘
- 拥挤交易
- 行业和市值暴露伪装成 alpha

### 复盘问题

- 这笔交易或这套模型赚的是什么钱：风险补偿、行为偏差、流动性溢价，还是偶然噪音？
- 如果市场环境反过来，最大亏损和最长恢复期会是多少？
- 当前结论是否依赖某个不可持续假设，例如低利率、低波动、充裕流动性或监管套利？
- 有没有一个更简单的基准策略能取得接近效果？
