# 过拟合识别与防御

> [!note] 核心概念
> 过拟合（Overfitting）是量化交易的头号杀手。你的模型不是在学真实规律，而是在记忆历史数据的随机噪声。

## 什么是过拟合？

**类比**：假设你要预测明天天气，翻看过去100天记录发现了"规律"——每次穿红色衣服的人超过30%，第二天就下雨。这个规律在过去100天成立（纯巧合），但预测未来显然荒谬。量化交易中的过拟合就是找到这种"看起来很强，实际上毫无因果关系"的规律。

## 为什么量化特别容易过拟合？

| 原因 | 说明 |
|------|------|
| 参数太多 | 100个参数，只有1000个样本 |
| 多重检验 | 测试1000个策略，总有几个"有效" |
| 信噪比低 | 市场噪音大，真实信号弱 |
| 分布不稳定 | 过去的规律未来可能失效 |

## 过拟合的识别

### 识别信号

1. **策略参数异常敏感**：均线20日效果好，换成19日或21日就大幅恶化
2. **收益曲线过于完美**：回测净值几乎是一条直线，没有明显回撤区
3. **策略逻辑缺乏经济学解释**："每周三下午2:30买入"——没有逻辑
4. **样本内/外表现差异巨大**：训练集年化50%，测试集年化5%

### 常见过拟合模式

| 模式 | 症状 | 风险等级 |
|------|------|---------|
| 完美曲线 | 无比平滑，无任何回撤 | 🔴 极高 |
| 极端参数敏感 | 参数微小变化导致结果剧变 | 🔴 高 |
| 信号数量正相关 | 信号越多收益越高（反常） | 🔴 高 |
| 训练/测试差距大 | 训练集 >> 测试集 | 🟡 中 |

## 防御方法

### 1. 减少参数（奥卡姆剃刀）
在同等解释力下，参数越少的模型越好。3个参数的策略比15个参数的更可靠。

### 2. 严格的样本外测试
```
完整数据划分：
├─ 训练集 (60%)：参数优化、策略开发
├─ 验证集 (20%)：参数最终选择
├─ 测试集 (20%)：最终绩效评估 ← 只用一次！
```

**关键原则**：测试集**仅使用一次**用于最终评估，不可回看也不可调参。

### 3. Walk-Forward 验证
```
第1轮：训练[1月-6月] → 测试[7月]
第2轮：训练[2月-7月] → 测试[8月]
第3轮：训练[3月-8月] → 测试[9月]
...
每轮都是严格的样本外测试
```

### 4. Monte Carlo 模拟
随机扰动测试策略鲁棒性：
- 随机打乱交易顺序
- 随机调整入场时间 ±1天
- 随机调整成本 ±20%
- → 如果90%模拟结果为正且分布集中，策略可靠

### 5. 正则化
- **Ridge (L2)**：惩罚大系数，使因子系数趋于平滑
- **Lasso (L1)**：稀疏化系数，自动特征选择

### 6. 跨市场/跨时期验证
- 美股发现的策略，在A股、港股、欧洲市场也有效吗？
- 2015-2019训练的策略，在2020-2023表现如何？

## 实战清单

- [ ] 参数数量控制在最小（≤5个核心参数）
- [ ] 使用了严格的样本外测试（训练/验证/测试三段划分）
- [ ] 策略有清晰的经济逻辑
- [ ] t统计量超过3.0（而非2.0）
- [ ] 参数±20%变化，收益变化<30%
- [ ] 回测涵盖至少1个牛熊周期
- [ ] 如果回测收益减半后仍然可接受，再考虑实盘

## 行业经验

> **回测到实盘性能衰减平均30-50%**——这不是悲观估计，而是无数实盘验证后的行业共识。

| 衰减幅度 | 占比 | 主要原因 |
|---------|------|---------|
| <20% | 10% | 低频策略、优秀成本建模 |
| 20-30% | 25% | 正常衰减 |
| 30-50% | 40% | 典型情况 |
| 50-70% | 15% | 成本低估、轻度过拟合 |
| >70% | 10% | 严重过拟合 |

## 课程化学习补充

> [!important] 学习定位
> 回测的首要任务是证伪策略，而不是证明策略赚钱；可靠回测必须同时处理数据、时间、成本、容量和风控。本文仅用于学习、研究与复盘，不构成任何投资建议。

### 必须掌握的问题

- 是否使用 point-in-time 数据
- 信号是否滞后一根 bar 执行
- 手续费/滑点/冲击成本是否计入
- 样本外和 walk-forward 是否通过

### 实战应用流程

1. 先写清楚你的投资假设：为什么这个信号、资产或方法应该产生收益。
2. 明确数据口径：样本范围、更新时间、复权/分红/停牌处理和交易日历。
3. 做最小可行验证：先用简单规则验证方向，再逐步加入复杂模型。
4. 把成本和约束前置：手续费、滑点、冲击成本、保证金、流动性和容量都要进入测算。
5. 上线后持续复盘：记录信号、下单、成交、持仓、回撤和失效原因。

### 风险与失效条件

- 前视偏差
- 幸存者偏差
- 参数过拟合
- 实盘容量与流动性不足

### 复盘问题

- 这笔交易或这套模型赚的是什么钱：风险补偿、行为偏差、流动性溢价，还是偶然噪音？
- 如果市场环境反过来，最大亏损和最长恢复期会是多少？
- 当前结论是否依赖某个不可持续假设，例如低利率、低波动、充裕流动性或监管套利？
- 有没有一个更简单的基准策略能取得接近效果？
