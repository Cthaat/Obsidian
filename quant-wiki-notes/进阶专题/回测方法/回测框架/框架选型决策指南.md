# 框架选型决策指南

> [!note] 决策框架
> 选择回测框架没有唯一正确答案，取决于你的策略类型、编程水平、实盘需求和团队规模。

## 决策树

```
你的策略类型？
├── 简单技术指标策略 ─── PyBroker / Backtrader
├── 高频/日内策略 ─── VectorBT / WonderTrader
├── AI/ML策略 ─── qlib / Backtrader + PyTorch
├── 多因子选股 ─── qlib / Alphalens + Backtrader
└── 多资产组合 ─── Backtrader / vnpy

你的编程水平？
├── 新手 ─── PyBroker / Zipline
├── 有Python基础 ─── Backtrader / VectorBT
└── 资深开发者 ─── any framework + 自研扩展

需要实盘吗？
├── 纯研究/回测 ─── Backtrader / Zipline / qlib
├── A股实盘 ─── vnpy / WonderTrader
└── 美股/加密货币 ─── Backtrader + IB/Oanda API
```

## 向量化回测 vs 事件驱动回测

| 特性 | 向量化回测 | 事件驱动回测 |
|------|----------|------------|
| 速度 | 极快（批量运算） | 较慢（逐条处理） |
| 复杂度 | 低 | 高 |
| 真实度 | 简化（假设成交） | 高（模拟撮合） |
| 适用 | 快速验证想法 | 生产级回测 |
| 代表 | VectorBT | Backtrader、Zipline |

## 生产级回测的考量

1. **数据质量**：数据源是否包含退市股票？
2. **撮合模型**：是否模拟了真实的市场微观结构？
3. **成本模型**：滑点、冲击成本是否合理？
4. **性能**：能否处理多年、多品种的分钟级数据？
5. **可扩展性**：能否对接实盘？能否集成自定义组件？


## 实战掌握清单

> [!tip] 交易者视角
> 框架选型决策指南 的学习重点不是记住术语，而是把它放进研究、组合、执行和复盘的闭环。回测是研究假设的压力测试，不是寻找漂亮收益曲线的工具。

### 关键判断

- 确认数据时间戳、复权、停牌、退市、公告日和交易日历。
- 分清样本内开发、样本外验证和实盘跟踪。
- 把手续费、滑点、冲击成本和容量限制前置。

### 落地动作

1. 先建立最简单基准，再逐步加入过滤条件。
2. 检查参数稳定性、收益来源和不同市场阶段表现。
3. 输出回撤、换手、暴露、交易笔数和失败案例。

### 失效边界

- 前视偏差和幸存者偏差。
- 过度调参。
- 忽略资金规模放大后的成交约束。

### 复盘问题

- 这项知识改变了哪一个具体决策：标的、方向、仓位、退出、对冲还是不交易？
- 如果判断相反，最大亏损、最长恢复期和退出触发条件是什么？
- 有没有一个更简单的基准方法可以取得相近结果？

## 深度案例与训练

### 回测审计

围绕 框架选型决策指南 检查数据时点、交易日历、复权、停牌、退市、手续费、滑点、冲击成本、容量和参数选择。回测报告必须能解释失败样本。

### 验证动作

- 样本内、样本外和滚动窗口分开。
- 对参数做扰动，检查收益是否脆弱。
- 与简单基准比较，避免复杂策略没有真实增益。

### 结论标准

回测通过只能说明值得小规模实盘验证，不代表策略已经可靠。
