# 框架选型决策指南

> [!note] 决策框架
> 选择回测框架没有唯一正确答案，取决于你的策略类型、编程水平、实盘需求和团队规模。

## 决策树

```
你的策略类型？
├── 简单技术指标策略 ─── PyBroker / Backtrader
├── 高频/日内策略 ─── VectorBT / WonderTrader
├── AI/ML策略 ─── qlib / Backtrader + PyTorch
├── 多因子选股 ─── qlib / Alphalens + Backtrader
└── 多资产组合 ─── Backtrader / vnpy

你的编程水平？
├── 新手 ─── PyBroker / Zipline
├── 有Python基础 ─── Backtrader / VectorBT
└── 资深开发者 ─── any framework + 自研扩展

需要实盘吗？
├── 纯研究/回测 ─── Backtrader / Zipline / qlib
├── A股实盘 ─── vnpy / WonderTrader
└── 美股/加密货币 ─── Backtrader + IB/Oanda API
```

## 向量化回测 vs 事件驱动回测

| 特性 | 向量化回测 | 事件驱动回测 |
|------|----------|------------|
| 速度 | 极快（批量运算） | 较慢（逐条处理） |
| 复杂度 | 低 | 高 |
| 真实度 | 简化（假设成交） | 高（模拟撮合） |
| 适用 | 快速验证想法 | 生产级回测 |
| 代表 | VectorBT | Backtrader、Zipline |

## 生产级回测的考量

1. **数据质量**：数据源是否包含退市股票？
2. **撮合模型**：是否模拟了真实的市场微观结构？
3. **成本模型**：滑点、冲击成本是否合理？
4. **性能**：能否处理多年、多品种的分钟级数据？
5. **可扩展性**：能否对接实盘？能否集成自定义组件？

## 推荐技术栈组合

### 个人量化入门
```
Data: akshare / yfinance
Backtest: Backtrader
Analysis: pandas + matplotlib
```

### 专业研究
```
Data: JoinQuant(JQData) / Wind
Backtest: Backtrader / qlib
Analysis: Alphalens + Pyfolio + Empyrical
ML: scikit-learn + PyTorch
```

### 实盘交易
```
Data: 券商API + 数据商
Execution: vnpy / WonderTrader
Risk: 自研风控模块
Monitoring: Grafana + Prometheus
```

## 聚宽量化平台

中国市场的一个便捷选择是[JoinQuant（聚宽）](https://www.joinquant.com/)：
- 在线回测，无需本地部署
- 内置A股全量数据
- 支持Python策略编写
- 提供JQData本地数据服务
- 可直接对接实盘

## 总结

| 场景 | 推荐方案 |
|------|---------|
| 快速学习 | Backtrader (文档最全) |
| 快速验证想法 | VectorBT (速度最快) |
| AI量化研究 | qlib (ML native) |
| A股实盘 | vnpy (生态最完整) |
| 机构级回测 | 自研 + Backtrader参考 |
