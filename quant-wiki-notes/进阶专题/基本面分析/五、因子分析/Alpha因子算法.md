# Alpha因子算法

> [!note] 核心要点
> Alpha因子挖掘是从海量数据中发现具有预测能力的选股因子的过程。主要方法包括人工挖掘、算法自动挖掘（遗传规划等）和LLM辅助挖掘。

## 一、因子挖掘方法分类

| 方法 | 优势 | 劣势 |
|-----|------|------|
| **人工挖掘** | 逻辑清晰、可解释性强 | 效率低、成本高 |
| **遗传规划（GP）** | 自动化搜索、覆盖面广 | 过拟合、可解释性差 |
| **LLM辅助挖掘** | 兼顾效率与可解释性 | 模型依赖、需验证 |

## 二、遗传规划（Genetic Programming）

### 基本原理
遗传规划模拟生物进化过程，通过**选择、交叉、变异**操作迭代优化因子表达式树。

### 算子体系

| 算子类型 | 示例 |
|---------|------|
| 时序算子 | TS_Mean, TS_Std, TS_Max, TS_Min, TS_Rank |
| 截面算子 | CS_Rank, CS_Zscore, CS_Winsorize |
| 元素算子 | Add, Sub, Mul, Div, Abs, Log, Sign |
| 聚合算子 | Mean, Std, Corr, Cov |

### 适应度评估
- **IC（信息系数）**：因子值与未来收益的相关性
- **IC_IR**：IC的均值/标准差，衡量稳定性
- **换手率**：因子信号的稳定性
- **多空收益**：因子分组的收益区分度

## 三、因子生成流程

```
底层数据 → 算子组合 → 因子表达式 → 回测评估 → 筛选入库
```

### 1. 数据层
- 量价数据：开高低收、成交量、成交额
- 基本面数据：财务报表三表字段
- 高频数据：分钟级量价、逐笔委托

### 2. 表达式生成
通过算子组合生成因子表达式，例如：
```
Rank_Mul(Decay_Linear(TS_Return(CLOSE, 10), 10),
         Rank(Div(VOLUME, TS_Mean(VOLUME, 10))))
```

### 3. 回测评估
- 中低频因子：5天滚动，周度调仓
- 高频因子：日频调仓，考虑成本可降至周频

## 四、因子优化技术

| 技术 | 说明 |
|-----|------|
| **因子正交化** | 去除因子间的共线性 |
| **因子中性化** | 剔除行业和市值的影响 |
| **因子衰减处理** | 对因子值进行时间衰减加权 |
| **异常值处理** | Winsorize/截面标准化 |

## 五、因子库管理

> [!tip] 因子入库标准
> - IC均值 > 0.03（或绝对值 > 0.03）
> - IC_IR > 0.5
> - 多空年化收益 > 5%
> - 因子间相关性 < 0.7（保持多样性）

## 六、算法改进方向

1. **深度学习替代GP**：用神经网络自动学习因子表达式
2. **LLM辅助生成**：利用大语言模型的逻辑推理能力
3. **多目标优化**：同时优化IC、换手率、最大回撤等
4. **动态因子库**：根据市场状态自动调整因子权重

## 课程化学习补充

> [!important] 学习定位
> 基本面研究把企业经营翻译成可验证的现金流、盈利质量和资本回报，最终落到估值和仓位。本文仅用于学习、研究与复盘，不构成任何投资建议。

### 必须掌握的问题

- 收入和利润是否可持续
- 现金流是否支持利润
- ROE 来自效率还是杠杆
- 估值是否包含足够安全边际

### 实战应用流程

1. 先写清楚你的投资假设：为什么这个信号、资产或方法应该产生收益。
2. 明确数据口径：样本范围、更新时间、复权/分红/停牌处理和交易日历。
3. 做最小可行验证：先用简单规则验证方向，再逐步加入复杂模型。
4. 把成本和约束前置：手续费、滑点、冲击成本、保证金、流动性和容量都要进入测算。
5. 上线后持续复盘：记录信号、下单、成交、持仓、回撤和失效原因。

### 风险与失效条件

- 财报粉饰
- 周期顶部外推
- 只看利润不看现金流
- 忽视治理和资本开支压力

### 复盘问题

- 这笔交易或这套模型赚的是什么钱：风险补偿、行为偏差、流动性溢价，还是偶然噪音？
- 如果市场环境反过来，最大亏损和最长恢复期会是多少？
- 当前结论是否依赖某个不可持续假设，例如低利率、低波动、充裕流动性或监管套利？
- 有没有一个更简单的基准策略能取得接近效果？
