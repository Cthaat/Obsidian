# 量化交易员24小时工作流

> [!note] 概述
> 一个量化交易员的典型日常操作流程，从盘前准备到盘后分析的全流程拆解。

## 典型一日流程

### 盘前准备（08:00-09:00）

```
┌─────────────────────────────────────┐
│ 1. 系统健康检查                     │
│    · 数据库连接状态                  │
│    · API接口连通性                   │
│    · 消息队列积压情况                │
├─────────────────────────────────────┤
│ 2. 数据校验                         │
│    · 昨日数据完整性检查              │
│    · 复权除息数据处理                │
│    · 数据缺失告警处理                │
├─────────────────────────────────────┤
│ 3. 策略初始化                       │
│    · 加载最新模型参数                │
│    · 计算当日信号阈值                │
│    · 风控参数确认                    │
└─────────────────────────────────────┘
```

### 盘中监控（09:30-15:00）

```
┌─────────────────────────────────────┐
│ 实时监控面板                         │
│ · PnL实时变动                        │
│ · 持仓风险指标                       │
│ · 信号生成频率                       │
│ · 订单成交率                         │
│ · 系统延迟监控                       │
├─────────────────────────────────────┤
│ 异常处理                             │
│ → 订单异常: 撤单 + 人工复核          │
│ → 数据断连: 切换备选数据源           │
│ → 风控触发: 暂停策略 + 平仓          │
│ → 系统故障: 启动灾备流程             │
└─────────────────────────────────────┘
```

### 盘后分析（15:00-17:00）

```
┌─────────────────────────────────────┐
│ 1. 绩效归因                         │
│    · 当日盈亏分解（Alpha/Beta/残差）  │
│    · 各策略贡献分析                  │
│    · 交易成本统计                    │
├─────────────────────────────────────┤
│ 2. 风控回顾                         │
│    · 最大回撤 vs 风控阈值            │
│    · 换手率是否符合预期              │
│    · 是否有异常信号                  │
├─────────────────────────────────────┤
│ 3. 策略维护                         │
│    · 如果达到模型更新周期 → 再训练   │
│    · 因子表现跟踪                    │
│    · 策略拥挤度监控                  │
└─────────────────────────────────────┘
```

### 研究与开发（17:00-20:00）

- 新策略研究（回测 + 分析）
- 代码重构与优化
- 论文复现
- 数据源拓展
- 系统性能优化

## 周度/月度任务

### 每周
- 策略表现周报
- 因子IC周跟踪
- 模型再训练判断

### 每月
- 全面绩效报告
- 策略调整决策
- 风控规则审查

## 实战掌握清单

> [!tip] 交易者视角
> 量化交易员24小时工作流 的学习重点不是记住术语，而是把它放进研究、组合、执行和复盘的闭环。量化部署关注系统能否稳定、可追踪、可停止地运行，核心是数据、信号、订单、风控和监控链路。

### 关键判断

- 明确研究、回测、模拟、实盘之间的数据和代码一致性。
- 检查订单状态、成交回报、撤单、风控拒单和异常告警。
- 评估延迟、可用性、容灾和人工接管流程。

### 落地动作

1. 上线前做权限隔离、资金上限、品种白名单和熔断规则。
2. 保留行情、信号、订单、成交和持仓日志。
3. 定期演练断网、接口失败、数据异常和策略失控。

### 失效边界

- 没有 kill switch。
- 日志不完整。
- 研究环境与实盘环境口径不一致。

### 复盘问题

- 这项知识改变了哪一个具体决策：标的、方向、仓位、退出、对冲还是不交易？
- 如果判断相反，最大亏损、最长恢复期和退出触发条件是什么？
- 有没有一个更简单的基准方法可以取得相近结果？
