# 事件驱动架构详解

> [!note] 概述
> 事件驱动架构（Event-Driven Architecture）是量化交易系统的核心设计模式，使回测代码能无缝迁移到实盘环境。

## 什么是事件驱动？

在事件驱动系统中，所有操作都由**事件**触发，按时间顺序逐一处理：

```
时间线：09:30:00 → 09:30:01 → 09:30:02 → ...

事件流：
  MarketEvent (新K线/新Tick)
      ↓
  SignalEvent (策略产生信号)
      ↓
  OrderEvent (订单发送)
      ↓
  FillEvent (成交回报)
```

## 核心组件

### 1. 事件队列（Event Queue）

```python
from collections import deque

class EventQueue:
    """FIFO事件队列"""
    def __init__(self):
        self.queue = deque()

    def push(self, event):
        self.queue.append(event)

    def pop(self):
        return self.queue.popleft() if self.queue else None
```

```python
class Strategy:
```

### 5. 执行处理器（ExecutionHandler）

```python
    """将OrderEvent发送到券商/交易所"""

        # 实盘模式：真实下单
        pass
```

## 回测 vs 实盘统一架构

```
        回测                        实盘
        ────                        ────
DataHandler(读CSV/DB)  ←→  DataHandler(读WebSocket/API)
Portfolio(模拟撮合)     ←→  Portfolio(真实成交)
Execution(模拟)         ←→  Execution(券商API)
```

**同源代码收益**：回测和实盘使用相同的Strategy、Portfolio、RiskManager代码，只替换DataHandler和ExecutionHandler的实现。

## 事件类型设计

```python
class EventType(Enum):
    MARKET = 'MARKET'         # 行情更新
    SIGNAL = 'SIGNAL'         # 交易信号
    ORDER = 'ORDER'           # 订单
    FILL = 'FILL'             # 成交
    RISK = 'RISK'             # 风控事件
    SYSTEM = 'SYSTEM'         # 系统事件（启动/关闭）
```

## Homalos 事件驱动系统

[Homalos](https://homalos.github.io/) 是一个开源的事件驱动量化系统：

- 数据中心：统一管理多数据源
- 事件总线：组件间异步解耦通信
- 策略容器：热加载/热更新策略
- 实时风控：规则引擎 + 动态阈值

## 设计要点

1. **单一职责**：每个组件只做一件事
2. **松耦合**：组件通过事件通信，不直接依赖
3. **可替换性**：DataHandler/ExecutionHandler可插拔
4. **时间一致性**：回测和实盘遵循相同的时间语义
5. **状态管理**：系统状态可序列化、可恢复

## 课程化学习补充

> [!important] 学习定位
> 部署阶段把研究结果变成生产系统，核心是数据可靠、订单可追踪、风控可熔断、异常可恢复。本文仅用于学习、研究与复盘，不构成任何投资建议。

### 必须掌握的问题

- 研究环境与实盘数据口径是否一致
- 订单状态机是否完整
- 风控是否前置
- 日志、告警和回滚是否可用

### 实战应用流程

1. 先写清楚你的投资假设：为什么这个信号、资产或方法应该产生收益。
2. 明确数据口径：样本范围、更新时间、复权/分红/停牌处理和交易日历。
3. 做最小可行验证：先用简单规则验证方向，再逐步加入复杂模型。
4. 把成本和约束前置：手续费、滑点、冲击成本、保证金、流动性和容量都要进入测算。
5. 上线后持续复盘：记录信号、下单、成交、持仓、回撤和失效原因。

### 风险与失效条件

- 接口限频和断连
- 重复下单
- 数据延迟
- 风控只在事后统计

### 复盘问题

- 这笔交易或这套模型赚的是什么钱：风险补偿、行为偏差、流动性溢价，还是偶然噪音？
- 如果市场环境反过来，最大亏损和最长恢复期会是多少？
- 当前结论是否依赖某个不可持续假设，例如低利率、低波动、充裕流动性或监管套利？
- 有没有一个更简单的基准策略能取得接近效果？

## 跨领域进阶扩展

> [!tip] 交易者视角

- 能否说清楚这个主题影响的是收益来源、风险来源、交易成本、流动性还是心理纪律？
