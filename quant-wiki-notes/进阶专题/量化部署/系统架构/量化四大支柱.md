# 量化四大支柱

> [!note] 概述
> 一个完整的量化交易系统由四大支柱构成：数据、策略、风控、执行。四者缺一不可。

## 支柱一：数据

### 数据源选择

| 来源 | 资产覆盖 | 成本 | 适合阶段 |
|------|---------|------|---------|
| **akshare/Tushare** | A股、期货、基金 | 免费 | 入门研究 |
| **yfinance** | 美股、ETF、全球 | 免费 | 入门研究 |
| **JQData**（聚宽） | A股全量 | 付费 | 专业研究 |
| **Wind** | 全球多资产 | 高 | 机构级 |
| **Bloomberg** | 全球全资产 | 极高 | 顶级机构 |

### 数据质量检查

```python
def validate_price_data(df):
    """基础价格数据验证"""
    if (df['close'] <= 0).any():
        raise ValueError("发现非正收盘价")
    if (df['volume'] < 0).any():
        raise ValueError("成交量数据不合法")
    # 检查缺失值、异常跳空
    return True
```

### 数据频率与延迟

| 频率 | 延迟 | 适合策略 |
|------|------|---------|
| 日线 | T+1 | 低频选股 |
| 分钟线 | 实时-分钟 | 日内交易 |
| Tick | 实时-秒 | 高频交易 |
| L2盘口 | <1ms | 做市/HFT |

## 支柱二：策略

### 策略开发流程

```
想法 → 数据验证 → 回测 → 参数优化 → 样本外测试 → 模拟盘 → 小资金实盘 → 全量实盘
```

- 回测不是终点，只是**降低风险的初始筛选**
- 策略必须有清晰的经济学逻辑

### 策略评估指标

| 指标 | 含义 | 参考阈值 |
|------|------|---------|
| 年化收益率 | 平均年收益 | >无风险利率+5% |
| 夏普比率 | 风险调整收益 | >1.0为良好 |
| 最大回撤 | 最大累计亏损 | <20% |
| 胜率 | 盈利交易占比 | >45% |
| 盈亏比 | 平均盈利/平均亏损 | >1.5 |
| Calmar比率 | 年化/最大回撤 | >1.0 |

## 支柱三：风控

### 多层级风控体系

```
Level 1: 策略级风控
├── 单笔仓位上限（如<10%）
├── 止损止盈设置
└── 最大持仓数量

Level 2: 组合级风控
├── 行业/板块集中度
├── Beta/因子暴露控制
└── VaR/CVaR限制

Level 3: 系统级风控
├── 日亏损限额（熔断）
├── 流动性监控
└── 异常交易检测
```

### 尾部风险

- 黑天鹅事件：模型无法预测的极端事件
- 2020年COVID：标普500 23天暴跌34%
- 防御：压力测试 + 极端场景模拟 + 期权保护

## 支柱四：执行

### 执行关键因素

| 因素 | 回测 | 实盘 |
|------|------|------|
| 成交价 | 收盘价/假设价 | 实际成交价（通常更差） |
| 滑点 | 固定假设 | 随市场变化 |
| 市场冲击 | 忽略 | 大单显著抬高成本 |
| 成交量 | 假设100%成交 | 可能部分成交 |
| 延迟 | 忽略 | 50-500ms |
| 故障 | 无 | 网络断、API错误 |

### 执行系统要求

- 低延迟：<10ms（非HFT）/ <1μs（HFT）
- 高可用：99.99%+ uptime
- 容错：异常自动恢复
- 监控：实时告警、日志审计

## 四大支柱的关系

```
数据 → 策略 ← → 风控
  ↓               ↓
策略 → 执行 ← → 风控

策略在回测中验证 → 风控在实盘中把关 → 执行将信号变为真实订单
三大环节（策略→风控→执行）共享同一数据底座
```

## 课程化学习补充

> [!important] 学习定位
> 部署阶段把研究结果变成生产系统，核心是数据可靠、订单可追踪、风控可熔断、异常可恢复。本文仅用于学习、研究与复盘，不构成任何投资建议。

### 必须掌握的问题

- 研究环境与实盘数据口径是否一致
- 订单状态机是否完整
- 风控是否前置
- 日志、告警和回滚是否可用

### 实战应用流程

1. 先写清楚你的投资假设：为什么这个信号、资产或方法应该产生收益。
2. 明确数据口径：样本范围、更新时间、复权/分红/停牌处理和交易日历。
3. 做最小可行验证：先用简单规则验证方向，再逐步加入复杂模型。
4. 把成本和约束前置：手续费、滑点、冲击成本、保证金、流动性和容量都要进入测算。
5. 上线后持续复盘：记录信号、下单、成交、持仓、回撤和失效原因。

### 风险与失效条件

- 接口限频和断连
- 重复下单
- 数据延迟
- 风控只在事后统计

### 复盘问题

- 这笔交易或这套模型赚的是什么钱：风险补偿、行为偏差、流动性溢价，还是偶然噪音？
- 如果市场环境反过来，最大亏损和最长恢复期会是多少？
- 当前结论是否依赖某个不可持续假设，例如低利率、低波动、充裕流动性或监管套利？
- 有没有一个更简单的基准策略能取得接近效果？

## 跨领域进阶扩展

> [!tip] 交易者视角
> 学到 `量化四大支柱` 时，不要只把它当成孤立知识点。把研究成果放入可监控、可追溯、可熔断的生产系统。优秀投资交易者会把它放入“宏观背景 - 资产选择 - 估值/信号 - 组合风险 - 交易执行 - 复盘反馈”的闭环。

### 与其他知识的连接

- 数据层、信号层、风控层、执行层和监控层
- 订单状态机和异常恢复
- 权限、日志和审计
- 仿真、小资金和灰度上线

### 进阶训练

1. 画出策略从数据到成交的链路图
2. 设计下单前风控和异常熔断
3. 为每笔信号保存可复盘记录

### 能力验收

- 能否说清楚这个主题影响的是收益来源、风险来源、交易成本、流动性还是心理纪律？
- 能否指出它在什么市场环境、资产类别或交易周期中更有效？
- 能否把它写成一条可复盘的研究或交易规则？
- 能否说明如果判断错误，组合最大损失和退出机制是什么？
