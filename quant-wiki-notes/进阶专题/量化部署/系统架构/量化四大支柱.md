# 量化四大支柱

> [!note] 概述
> 一个完整的量化交易系统由四大支柱构成：数据、策略、风控、执行。四者缺一不可。

## 支柱一：数据

### 数据源选择

| 来源 | 资产覆盖 | 成本 | 适合阶段 |
|------|---------|------|---------|
| **akshare/Tushare** | A股、期货、基金 | 免费 | 入门研究 |
| **yfinance** | 美股、ETF、全球 | 免费 | 入门研究 |
| **JQData**（聚宽） | A股全量 | 付费 | 专业研究 |
| **Wind** | 全球多资产 | 高 | 机构级 |
| **Bloomberg** | 全球全资产 | 极高 | 顶级机构 |

### 数据质量检查

```python
def validate_price_data(df):
    """基础价格数据验证"""
    if (df['close'] <= 0).any():
        raise ValueError("发现非正收盘价")
    if (df['volume'] < 0).any():
        raise ValueError("成交量数据不合法")
    # 检查缺失值、异常跳空
    return True
```

### 数据频率与延迟

| 频率 | 延迟 | 适合策略 |
|------|------|---------|
| 日线 | T+1 | 低频选股 |
| 分钟线 | 实时-分钟 | 日内交易 |
| Tick | 实时-秒 | 高频交易 |
| L2盘口 | <1ms | 做市/HFT |

## 支柱二：策略

### 策略开发流程

```
想法 → 数据验证 → 回测 → 参数优化 → 样本外测试 → 模拟盘 → 小资金实盘 → 全量实盘
```

- 回测不是终点，只是**降低风险的初始筛选**
- 策略必须有清晰的经济学逻辑

### 策略评估指标

| 指标 | 含义 | 参考阈值 |
|------|------|---------|
| 年化收益率 | 平均年收益 | >无风险利率+5% |
| 夏普比率 | 风险调整收益 | >1.0为良好 |
| 最大回撤 | 最大累计亏损 | <20% |
| 胜率 | 盈利交易占比 | >45% |
| 盈亏比 | 平均盈利/平均亏损 | >1.5 |
| Calmar比率 | 年化/最大回撤 | >1.0 |

## 支柱三：风控

### 多层级风控体系

```
Level 1: 策略级风控
├── 单笔仓位上限（如<10%）
├── 止损止盈设置
└── 最大持仓数量

Level 2: 组合级风控
├── 行业/板块集中度
├── Beta/因子暴露控制
└── VaR/CVaR限制

Level 3: 系统级风控
├── 日亏损限额（熔断）
├── 流动性监控
└── 异常交易检测
```

### 尾部风险

- 黑天鹅事件：模型无法预测的极端事件
- 2020年COVID：标普500 23天暴跌34%
- 防御：压力测试 + 极端场景模拟 + 期权保护

## 支柱四：执行

### 执行关键因素

| 因素 | 回测 | 实盘 |
|------|------|------|
| 成交价 | 收盘价/假设价 | 实际成交价（通常更差） |
| 滑点 | 固定假设 | 随市场变化 |
| 市场冲击 | 忽略 | 大单显著抬高成本 |
| 成交量 | 假设100%成交 | 可能部分成交 |
| 延迟 | 忽略 | 50-500ms |
| 故障 | 无 | 网络断、API错误 |

### 执行系统要求

- 低延迟：<10ms（非HFT）/ <1μs（HFT）
- 高可用：99.99%+ uptime
- 容错：异常自动恢复
- 监控：实时告警、日志审计

## 四大支柱的关系

```
数据 → 策略 ← → 风控
  ↓               ↓
策略 → 执行 ← → 风控

策略在回测中验证 → 风控在实盘中把关 → 执行将信号变为真实订单
三大环节（策略→风控→执行）共享同一数据底座
```

## 实战掌握清单

> [!tip] 交易者视角
> 量化四大支柱 的学习重点不是记住术语，而是把它放进研究、组合、执行和复盘的闭环。量化部署关注系统能否稳定、可追踪、可停止地运行，核心是数据、信号、订单、风控和监控链路。

### 关键判断

- 明确研究、回测、模拟、实盘之间的数据和代码一致性。
- 检查订单状态、成交回报、撤单、风控拒单和异常告警。
- 评估延迟、可用性、容灾和人工接管流程。

### 落地动作

1. 上线前做权限隔离、资金上限、品种白名单和熔断规则。
2. 保留行情、信号、订单、成交和持仓日志。
3. 定期演练断网、接口失败、数据异常和策略失控。

### 失效边界

- 没有 kill switch。
- 日志不完整。
- 研究环境与实盘环境口径不一致。

### 复盘问题

- 这项知识改变了哪一个具体决策：标的、方向、仓位、退出、对冲还是不交易？
- 如果判断相反，最大亏损、最长恢复期和退出触发条件是什么？
- 有没有一个更简单的基准方法可以取得相近结果？
