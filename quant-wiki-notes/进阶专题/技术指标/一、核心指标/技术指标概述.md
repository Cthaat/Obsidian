# 技术指标概述

> [!note] 💡 概念解析
> 技术指标概述全面梳理了量化交易中常用的技术指标分类及其策略分析框架，包括趋势类、动量类、波动率类、成交量类和综合类指标，并介绍了多指标确认、趋势-震荡切换等经典策略框架。

## 一、价格趋势类指标

| 指标 | 核心功能 | 关键信号 |
|------|---------|---------|
| MA（移动平均线） | 平滑价格波动，识别趋势方向 | 金叉买入、死叉卖出 |
| SAR（抛物线转向） | 趋势跟踪和止损设置 | SAR在价格下方看多，上方看空 |
| MACD | 结合趋势和动量 | MACD线与信号线交叉，柱状图变化 |

## 二、动量/震荡类指标

| 指标 | 核心功能 | 关键信号 |
|------|---------|---------|
| RSI | 衡量超买超卖状态 | >70超买，<30超卖，背离预示反转 |
| KDJ | K、D、J三线交互分析 | K与D线交叉，J线极值区间 |
| Williams %R | 价格在高低区间的相对位置 | 超买超卖判断 |
| Momentum | 价格变化速率 | 动量增减判断 |

## 三、波动率类指标

| 指标 | 核心功能 | 关键信号 |
|------|---------|---------|
| Volatility | 衡量市场波动程度 | 低波动后突破更可靠 |
| 布林带 | 标准差通道 | "W底"和"M顶"形态，带宽变化预测波动率突变 |
| ATR | 价格波动幅度 | 设置止损和调整仓位的理想工具 |

## 四、成交量类指标

| 指标 | 核心功能 | 关键信号 |
|------|---------|---------|
| OBV | 累积成交量确认趋势 | 量价配合验证 |
| CMF | 资金流入流出平衡 | 资金流向判断 |

## 五、综合类指标

| 指标 | 核心功能 | 关键信号 |
|------|---------|---------|
| ADX | 衡量趋势强度（非方向） | >25表示强趋势，配合+DI/-DI判断方向 |
| 江恩理论 | 价格与时间关系 | 支撑阻力位和市场周期分析 |

## 六、五大经典策略框架

### 6.1 多指标确认系统

> [!important] 🔑 核心思想
> "一个指标可能说谎，但多个指标很少同时撒谎。"

使用不同类别的指标相互验证，提高信号可靠性：

```python
# 示例：多指标确认买入信号
if ma_trend == "up" and rsi_value > 30 and macd_histogram > 0:
    return "strong_buy"
```

### 6.2 趋势-震荡组合策略

根据市场环境动态切换指标组合：

| 市场状态 | 适用指标 | 判断依据 |
|----------|---------|---------|
| 强趋势市场 | MA、MACD | ADX > 25 |
| 震荡市场 | RSI、KDJ | ADX ≤ 25 |

```python
if adx_value > 25:  # 强趋势市场
    return analyze_with_trend_indicators()
else:  # 震荡市场
    return analyze_with_oscillator_indicators()
```

### 6.3 波动率调整策略

低波动率后的突破信号通常更可靠，高波动率环境需要更保守的仓位管理：

```python
position_size = base_position * (1 - volatility_percentile/100)
```

### 6.4 周期分解策略

"自上而下"的多周期分析方法：

```python
# 先确认月线趋势，再分析周线机会，最后寻找日线入场点
if monthly_trend == weekly_trend == daily_trend == "up":
    return "strong_buy_signal"
```

### 6.5 背离识别策略

价格与指标之间的背离是可能反转的重要信号：

- **顶背离**：价格创新高，RSI/MACD未创新高 → 看跌
- **底背离**：价格创新低，RSI/MACD未创新低 → 看涨

## 七、综合策略分析系统

| 维度 | 权重 | 指标 |
|------|------|------|
| 趋势确认 | 40% | MA、MACD |
| 动量评估 | 30% | RSI、KDJ |
| 成交量确认 | 20% | OBV、CMF |
| 背离检测 | 10% | RSI、MACD背离 |

```python
bullish_score = (trend_score * 0.4 +
                momentum_score * 0.3 +
                volume_score * 0.2 +
                divergence_score * 0.1)

if bullish_score > 70:
    return "strong_buy"
elif bullish_score > 50:
    return "moderate_buy"
```

## 八、策略实施关键考量

1. **避免过度拟合**：不要使用过多指标或参数
2. **适应市场环境**：不同阶段需要不同的策略组合
3. **风险管理优先**：再好的策略也需要严格的风险控制
4. **持续验证**：定期回测和评估策略表现

## 课程化学习补充

> [!important] 学习定位
> 技术指标是价格与成交量的压缩表达，适合做信号过滤、风险控制和交易纪律，不适合孤立预测未来。本文仅用于学习、研究与复盘，不构成任何投资建议。

### 必须掌握的问题

- 指标参数是否符合交易周期
- 信号是否经过样本外验证
- 是否与趋势/量能/波动率共振
- 是否明确无效条件

### 实战应用流程

1. 先写清楚你的投资假设：为什么这个信号、资产或方法应该产生收益。
2. 明确数据口径：样本范围、更新时间、复权/分红/停牌处理和交易日历。
3. 做最小可行验证：先用简单规则验证方向，再逐步加入复杂模型。
4. 把成本和约束前置：手续费、滑点、冲击成本、保证金、流动性和容量都要进入测算。
5. 上线后持续复盘：记录信号、下单、成交、持仓、回撤和失效原因。

### 风险与失效条件

- 指标共线导致虚假确认
- 震荡市和趋势市参数错配
- 过度优化
- 忽略滑点和交易成本

### 复盘问题

- 这笔交易或这套模型赚的是什么钱：风险补偿、行为偏差、流动性溢价，还是偶然噪音？
- 如果市场环境反过来，最大亏损和最长恢复期会是多少？
- 当前结论是否依赖某个不可持续假设，例如低利率、低波动、充裕流动性或监管套利？
- 有没有一个更简单的基准策略能取得接近效果？

## 跨领域进阶扩展

> [!tip] 交易者视角
> 学到 `技术指标概述` 时，不要只把它当成孤立知识点。把指标当成信号过滤器和纪律工具，不能替代交易系统。优秀投资交易者会把它放入“宏观背景 - 资产选择 - 估值/信号 - 组合风险 - 交易执行 - 复盘反馈”的闭环。

### 与其他知识的连接

- 趋势、动量、均值回归和波动率
- 成交量和资金流验证
- 多周期共振与冲突
- 成本、滑点和过度交易

### 进阶训练

1. 比较指标在趋势市和震荡市的表现
2. 给每个信号定义入场、退出、止损和暂停条件
3. 用样本外数据检查参数稳定性

### 能力验收

- 能否说清楚这个主题影响的是收益来源、风险来源、交易成本、流动性还是心理纪律？
- 能否指出它在什么市场环境、资产类别或交易周期中更有效？
- 能否把它写成一条可复盘的研究或交易规则？
- 能否说明如果判断错误，组合最大损失和退出机制是什么？
