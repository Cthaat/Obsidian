# 融资融券分析

> [!note] 💡 概念解析
> 融资融券分析介绍了两融交易的核心数据字段、典型使用场景和量化查询方法，帮助投资者通过杠杆资金的动向判断市场情绪和个股走势。

## 一、融资融券概述

融资融券（两融）是投资者向券商借入资金买入股票（融资）或借入股票卖出（融券）的交易方式，是市场杠杆资金的重要体现。

### 1.1 核心数据字段

| 字段 | 说明 | 分析意义 |
|------|------|---------|
| 融资余额 | 投资者融资买入未偿还的总金额 | 市场情绪领先指标 |
| 融资买入额 | 当日新增融资买入金额 | 反映杠杆资金加仓力度 |
| 融资偿还额 | 当日偿还的融资金额 | 与买入额对比看净增减 |
| 融券余量 | 融券卖出尚未偿还的股票数量 | 做空力量指标 |
| 融券卖出量 | 当日新增融券卖出股数 | 当日做空力度 |
| 净买入 | 融资买入-融资偿还+融券偿还-融券卖出 | 杠杆资金真实方向 |

### 1.2 数据特点

- 每日T+1披露
- 金额单位统一为"万"，股数为"万股"
- 支持历史数据回溯

## 二、典型使用场景

### 2.1 趋势确认

筛选近5日融资余额加速净流入的标的，验证强势行情起点。

> [!tip] 💡 应用方法
> 融资余额持续攀升通常意味着市场做多情绪高涨，主力资金持续流入。

### 2.2 主力跟踪

| 信号 | 含义 |
|------|------|
| 融资买入额突然放大 | 杠杆资金入场 |
| 放大 + 股价突破前高 | 强势确认信号 |

### 2.3 顶部预警

| 信号 | 含义 |
|------|------|
| 融资余额单日骤降 ≥ 3% | 获利盘涌出 |
| 骤降 + 放量长上影 | 阶段性高点警示 |

### 2.4 板块轮动

按行业聚合两融余额变化，找出资金净流入最强的细分赛道。

### 2.5 风险规避

> [!warning] ⚠️ 风险信号
> 融资余额持续下行的标的，通常对应中期下跌趋势。

## 三、融资余额分析方法

### 3.1 融资余额变化方向

| 方向 | 含义 | 操作建议 |
|------|------|---------|
| 持续上升 | 多头情绪高涨 | 关注做多机会 |
| 持续下降 | 多头信心不足 | 规避或做空 |
| 横盘震荡 | 多空分歧 | 观望 |

### 3.2 融资买入额异常放大

| 情况 | 含义 |
|------|------|
| 买入额大幅增加 + 股价上涨 | 杠杆资金积极做多 |
| 买入额大幅增加 + 股价下跌 | 被动买入，可能套牢 |
| 买入额大幅减少 + 股价上涨 | 上涨动能不足 |

### 3.3 融券余量分析

| 变化 | 含义 |
|------|------|
| 融券余量激增 | 做空力量增强，短期抛压增大 |
| 融券余量持续减少 | 做空力量减弱，可能企稳 |

## 四、量化查询工具

### 4.1 工具功能

- 覆盖范围：支持查询历史日期的两融数据
- 筛选条件：日期区间与股票名称组合筛选
- 数据导出：支持CSV格式导出
- 月度分片：便于按月分析趋势

### 4.2 使用方法

1. **设置查询日期**：输入单日或起止日期
2. **输入股票名称**：模糊检索具体标的
3. **比较关键字段**：排序融资余额、融券余量等指标
4. **导出数据**：与量化策略、行情监控表联动

## 五、两融数据与其他指标的结合

| 结合指标 | 分析方法 |
|----------|---------|
| 成交量 | 融资买入放大 + 成交量放大 = 资金入场确认 |
| 龙虎榜 | 机构融资买入 + 龙虎榜机构席位 = 双重确认 |
| 北向资金 | 两融 + 北向资金同向流入 = 强势信号 |
| 技术指标 | 融资余额回升 + MACD金叉 = 趋势反转信号 |

## 六、注意事项

> [!warning] ⚠️ 风险提示
> 1. 两融数据仅供参考，不构成投资建议
> 2. 杠杆交易风险较大，需严格控制仓位
> 3. 融资余额变化可能存在滞后性
> 4. 需结合其他指标综合判断

## 课程化学习补充

> [!important] 学习定位
> 技术指标是价格与成交量的压缩表达，适合做信号过滤、风险控制和交易纪律，不适合孤立预测未来。本文仅用于学习、研究与复盘，不构成任何投资建议。

### 必须掌握的问题

- 指标参数是否符合交易周期
- 信号是否经过样本外验证
- 是否与趋势/量能/波动率共振
- 是否明确无效条件

### 实战应用流程

1. 先写清楚你的投资假设：为什么这个信号、资产或方法应该产生收益。
2. 明确数据口径：样本范围、更新时间、复权/分红/停牌处理和交易日历。
3. 做最小可行验证：先用简单规则验证方向，再逐步加入复杂模型。
4. 把成本和约束前置：手续费、滑点、冲击成本、保证金、流动性和容量都要进入测算。
5. 上线后持续复盘：记录信号、下单、成交、持仓、回撤和失效原因。

### 风险与失效条件

- 指标共线导致虚假确认
- 震荡市和趋势市参数错配
- 过度优化
- 忽略滑点和交易成本

### 复盘问题

- 这笔交易或这套模型赚的是什么钱：风险补偿、行为偏差、流动性溢价，还是偶然噪音？
- 如果市场环境反过来，最大亏损和最长恢复期会是多少？
- 当前结论是否依赖某个不可持续假设，例如低利率、低波动、充裕流动性或监管套利？
- 有没有一个更简单的基准策略能取得接近效果？
