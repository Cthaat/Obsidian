# 聚宽平台实现

> [!note] 概述
> 本文介绍如何在 JoinQuant（聚宽）量化交易平台上实现基于凯利公式的动态仓位管理策略。以双均线策略为例，集成凯利公式进行仓位控制。

## 策略架构

```
initialize() 初始化
    ↓
handle_data() 每日运行
    ├─ 计算均线信号
    ├─ 卖出逻辑 → 记录盈亏到 trade_history
    └─ 买入逻辑 → calculate_kelly_position() → 分数凯利 → 下单
```

## 完整代码实现

### 初始化函数

```python
def initialize(context):
    """初始化函数"""
    # 设定基准
    set_benchmark('000300.XSHG')
    # 开启动态复权模式(真实价格)
    set_option('use_real_price', True)
    # 过滤掉 order 系列 API 产生的比 error 级别低的 log
    log.set_level('order', 'error')
    
    # --- 策略参数设置 ---
    g.security = '000001.XSHE'  # 操作标的：平安银行
    g.short_window = 5          # 短周期均线
    g.long_window = 20          # 长周期均线
    
    # --- 凯利公式相关全局变量 ---
    g.trade_history = []        # 记录历史交易盈亏比例
    g.kelly_fraction = 0.5      # 分数凯利系数（半凯利）
    g.default_win_rate = 0.5    # 预设胜率
    g.default_odds = 2.0        # 预设盈亏比
```

### 凯利公式计算函数

```python
def calculate_kelly_position(trade_history):
    """
    根据历史交易记录计算凯利公式建议的仓位比例
    f = (p * (b + 1) - 1) / b
    """
    # 冷启动：交易记录太少时使用默认参数
    if len(trade_history) < 5:
        p = g.default_win_rate
        b = g.default_odds
    else:
        history = np.array(trade_history)
        
        # 1. 计算胜率 (p)
        wins = history[history > 0]
        losses = history[history <= 0]
        p = len(wins) / float(len(history))
        
        # 2. 计算盈亏比 (b)
        avg_win = np.mean(wins) if len(wins) > 0 else 0
        avg_loss = abs(np.mean(losses)) if len(losses) > 0 else 1.0
        
        # 避免除以 0
        if avg_loss == 0:
            b = 10.0
        else:
            b = avg_win / avg_loss

    # 3. 应用凯利公式
    if b == 0:
        return 0
    f = (p * (b + 1) - 1) / b
    
    # 期望收益为负，不开仓
    if f < 0:
        return 0
    return f
```

### 主策略逻辑

```python
def handle_data(context, data):
    """按天运行的策略逻辑"""
    security = g.security
    
    # 获取历史数据
    hist_data = attribute_history(security, g.long_window + 1, '1d', ['close'])
    close_prices = hist_data['close']
    
    if len(close_prices) < g.long_window:
        return

    # 计算均线
    ma_short = close_prices[-g.short_window:].mean()
    ma_long = close_prices[-g.long_window:].mean()
    
    # 获取当前持仓
    position = context.portfolio.positions[security]
    curr_price = close_prices[-1]
    
    # --- 卖出信号：死叉 ---
    if ma_short < ma_long and position.closeable_amount > 0:
        # 记录盈亏
        if position.avg_cost > 0:
            pnl_ratio = (curr_price - position.avg_cost) / position.avg_cost
            g.trade_history.append(pnl_ratio)
            log.info("平仓录入历史: 盈亏比率 %.2f%%" % (pnl_ratio * 100))
        
        order_target(security, 0)
        log.info("卖出 %s" % security)

    # --- 买入信号：金叉 ---
    elif ma_short > ma_long and position.total_amount == 0:
        # 计算凯利仓位
        kelly_pos = calculate_kelly_position(g.trade_history)
        
        # 应用分数凯利
        final_pos_ratio = kelly_pos * g.kelly_fraction
        
        # 边界检查
        final_pos_ratio = max(0.0, min(1.0, final_pos_ratio))
        
        # 计算目标买入金额
        total_value = context.portfolio.total_value
        target_value = total_value * final_pos_ratio
        
        if target_value > 0:
            log.info("凯利仓位: %.2f%%, 开仓金额: %.2f" % 
                     (final_pos_ratio * 100, target_value))
            order_target_value(security, target_value)
```

## 关键设计要点

### 1. 盈亏记录

在卖出时记录盈亏比例，用于后续凯利参数计算：

```python
pnl_ratio = (curr_price - position.avg_cost) / position.avg_cost
g.trade_history.append(pnl_ratio)
```

### 2. 冷启动处理

当交易次数少于 5 次时，使用预设的保守参数：

| 阶段 | 胜率来源 | 盈亏比来源 | 凯利分数 |
|------|---------|-----------|---------|
| 冷启动（< 5 笔） | 默认 50% | 默认 2.0 | 0.5 |
| 成长期（5-20 笔） | 实际计算 | 实际计算 | 0.5 |
| 成熟期（> 20 笔） | 实际计算 | 实际计算 | 可适当提高 |

### 3. 分数凯利

```python
g.kelly_fraction = 0.5  # 半凯利
final_pos_ratio = kelly_pos * g.kelly_fraction
```

> [!tip] 为什么用半凯利？
> 理论上的凯利公式假设胜率和赔率是已知且固定的，但在股市中这些参数是估算的且不稳定的。全仓凯利（Full Kelly）往往会导致巨大的资产回撤。

### 4. 边界检查

```python
final_pos_ratio = max(0.0, min(1.0, final_pos_ratio))
```

| 情况 | 处理 | 原因 |
|------|------|------|
| $f < 0$ | 返回 0 | 期望收益为负，不开仓 |
| $f > 1$ | 限制为 1.0 | 不加杠杆 |

## 常见问题

> [!faq] Q: 凯利公式计算出的仓位经常是负数？
> 凯利公式的前提是数学期望为正。如果策略历史胜率低且盈亏比也不高（即 $p(b+1) < 1$），公式会告诉你不应该下注。这说明策略本身需要优化。

> [!faq] Q: 可以直接用于实盘吗？
> 不建议。实盘需要考虑：
> 1. **加仓逻辑**：上述代码假设每次开仓都是针对总资金的重新分配
> 2. **多只股票**：需要升级为矩阵形式或平均分配
> 3. **止损影响**：凯利公式对最大亏损非常敏感，必须配合严格止损

> [!faq] Q: 如何在多只股票中使用？
> 将计算出的 `final_pos_ratio` 理解为单只股票占总资金的最大比例，或者将总资金平分成 N 份，每一份内部使用凯利公式决定投入比例。

## 扩展优化方向

| 优化方向 | 描述 | 难度 |
|---------|------|------|
| 动态凯利分数 | 根据市场波动率调整 kelly_fraction | 中 |
| 多标的支持 | 组合凯利比分配 | 中 |
| 滑点模拟 | 在盈亏计算中扣除滑点 | 低 |
| 资金曲线反馈 | 回撤时自动降低仓位 | 中 |
| 在线学习 | 实时更新凯利参数 | 高 |
