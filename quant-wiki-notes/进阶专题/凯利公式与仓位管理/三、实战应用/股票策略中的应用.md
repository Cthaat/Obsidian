# 股票策略中的应用

> [!note] 核心思路
> 在股票交易中，凯利公式的核心目的是**根据策略的历史胜率和盈亏比，动态计算最优的仓位比例**，从而在控制风险的前提下最大化长期复利增长。

## 策略实现思路

1. **数据记录**：记录每一笔平仓交易的盈亏情况
2. **参数计算**：根据历史记录动态计算胜率（$p$）和盈亏比（$b$）
3. **仓位控制**：在开仓信号触发时，调用凯利公式计算仓位，并结合"半凯利"等风控手段防止仓位过重

## 双均线策略 + 凯利公式

以下是一个完整的双均线策略示例，集成了凯利公式进行仓位管理：

### 策略参数设置

| 参数 | 值 | 说明 |
|------|------|------|
| 标的 | 平安银行 (000001.XSHE) | 操作标的 |
| 短周期均线 | 5 日 | 买入/卖出信号 |
| 长周期均线 | 20 日 | 趋势判断 |
| 凯利分数 | 0.5 | 半凯利 |
| 默认胜率 | 0.5 | 冷启动预设 |
| 默认盈亏比 | 2.0 | 冷启动预设 |

### 核心代码逻辑

```python
import numpy as np

# 全局变量
trade_history = []  # 记录历史交易盈亏比例
kelly_fraction = 0.5  # 半凯利
default_win_rate = 0.5
default_odds = 2.0

def calculate_kelly_position(trade_history):
    """根据历史交易记录计算凯利公式建议的仓位比例"""
    # 冷启动：交易记录太少时使用默认参数
    if len(trade_history) < 5:
        p = default_win_rate
        b = default_odds
    else:
        history = np.array(trade_history)
        
        # 计算胜率
        wins = history[history > 0]
        losses = history[history <= 0]
        p = len(wins) / float(len(history))
        
        # 计算盈亏比
        avg_win = np.mean(wins) if len(wins) > 0 else 0
        avg_loss = abs(np.mean(losses)) if len(losses) > 0 else 1.0
        
        b = avg_win / avg_loss if avg_loss > 0 else 10.0

    # 应用凯利公式
    if b == 0:
        return 0
    f = (p * (b + 1) - 1) / b
    
    # 期望收益为负，不开仓
    if f < 0:
        return 0
    return f

def execute_buy(context, security):
    """买入逻辑"""
    # 计算凯利仓位
    kelly_pos = calculate_kelly_position(trade_history)
    
    # 应用分数凯利
    final_pos_ratio = kelly_pos * kelly_fraction
    
    # 边界检查
    final_pos_ratio = max(0.0, min(1.0, final_pos_ratio))
    
    # 计算目标买入金额
    total_value = context.portfolio.total_value
    target_value = total_value * final_pos_ratio
    
    if target_value > 0:
        order_target_value(security, target_value)

def execute_sell(context, security, curr_price):
    """卖出逻辑"""
    position = context.portfolio.positions[security]
    
    # 记录盈亏比例
    if position.avg_cost > 0:
        pnl_ratio = (curr_price - position.avg_cost) / position.avg_cost
        trade_history.append(pnl_ratio)
    
    # 执行清仓
    order_target(security, 0)
```

### 交易流程

```
金叉信号（短均线上穿长均线）
    ↓
无持仓？
    ├─ 是 → 计算凯利仓位 → 应用半凯利 → 边界检查 → 买入
    └─ 否 → 忽略信号

死叉信号（短均线下穿长均线）
    ↓
有持仓？
    ├─ 是 → 记录盈亏 → 清仓卖出
    └─ 否 → 忽略信号
```

> [!warning] A股特殊性
> 在 A 股市场，由于不能做空且涨跌幅限制（10%/20%），夏普比率对上涨波动的"惩罚"不太合适。建议使用卡玛比率或其他更适合 A 股的指标来辅助评估。

## 多股票组合管理

### 简单方法

将计算出的 `final_pos_ratio` 理解为**单只股票占总资金的最大比例**，或者将总资金平分成 N 份，每一份内部使用凯利公式决定投入比例。

### 进阶方法：凯利比分配

$$\text{股票 A 资金比例} = \frac{f_A^*}{f_A^* + f_B^* + \cdots + f_N^*}$$

```python
def allocate_capital(kelly_values):
    """根据凯利值分配资金"""
    total = sum(kelly_values.values())
    if total == 0:
        return {k: 0 for k in kelly_values}
    return {k: v / total for k, v in kelly_values.items()}
```

## 常见问题

> [!faq] Q: 凯利公式计算出的仓位经常是负数？
> 凯利公式的前提是**数学期望为正**。如果策略历史胜率低且盈亏比也不高（即 $p(b+1) < 1$），公式会告诉你不应该下注。这说明策略本身需要优化，或者近期市场表现不适合该策略。

> [!faq] Q: 可以直接用于实盘吗？
> 不建议直接使用。实盘中需要考虑：
> 1. **加仓逻辑**：上述代码假设每次开仓都是针对总资金的重新分配
> 2. **多只股票**：需要升级为矩阵形式或平均分配
> 3. **止损影响**：凯利公式对最大亏损非常敏感，必须配合严格的止损策略

> [!faq] Q: 凯利公式对最大亏损非常敏感？
> 是的。如果一笔交易亏损了 50%，凯利公式会大幅降低建议仓位。因此**严格的止损**是使用凯利公式的前提。

## 实战建议

1. **永远保留安全边际**：即使凯利公式建议 30% 仓位，实际也不应超过 15-20%
2. **定期再平衡**：每月或每季度重新计算各策略的凯利值
3. **心理承受测试**：你的仓位应该让你晚上睡得着觉
4. **考虑杠杆成本**：如果使用融资，需在凯利公式中扣除利息成本
