# 半凯利与分数凯利

> [!note] 核心思想
> 理论上的凯利公式假设胜率和赔率是已知且固定的，但在实际市场中这些参数是估算的且不稳定的。全仓凯利（Full Kelly）往往会导致巨大的资产回撤。业界通常使用"半凯利"或"分数凯利"来平滑风险。

## 为什么需要分数凯利

### 凯利公式的实际问题

1. **参数不确定性**：胜率和赔率是估计值，存在误差
2. **非二元结果**：实际交易结果不是简单的赢/输，而是连续分布
3. **心理承受能力**：全凯利的净值波动可能超出投资者心理承受范围
4. **交易成本**：手续费、滑点等会侵蚀收益

> [!example] 参数误差的影响
> 假设真实胜率 55%，但估计为 60%：
> - 全凯利：$f^* = 0.4$（基于错误估计）
> - 实际最优：$f^* = 0.3$（基于真实胜率）
> - 误差导致**过度下注**，增加爆仓风险

## 半凯利策略（Half Kelly）

### 定义

$$f_{\text{half}} = \frac{f^*}{2} = \frac{bp - q}{2b}$$

### 数学性质

| 指标 | 全凯利 | 半凯利 | 变化 |
|------|--------|--------|------|
| 长期增长率 | $g(f^*)$ | $\approx 0.75 \cdot g(f^*)$ | 下降约 25% |
| 净值波动 | $\sigma$ | $\approx 0.5\sigma$ | 下降约 50% |
| 最大回撤 | 高 | 中 | 显著降低 |
| 爆仓概率 | 较高 | 低 | 大幅降低 |

> [!tip] 索普的建议
> "半凯利是普通交易者的最佳选择，因为它在收益与安全之间找到了更好的平衡。"
> 
> 半凯利虽然降低了长期复利增长率（从最优值下降约 1/4），但显著降低了爆仓概率和净值波动，更适合风险厌恶型交易员。

### 实际应用

```python
# 半凯利仓位计算
kelly_fraction = 0.5  # 半凯利系数

def calculate_position(trade_history):
    kelly_f = calculate_kelly_position(trade_history)
    final_pos = kelly_f * kelly_fraction
    return max(0.0, min(1.0, final_pos))
```

## 分数凯利策略（Fractional Kelly）

### 定义

$$f_{\text{fractional}} = k \cdot f^* = k \cdot \frac{bp - q}{b}$$

其中 $k \in (0, 1)$ 为分数系数。

### 常用分数系数

| 系数 $k$ | 名称 | 适用场景 | 风险偏好 |
|---------|------|---------|---------|
| 1.0 | 全凯利 | 理论场景，参数精确已知 | 激进 |
| 0.75 | 3/4 凯利 | 参数较可靠 | 较激进 |
| 0.5 | 半凯利 | 大多数实盘场景 | 中性 |
| 0.25 | 1/4 凯利 | 高不确定性市场 | 保守 |
| 0.1 | 1/10 凯利 | 初期参数不确定 | 极保守 |

### 选择指南

```
参数可靠性高 + 市场稳定 → k = 0.5 ~ 0.75
参数可靠性中 + 市场波动 → k = 0.25 ~ 0.5
参数可靠性低 + 市场不确定 → k = 0.1 ~ 0.25
初期冷启动阶段 → k = 0.1 ~ 0.25
```

> [!warning] 实盘注意事项
> 即使凯利公式建议 30% 仓位，实际也不应超过 15-20%。永远保留安全边际。

## 多策略组合的凯利比

当同时运行多个策略时，使用凯利比（Kelly Ratio）分配资金：

$$\text{策略 A 资金比例} = \frac{f_A^*}{f_A^* + f_B^* + f_C^* + \cdots}$$

> [!example] 示例
> 三个策略的凯利值分别为 0.3、0.2、0.1：
> - 策略 A：$0.3 / 0.6 = 50\%$
> - 策略 B：$0.2 / 0.6 = 33.3\%$
> - 策略 C：$0.1 / 0.6 = 16.7\%$
> 
> 资金自动向表现更好的策略倾斜，同时保持分散化。

## 冷启动处理

在没有足够交易记录时，需要使用预设的保守参数：

```python
# 冷启动配置
default_win_rate = 0.5    # 预设胜率 50%
default_odds = 2.0        # 预设盈亏比 2:1
kelly_fraction = 0.5      # 半凯利

# 当交易次数少于 5 次时使用默认参数
if len(trade_history) < 5:
    p = default_win_rate
    b = default_odds
else:
    # 使用实际历史数据计算
    p = calculate_win_rate(trade_history)
    b = calculate_odds(trade_history)
```

> [!tip] 为什么冷启动用保守参数？
> 防止初期样本不足导致仓位计算极端。随着交易数据积累，逐步过渡到基于实际数据的计算。

## 边界检查

计算出的凯利值需要进行边界检查：

```python
def apply_kelly_bounds(f, max_leverage=1.0):
    """边界检查与限制"""
    if f < 0:
        return 0  # 期望收益为负，不开仓
    if f > max_leverage:
        return max_leverage  # 不超过最大杠杆
    return f
```

| 情况 | 处理 | 原因 |
|------|------|------|
| $f < 0$ | 返回 0 | 期望收益为负，不应开仓 |
| $f > 1$ | 限制为 1.0 | 不加杠杆（保守策略） |
| $f > \text{max}$ | 限制为 max | 遵守风控规则 |
