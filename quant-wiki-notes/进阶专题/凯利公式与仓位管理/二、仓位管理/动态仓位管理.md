# 动态仓位管理

> [!note] 核心思想
> 直接应用基础凯利公式会遇到致命问题——市场中的胜率和赔率是动态变化的。动态仓位管理通过实时调整参数，使凯利公式更好地适应实际市场环境。

## 动态调整三要素

### 一、波动率适配（核心中的核心）

当市场波动率上升时，降低仓位；波动率下降时，可适当提高仓位。

```python
def dynamic_position_size(volatility, max_risk=0.02):
    """根据波动率动态调整仓位

    Args:
        volatility: 最近20日波动率（年化）
        max_risk: 单笔最大风险敞口
    Returns:
        调整后的仓位比例
    """
    base_size = 0.1  # 基础仓位比例
    # 波动率放大时缩减仓位
    adj_factor = min(1.0, 0.2 / (volatility + 1e-5))
    return base_size * adj_factor
```

| 波动率水平 | 调整因子 | 仓位变化 |
|-----------|---------|---------|
| < 10% | 1.0 | 正常仓位 |
| 10% ~ 20% | 0.5 ~ 1.0 | 适度减仓 |
| 20% ~ 30% | 0.33 ~ 0.5 | 明显减仓 |
| > 30% | < 0.33 | 大幅减仓 |
| > 50% | 熔断 | 极端行情保护 |

### 二、资金曲线反馈（防止死亡螺旋）

```
当前净值
    ↓
回撤 > 阈值?
    ├─ 是 → 仓位降至 50% → 等待净值恢复 80%
    └─ 否 → 正常凯利计算
```

> [!warning] 死亡螺旋
> 连续亏损 → 仓位不变 → 更大亏损 → 更大回撤 → 最终爆仓。资金曲线反馈机制通过在回撤时主动降低仓位来打破这个恶性循环。

### 三、黑天鹅防护（极端行情仓位熔断）

```python
def black_swan_protection(position, volatility):
    """极端行情仓位熔断

    Args:
        position: 当前计算的仓位
        volatility: 当前波动率
    Returns:
        调整后的仓位
    """
    if volatility > 0.5:  # 波动率超过 50%
        return min(position, 0.02)  # 单品种仓位不超过总资金 2%
    return position
```

> [!example] 2020年原油宝事件启示
> 极端行情下，常规的仓位管理可能失效。需要设置硬性的仓位上限，防止黑天鹅事件导致不可挽回的损失。

## 动态再平衡规则

| 条件 | 动作 | 原因 |
|------|------|------|
| 净值增长 20% | 重新计算仓位 | 适应新的资金规模 |
| 连续 3 次止损 | 仓位降至 50% | 策略可能失效 |
| 波动率 VIX > 35 | 启动半仓模式 | 市场不确定性增加 |
| 波动率 > 50% | 熔断，仓位上限 2% | 极端行情保护 |

## 滚动窗口参数估计

避免使用全部历史数据导致过度拟合，采用滚动窗口：

```python
import pandas as pd

def rolling_kelly_params(data, window_years=3):
    """滚动窗口计算凯利参数

    Args:
        data: 包含交易记录的 DataFrame
        window_years: 滚动窗口年数
    Returns:
        各时点的凯利参数
    """
    params = {}
    for end_date in pd.date_range(start='2018-01-01', periods=24, freq='M'):
        start_date = end_date - pd.DateOffset(years=window_years)
        data_window = data.loc[start_date:end_date]

        # 计算胜率和赔率
        wins = data_window[data_window['pnl'] > 0]
        losses = data_window[data_window['pnl'] <= 0]

        p = len(wins) / len(data_window)
        b = abs(wins['pnl'].mean() / losses['pnl'].mean()) if len(losses) > 0 else 10.0

        f = (b * p - (1 - p)) / b
        params[end_date] = {'p': p, 'b': b, 'f': f}

    return params
```

> [!tip] 窗口选择
> - **太短**（< 1年）：样本不足，参数不稳定
> - **太长**（> 5年）：可能包含已过时的市场结构
> - **推荐**：2-3 年滚动窗口

## 滑点对赔率的影响

> [!warning] 回测 vs 实盘
> 回测假设滑点 0.1% vs 实盘滑点 0.5% = 策略直接由盈转亏

高频策略尤其要注意滑点成本对赔率 $b$ 的侵蚀。建议在计算赔率时扣除预估的滑点和手续费。

## 组合风险控制

多品种组合时，各品种仓位简单叠加会导致风险敞口失控。必须用组合 VaR 控制：

$$\text{VaR} = \sqrt{\mathbf{w}^T \Sigma \mathbf{w}} \cdot \Phi^{-1}(\alpha)$$

其中 $\Sigma$ 是资产协方差矩阵，$\Phi^{-1}$ 是标准正态逆累积分布。

```python
def risk_parity_weight(returns):
    """风险平价权重计算"""
    cov_matrix = returns.cov()
    inv_vol = 1 / np.diag(cov_matrix)
    return inv_vol / inv_vol.sum()
```

## 完整的动态凯利系统架构

```
市场数据源
    ↓
波动率计算模块
    ↓
风险检查 ──→ 异常 ──→ 熔断机制
    ↓ 正常
凯利核心计算
    ↓
头寸规模调整
    ↓
订单执行
    ↓
持仓监控
```

```python
class DynamicKellySystem:
    def __init__(self, capital=1000000):
        self.capital = capital
        self.positions = {}

    def calculate_position(self, symbol):
        """核心仓位计算"""
        # 1. 计算基础凯利值
        win_rate = self._calc_win_rate(symbol)
        payoff_ratio = self._calc_payoff_ratio(symbol)
        kelly_f = (payoff_ratio * win_rate - (1 - win_rate)) / payoff_ratio

        # 2. 波动率调节
        vol_adj = min(1.0, 0.25 / self.volatility)
        adj_f = kelly_f * vol_adj

        # 3. 回撤控制
        if self.max_drawdown > 0.2:
            adj_f *= 0.5

        # 4. 黑天鹅熔断
        if self.volatility > 0.4:
            adj_f = min(adj_f, 0.05)

        return self.capital * adj_f
```

## 课程化学习补充

> [!important] 学习定位
> 仓位管理解决的是在不确定世界中活得足够久；凯利公式提供上限，实盘通常用分数凯利和风险预算。本文仅用于学习、研究与复盘，不构成任何投资建议。

### 必须掌握的问题

- 胜率和赔率是否有稳定样本
- 最大亏损是否可承受
- 相关仓位是否合并计算
- 连续亏损时是否自动降杠杆

### 实战应用流程

1. 先写清楚你的投资假设：为什么这个信号、资产或方法应该产生收益。
2. 明确数据口径：样本范围、更新时间、复权/分红/停牌处理和交易日历。
3. 做最小可行验证：先用简单规则验证方向，再逐步加入复杂模型。
4. 把成本和约束前置：手续费、滑点、冲击成本、保证金、流动性和容量都要进入测算。
5. 上线后持续复盘：记录信号、下单、成交、持仓、回撤和失效原因。

### 风险与失效条件

- 高估胜率
- 忽略尾部风险
- 相关性在危机中上升
- 满凯利导致心理和资金双重崩溃

### 复盘问题

- 这笔交易或这套模型赚的是什么钱：风险补偿、行为偏差、流动性溢价，还是偶然噪音？
- 如果市场环境反过来，最大亏损和最长恢复期会是多少？
- 当前结论是否依赖某个不可持续假设，例如低利率、低波动、充裕流动性或监管套利？
- 有没有一个更简单的基准策略能取得接近效果？
