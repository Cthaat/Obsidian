# 前沿研报与论文学习导航

> [!note] 核心问题
> `量化前沿`、`学术论文`、`人工智能` 区信息密度高、营销与洞见混杂。本篇规定 **读什么顺序、怎么做四行摘要、如何接到因子/ML/作业**，并给 AI 应用文降噪原则。

## 学习目标

1. 区分：券商研报 / 学术论文 / AI 产品文 的用途。  
2. 用统一摘要模板消化任意一篇材料。  
3. 会从研报目录选「多因子/AI/高频」系列抽样。  
4. 把结论接到 [[因子投资实操导航]] 或 [[机器学习交易实操导航]]，而非直接实盘。  
5. 设置每周前沿阅读上限。  

## 三类材料怎么用

| 类型 | 位置 | 用途 | 风险 |
|---|---|---|---|
| 券商研报系列 | `量化前沿/券商研报/` | A 股因子/ML 本地化线索 | 样本期、拥挤、不可复现细节 |
| 学术/列表 | [[学术论文/目录]]、arXiv | 方法与证据层级 | 预印本质量参差 |
| AI 应用/工具文 | `人工智能/` | 工具想象力、NLP/Agent | 易夸大、缺严格回测 |

外链习惯：[arXiv q-fin](https://arxiv.org/archive/q-fin)、[SSRN](https://www.ssrn.com/)；见 [[学习网站与社区导航]]。

## 阅读纪律

| 规则 | 说明 |
|---|---|
| 先方法后故事 | 数据、区间、成本、是否样本外 |
| 四行摘要 | 假设 / 数据 / 结论 / 可复现性 |
| 每周上限 | 建议 ≤2 篇深读 + 若干标题扫描 |
| 禁止 | 研报曲线 → 次日跟单 |
| 对接 | 有启发则开 EXP 或写入否决库 |

## 四行摘要模板

见可复制范例：[[研报四行摘要范例]]（含虚拟样例 + 空白作业位）。

```markdown
## 材料：标题 / 来源 / 日期
- 假设：
- 数据与样本：
- 结论（作者声称）：
- 可复现性与我的质疑：
- 是否进入我的研究管道：是/否/仅灵感
```

## 推荐路径（2 周）

| 天 | 动作 |
|---:|---|
| 1 | 读本导航 + [[量化前沿/目录]] 结构 |
| 2 | 选一个多因子系列目录（如华泰/中信）只读其目录与 1 篇 |
| 3 | 四行摘要 + 与 [[因子投资实操导航]] 对照 |
| 4 | AI/深度学习系列中选 1 篇略读 |

## AI 区降噪原则

`人工智能/目录` 中有偏宣传的表述（如「革命性改变」）。阅读时强制问：

1. 有无严格样本外与成本？  
2. 是否只是投研效率工具（摘要、代码助手）？  
3. 是否把预测难度说清楚了？  

合格用途示例：研报/新闻结构化、代码辅助、实验记录——**不是**替代边缘与风控。

## 作业（P4-1）

| 交付 | 标准 |
|---|---|
| 四行摘要 ×2 | 一篇因子研报向 + 一篇 ML/AI 向 |
| 挑刺 ≥3 | 针对 ML/AI 文 |
| 管道决定 | 灵感 / EXP / 否决库 三选一写清 |

## 常见误区

| 误区 | 更好的理解 |
|---|---|
| 研报因子列表=可交易 | 拥挤与成本 |
| 论文引用多=可实盘 | 执行与容量 |
| AI 文标题党 | 用清单打回原形 |
| 同时追所有券商系列 | 选 1 家系列跟完更有用 |
