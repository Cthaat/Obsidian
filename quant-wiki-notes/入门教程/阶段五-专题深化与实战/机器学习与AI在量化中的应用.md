# 机器学习与AI在量化中的应用

> [!note] 核心问题
> 机器学习能处理高维特征、捕捉非线性关系、自动化研究流程，但它不能凭空造出不存在的信号，也不能消除过拟合。金融数据信噪比极低，ML 在这里不是印钞机，而是一把容易割伤自己的刀。这篇要回答的是：它到底能做什么、不能做什么、怎么用才不自欺。

## 学习目标

读完这篇，你要能做到：

1. 分清机器学习在量化里擅长和不擅长的任务，不对 AI 抱有不切实际的期待。
2. 理解金融数据为什么比一般机器学习问题难得多：信噪比低、非平稳、样本少、有反身性。
3. 把因子、价量、基本面和另类数据加工成规范的特征与标签，并避开常见泄露点。
4. 识别时间序列交叉验证的陷阱，掌握时序切分、清洗（purge）、禁运（embargo）与前向滚动。
5. 在线性、树模型、神经网络之间做出有依据的取舍，并优先选择可解释、稳健的方案。

## 一、先校准期待：ML 能做什么，不能做什么

机器学习是一组从数据中拟合规律的方法。在量化里，它真正的价值在于处理人脑难以同时权衡的高维信息，而不是“预测涨跌”这种被过度宣传的承诺。

| ML 擅长的 | ML 不擅长甚至做不到的 |
|---|---|
| 处理几十上百个特征的高维问题 | 在没有信号的地方造出信号 |
| 捕捉特征之间的非线性与交互 | 预测真正随机的部分（大量价格变动就是噪声） |
| 自动化重复研究：特征筛选、参数搜索 | 替你消除过拟合（它反而更容易过拟合） |
| 从文本、图像等非结构化数据提取信息 | 理解它从未见过的市场状态（非平稳） |
| 在大量样本上做稳定的模式识别 | 在样本极少时给出可靠的尾部判断 |

一句话：**机器学习能更高效地榨取数据里已有的信息，但不能创造数据里没有的信息。** 如果一个标的的未来收益里 95% 是噪声、5% 是可预测成分，再强的模型也只能在那 5% 上努力，而且很容易把噪声当成信号学进去。

> [!warning] 把因果讲清楚
> ML 默认学的是相关，不是因果。它会忠实地放大数据里的一切规律，包括泄露、偏差和巧合。模型越强，把这些假规律学得越像真的，反而越危险。

## 二、金融数据为什么这么难

很多在图像、文本上奏效的做法，搬到金融里会失灵。原因不在算法，而在数据本身的性质。

| 特性 | 一般 ML 场景 | 金融数据 | 后果 |
|---|---|---|---|
| 信噪比 | 高（猫狗清晰可分） | 极低（收益绝大部分是噪声） | 容易把噪声学成信号 |
| 平稳性 | 大体平稳 | 非平稳，市场状态会变 | 历史规律可能失效 |
| 样本量 | 海量且独立 | 有限且高度自相关 | 有效样本远少于看起来的 |
| 反身性 | 预测不改变对象 | 信号被用了就可能失效 | 公开有效的策略会衰减 |
| 标签质量 | 标注准确 | 标签含噪（未来收益本身随机） | 模型在学一个很模糊的目标 |

这几点叠加，意味着：在金融里，一个看起来很漂亮的模型，更可能是过拟合而不是真发现。承接 [[回测方法论]] 里反复强调的警告，这里把怀疑精神继续贯彻到建模环节。

### 有效样本比你以为的少

很多人用十年日线，觉得有约 2500 个样本。但日收益高度自相关、市场状态成块出现：2015 杠杆牛、2018 熊市、2020 疫情冲击各自是一段，真正“独立的市场情景”可能只有十几段。模型能见到的极端环境屈指可数，却要对未来未见过的环境负责。

## 三、ML 量化研究流程

整体流程和 [[回测方法论]] 一致，只是把“信号”换成“模型”。关键在于：**每一步都可能引入数据泄露**，下图把容易出问题的环节标了出来。

```mermaid
graph LR
    A[原始数据] --> B[特征工程]
    B --> C[标签构建]
    C --> D[模型训练]
    D --> E[时序验证]
    E --> F[策略回测]
    F --> G[实盘部署]
    G --> H[持续监控]
    H -->|衰减则重训/下线| D
```

| 环节 | 常见泄露/陷阱 |
|---|---|
| 特征工程 | 用全样本统计量做标准化（偷看了未来） |
| 标签构建 | 标签窗口与特征窗口重叠 |
| 模型训练 | 在验证集上反复调参，等于变相拟合 |
| 时序验证 | 普通 K 折打乱时间，未来信息泄露到过去 |
| 策略回测 | 忽略成本、涨跌停、停牌、T+1 |
| 实盘部署 | 训练用的数据在实盘当下根本拿不到 |

## 四、特征工程：把投资逻辑变成数字

特征是模型的输入。好特征来自对市场的理解，而不是把几百个指标一股脑塞进去。这一步直接呼应 [[因子投资体系]] 的特征思想：你已经会构造因子，ML 只是同时使用更多因子并允许非线性组合。

常见特征来源：

| 来源 | 例子 | 呼应 |
|---|---|---|
| 因子 | 价值、动量、质量、低波、规模 | [[因子投资体系]] |
| 价量 | 收益率、波动率、换手率、量价背离 | 技术面 |
| 基本面 | ROE、毛利率、现金流质量、负债率 | 财报分析 |
| 另类数据 | 文本情绪、卫星、供应链、资金流 | [[另类数据与信息优势]] |

特征加工的基本动作，和因子处理完全一致：

1. **去极值**：截断或缩尾极端值，避免个别数据主导。
2. **标准化**：转成可比分数（如 Z-score），但统计量只能用训练期数据估计。
3. **行业中性**：剔除行业系统性差异，例如银行天然低 PB。
4. **缺失处理**：财报缺失、停牌、上市不足，需要明确规则，不能默默填未来值。

> [!tip] 特征不是越多越好
> 在低信噪比环境里，每增加一个无用特征，都给模型多一个“记住噪声”的机会。少而有逻辑的特征，往往比几百个堆砌的特征更稳健。

## 五、标签构建：你到底在预测什么

标签是模型要学习的目标。金融里标签的定义比特征更容易出错，因为“未来收益”本身就含大量噪声。

### 固定时间跨度收益

最直接的做法：预测未来 $k$ 天的收益。

$$
y_t = \frac{P_{t+k}}{P_{t}} - 1
$$

简单，但有两个隐患：一是固定持有期忽略了中途的大幅回撤；二是如果特征里用到了 $t$ 到 $t+k$ 之间的信息，就和标签窗口重叠，造成泄露。

### 三重障碍法

更贴近真实交易的做法是三重障碍法（triple barrier）：为每笔模拟交易同时设三条线——止盈、止损、到期，看哪条先被触及，标签就是那个结果。

| 先触及 | 标签直觉 |
|---|---|
| 上沿（止盈） | +1，方向判断成功 |
| 下沿（止损） | -1，方向判断失败 |
| 到期（时间） | 0 或按到期收益，趋势不明显 |

它的好处是把风险管理和持有期纳入标签本身，比“固定 5 日收益”更接近你实际会怎么交易。代价是实现更复杂，且障碍宽度的选择本身又是一个容易过拟合的参数。

## 六、头号敌人：过拟合

过拟合在任何 ML 任务里都存在，但在金融里尤其致命，原因正是第二节那几条性质的组合：

- **信噪比低**：模型很容易把噪声当信号记下来。
- **反复试错**：你换模型、调参数、改特征几十上百次，总有一次在历史上“碰巧”很好。
- **样本少**：可供检验的独立市场环境本就不多，经不起反复消耗。

这意味着即便你每一步都规范，只要在同一份历史数据上试得够多，最终选出的“最好模型”也大概率是被噪声选出来的。点名 [[回测方法论]]：那里讲的数据挖掘偏差，在 ML 流程里会被自动化放大成百上千倍——网格搜索一次就是几百个策略。

> [!warning] 自由度即风险
> 模型参数、特征数量、调参次数，本质都是自由度。自由度越高，越能完美拟合历史，也越可能在实盘原形毕露。在低信噪比环境里，克制自由度比追求精度更重要。

## 七、交叉验证的陷阱

这是 ML 量化里最隐蔽、也最致命的错误：直接套用普通机器学习的 K 折交叉验证。

普通 K 折会随机打乱样本，于是“未来”的数据被分到了训练集，模型在预测“过去”——这在时间序列里是彻底的数据泄露，会让验证分数虚高，实盘立刻崩塌。

| 错误做法 | 为什么错 | 正确做法 |
|---|---|---|
| 随机打乱 K 折 | 未来信息泄露进训练集 | 按时间先后切分，训练永远在验证之前 |
| 训练/验证紧邻无间隔 | 自相关使两端信息渗透 | 在交界处做清洗（purge）与禁运（embargo） |
| 全样本标准化后再切分 | 标准化偷看了未来统计量 | 标准化参数只用训练期估计 |
| 单一固定切分 | 只反映一个时期 | 前向滚动（walk-forward）多窗口验证 |

几个关键概念：

- **时序切分**：训练集必须完全早于验证集，模拟真实中“只能用过去预测未来”。
- **清洗（purge）**：当标签是未来 $k$ 天收益时，训练样本若其标签窗口与验证期重叠，必须剔除，否则信息从标签端泄露。
- **禁运（embargo）**：在训练与验证交界后再空出一小段时间不用，隔断自相关带来的渗透。
- **前向滚动（walk-forward）**：窗口不断向前推移，反复“用过去训练、在紧邻的未来检验”，更贴近实盘上线后的处境。

```python
def walk_forward_splits(n, train_size, test_size, embargo):
    # 前向滚动：训练窗口始终早于测试窗口，并在交界处留出禁运间隔
    start = 0
    while start + train_size + embargo + test_size <= n:
        train = range(start, start + train_size)
        test_lo = start + train_size + embargo
        test = range(test_lo, test_lo + test_size)
        yield train, test
        start += test_size
```

把验证做对，往往比把模型做复杂更能提升实盘可靠性。一个验证流程有泄露的项目，模型分数再高也没有意义。

## 八、常见模型与取舍

没有“最好”的模型，只有适合数据量、信噪比和可解释性要求的模型。在金融这种低信噪比、表格化、样本有限的场景里，简单模型常常胜出。

| 模型 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| 线性 / 正则化（Rid, Lasso） | 透明、稳健、参数少、不易过拟合 | 抓不到非线性 | 信噪比低、样本少、首选基线 |
| 树模型 / 梯度提升（GBDT） | 抓非线性与交互、表格数据强、需调参少 | 可能过拟合、外推差 | 因子组合、结构化特征丰富 |
| 神经网络 / 深度学习 | 表达力强、能处理序列与非结构化数据 | 数据需求大、易过拟合、难解释 | 海量数据、文本/序列建模 |

实务上的经验顺序：**先用正则化线性模型建立基线，再考虑 GBDT，只有在数据量和问题结构真正需要时才动用深度学习。** 如果复杂模型相对线性基线没有稳定、显著、扣除成本后的提升，就该选简单的那个。

> [!note] 复杂度要靠收益买单
> 每提升一级模型复杂度，都要用样本外的、扣成本的、跨时期稳定的超额收益来证明它值得。证明不了，就退回更简单的版本。

## 九、可解释性与稳健性

在低信噪比环境里，能解释、结构简单的模型往往更可靠——不是因为它精度更高，而是因为它更难把噪声学进去，也更容易在失效时被你看懂。

| 工具 | 作用 | 注意 |
|---|---|---|
| 特征重要性 | 看模型主要依赖哪些特征 | 高度相关的特征会互相稀释重要性 |
| 单调性约束 | 强制某特征方向符合经济逻辑 | 用先验对抗过拟合 |
| 稳定性检验 | 看不同时期特征重要性是否一致 | 排序剧烈变化是失效信号 |
| 子样本/扰动测试 | 换时段、加噪后结论是否稳定 | 一碰就变说明在拟合噪声 |

判断一个特征是否真有用，不只看它在样本内贡献多大，更要看它的方向和重要性在不同时期是否稳定。一个只在某一段历史里突然很重要的特征，更可能是巧合。

## 十、深度学习与大语言模型的角色

近年深度学习和大语言模型（LLM）很受关注，但要冷静看待它们在量化里的位置：它们扩展了能处理的数据类型，并没有改变金融信噪比低的根本约束。

| 应用方向 | 能带来什么 | 主要风险 |
|---|---|---|
| 序列建模（RNN/Transformer） | 建模时间依赖与长程关系 | 数据需求大、极易过拟合 |
| 另类数据/文本信息提取 | 从公告、新闻、研报中结构化信息 | 时间戳错配导致前视 |
| 研究与代码加速（LLM） | 写样板代码、整理资料、做思路探索 | 幻觉、看似合理实则错误 |

几条务实的纪律：

- **NLP 用于信息提取，而非直接预测涨跌。** 把文本变成一个干净、可回测的特征，再交给后续流程，比让模型端到端“读新闻出买卖”要可靠得多。
- **警惕前视。** 文本类数据最容易出问题的是时间戳：必须用“信息真正可获得的时刻”，而不是事件发生或数据入库的时刻。这与 [[另类数据与信息优势]] 的核心提醒一致。
- **LLM 是研究助手，不是策略本身。** 它能加速你写代码、读资料，但它会一本正经地编造看似合理的结论，任何由它产出的逻辑、引用和数字都要独立核对。

深度学习不是“一定比线性强”的升级版，而是另一类工具：数据对路、样本充足时它有独特价值；否则它只是一个更容易过拟合、更难解释的选择。

## 十一、ML 不是银弹：常见失败模式

把容易踩的坑列成清单，建模前后各对照一遍：

1. 验证流程有泄露（随机 K 折、全样本标准化、标签窗口重叠）。
2. 在验证集上反复调参，把验证集也变相拟合了。
3. 特征里混入了当时拿不到的信息（财报用报告期而非公告日）。
4. 回测忽略成本、滑点、涨跌停、停牌、T+1，超额收益是假的。
5. 训练所需数据在实盘当下根本无法及时获得。
6. 模型在历史唯一一段行情上有效，换环境即失效。
7. 用复杂模型替代了缺失的经济逻辑，结果只是高级版的曲线拟合。

> [!warning] 看起来能赚 ≠ 真能赚
> 以上每一条都能单独让一个“样本内表现优异”的模型在实盘归零。ML 不会替你回避它们，只会让错误更隐蔽、更自信。

## 常见误区

| 误区 | 实际情况 |
|---|---|
| 准确率高就能赚钱 | 方向准确率和盈利不是一回事；少数大亏可吃光多数小赚，还要扣成本 |
| 模型越复杂越好 | 低信噪比下复杂模型更易过拟合；简单稳健模型常更可靠 |
| 样本内表现好就有效 | 样本内好几乎是必然；只有扣成本的样本外稳定表现才算数 |
| AI 能预测市场 | 大量价格变动是噪声，无法预测；ML 只能榨取已有的微弱信号 |
| 深度学习一定优于线性 | 取决于数据量与问题结构；金融表格数据上线性/GBDT 常不输甚至更好 |
| 数据越多模型越准 | 若数据非平稳，旧样本可能是误导；样本多不等于有效信息多 |

## 练习：给一个「用 ML 预测明日涨跌」的方案挑刺

某人提出如下方案，请找出其中至少 3 个可能的数据泄露或过拟合点：

> 取全市场十年日线，特征用过去若干日收益、成交量，以及当日全样本计算的 Z-score 标准化值；标签为“明日是否上涨”。把所有样本随机打乱后做 5 折交叉验证，网格搜索了约 500 组超参数，选出验证准确率最高的深度网络。回测不计佣金与滑点，按收盘价成交，样本内准确率 58%，宣布策略有效。

可对照以下要点自查（不止 3 个）：

1. **全样本标准化泄露**：用全样本（含未来）算 Z-score，标准化里就偷看了未来；应只用训练期统计量。
2. **随机打乱 K 折泄露**：时间被打乱，未来样本进了训练集，验证分数虚高；应做时序切分加 purge/embargo。
3. **缺少清洗与禁运**：标签依赖明日，训练与验证紧邻会因自相关渗透，需要禁运间隔。
4. **网格搜索过拟合**：500 组超参等于试了 500 个策略，最高分大概率是被噪声选出的；应在独立、未参与调参的数据上确认。
5. **忽略交易成本**：每日预测意味着高换手，不计佣金滑点的 58% 准确率毫无说服力。
6. **准确率口径误导**：58% 准确率未必盈利，要看每次盈亏幅度和扣成本后的收益。
7. **模型复杂度与样本不匹配**：日频表格数据上直接上深度网络，数据量与信噪比都不支持，过拟合风险高。

如果一个方案在这些问题上大多没有交代，它的“有效”只能当灵感，不能当证据——这正是 [[回测方法论]] 的结论在 ML 场景下的复述。
