# 网站工具与资源导航

> [!note] 核心问题
> 网上量化资源无限，收藏夹会变成坟场。本篇按**用途**给出精选入口：从哪里注册、文档怎么读、社区怎么用、本库哪篇深挖，并给你一个可维护的书签体系与每周阅读节奏。

## 学习目标

读完这篇，你要能做到：

1. 按「数据 / 平台 / 框架 / 宏观 / 学习 / 求职」找到官方入口。
2. 用「主路径 3 站 + 扩展库」管理收藏，而不是 100 个未读链接。
3. 把外部资源映射回本 Vault 笔记。
4. 建立每周固定的信息饮食，避免信息过载。
5. 识别低质量「稳赚」内容并跳过。

对大多数国内学习者，阶段零只需钉住：

| 1 | [聚宽](https://www.joinquant.com/) 或你选的主研究平台 | 回测与作业 | [[国内量化平台对比与上手]] |

其余全部进「扩展」，需要时再打开。

## 官网文档怎么读（省时间）

```text
1. 安装 / Quick Start（只求跑通）
2. 核心概念页（数据频率、复权、订单）
3. 你正在用的那一个 API 的参数表
4. Changelog（报错前先看是否接口变更）
5. 示例库 / 官方策略模板
```

不要从头到尾读完文档再动手。见 [[从零开始的第一条实操路线]]。

## 建议书签结构（浏览器）

```text
quant/
  00-daily/          # 最多 5 个每日可能打开
  10-data/
  20-platforms/
  30-frameworks/
  40-macro-papers/
  90-someday/        # 想看但现在不看
```

每季度把 `90-someday` 清空一次：要么升级到正式分类，要么删除。

## 每周信息饮食（可执行）

| 日 | 内容 | 时长上限 |
|---|---|---|
| 固定研究日 | 主线笔记 + 代码实验 | 你的主学习块 |
| 1 次 | 官方文档/Changelog | 30 分钟 |
| 1 次 | 社区仅搜具体报错 | 20 分钟 |
| 可选 | 1 篇研报或论文四行摘要 | 40 分钟 |
| 禁止 | 无目的刷「荐股/跟单」 | 0 |

## 低质量内容识别

| 红旗 | 含义 |
|---|---|
| 只展示权益曲线无成本 | 可能误导 |
| 「稳赚」「必涨」「内部指标」 | 离开 |
| 不给样本外、不给失败案例 | 不完整 |
| 逼你立刻付费解锁代码 | 卖课漏斗 |
| 保证年化数字 | 忽略风险与制度 |

## 第一次上网的 90 分钟脚本

| 分钟 | 动作 |
|---:|---|
| 0–15 | 打开聚宽，注册，找到帮助文档 |
| 15–35 | 跑通研究环境取数示例 |
| 35–55 | 打开 AKShare 文档，本地或对照接口名 |
| 55–75 | 在 Vault 写 5 行：今日链接、报错、收获 |
| 75–90 | 把 3 个主站放进浏览器 `00-daily` |

## 常见误区

| 误区 | 更好的理解 |
|---|---|
| 资源越多越接近成功 | 消化率才重要 |
| 先收集再学习 | 边学边收，主路径优先 |
| 英文站 dig 才专业 | 中文平台足够完成阶段零 |
| 所有 GitHub 明星库都要装 | 装一条链 |
| 研报 = 交易指令 | 研报是线索 |
