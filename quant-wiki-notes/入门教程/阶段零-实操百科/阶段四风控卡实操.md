# 阶段四风控卡实操

> [!note] 核心问题
> 阶段四的知识要落成**一张你敢贴在桌前的风控卡**，并用 quant-lab 报告里的回撤/换手数字做「压力参照」，而不是空写百分比。

## 学习目标

做完本篇，你要能做到：

1. 填写个人/策略两级风控卡。  
2. 从 `reports/*.json` 读取 max_drawdown 等字段作为历史参照。  
3. 设定日/总回撤熔断与单票上限。  
4. 写 3 个压力情景与动作。  
5. 把风控卡链到模拟盘与毕业 IPS。

## 从报告提取数字

```powershell
cd ...\quant-lab
python scripts/summarize_report.py reports/你的结果.json --max-dd-limit 0.25
```

| 报告字段 | 写入风控卡的用法 |
|---|---|
| max_drawdown | 历史参照，**未来可能更差** |
| avg_turnover | 提示成本与执行负担 |
| total_return vs buy_hold | 是否只是 beta |

> [!warning]
> 历史最大回撤 ≠ 限额。限额应 ≤ 你能承受且不破坏生活的水平，通常比回测回撤更严。

若使用 [[组合层实操]]：

把 `port_*.json` 的 `avg_turnover` 抄到卡 B。

## 模拟盘联动

1. 卡 B 熔断阈值写入 [[模拟盘日志实操]] 备注。  
2. 破例 `rule_break=1` 累计达 N → 触发「强制减仓条件」。  
3. 周复盘对照：是否触线却未执行（执行风险）。  

## 完成定义

- [ ] 卡 A、卡 B 无空关键行  
- [ ] 至少 3 个压力情景有动作  
- [ ] 至少引用 1 份真实 report 数字作参照  
- [ ] 明确「触线谁有权停止交易」（自己）  

## 常见误区

| 误区 | 更好的理解 |
|---|---|
| 限额抄回测 mdd | 限额应对齐生活承受力 |
| 只写不做 | 模拟盘要练习触线 |
| 多策略共用一张糊卡 | 总账户卡 + 分策略卡 |
| 杠杆「先用着」 | 阶段四默认无杠杆 |
