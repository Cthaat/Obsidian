# 模拟盘日志实操

> [!note] 核心问题
> [[从模拟到小资金实盘]] 给了门槛；本篇给**可执行的日志格式**与 quant-lab 小工具，让你每天 5 分钟留下可复盘证据，而不是靠记忆。

## 学习目标

读完这篇，你要能做到：

1. 建立纸交易 CSV 日志并理解各字段。  
2. 用 `init_paper_log.py` / `append_paper_signal.py` 辅助记录双均线信号。  
3. 区分「计划价」与「模拟成交价」。  
4. 做每周执行率与规则外交易统计。  
5. 在日志与回测结果不一致时定位原因。  

## 日志字段

| 字段 | 含义 |
|---|---|
| date | 决策日 |
| strategy | 策略版本名 |
| symbol | 标的 |
| signal | LONG/FLAT 等 |
| target_pos | 目标仓位 0/1 或权重 |
| plan_price | 信号参考价（常为信号 bar 收盘） |
| fill_price | 你模拟的成交价（次日开盘等） |
| shares | 模拟股数 |
| deviation | fill/plan - 1 |
| mood_1_to_5 | 情绪 |
| rule_break | 是否破例 0/1 |
| notes | 自由备注 |

模板也在 Vault：可与 [[实验日志目录]] 的周复盘一起用。

## quant-lab 命令

```powershell
cd ...\quant-lab
# 确保有数据
python scripts/pull_akshare_example.py --symbol 000001

# 创建日志
python scripts/init_paper_log.py --name dual_ma_000001

# 根据最新 processed 收盘算双均线目标仓，并追加一行
python scripts/append_paper_signal.py --name dual_ma_000001 --symbol 000001

# 若你已按规则「模拟成交」，补成交价与情绪
python scripts/append_paper_signal.py --name dual_ma_000001 --symbol 000001 --fill-price 12.34 --mood 3 --shares 100 --notes "next open sim"
```

文件：`data/paper_logs/{name}.csv`（默认被 gitignore 思路忽略大数据；本地保留）。

> [!warning]
> 这不是券商接口，不会下真实订单。`append_paper_signal` 用**本地已拉取数据的最后一根 bar** 生成信号，实盘/仿真时你必须自己对齐「信号时点 vs 成交时点」。

## 推荐每日流程（5–10 分钟）

```text
1. 更新数据（或打开平台行情）
2. 运行 append 或手写计划信号
3. 按说明书规则写下 fill（次日再补也行）
4. rule_break 有则必须写 notes
5. 周末汇总执行率
```

## 每周统计（抄到周复盘）

| 指标 | 算法 |
|---|---|
| 执行率 | 有 fill 的信号日 / 应交易日 |
| 破例次数 | rule_break=1 计数 |
| 平均 \|deviation\| | 滑点体感 |
| 情绪均值 | mood 平均 |

模板：[[周复盘模板]]。

## 与回测对齐时常见偏差

| 偏差 | 可能原因 |
|---|---|
| 日志交易多于回测 | 手滑加仓、未冻结版本 |
| 日志收益差很多 | 成交时点不同、漏记空仓 |
| 情绪分与亏损同步 | 纪律问题，不是参数问题 |

## 常见误区

| 误区 | 更好的理解 |
|---|---|
| 自动化日志=已模拟 | 仍要填 fill 与是否执行 |
| 不写破例就不算破例 | 自欺 |
| 日志完美才开始 | 先有脏日志再变干净 |
| 用日志优化参数 | 那是新研究，回研究流 |
