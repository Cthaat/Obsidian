# 阶段三作业打通清单

> [!note] 核心问题
> 阶段三要求把「我觉得」变成「可验证」。本清单用 quant-lab 已有策略，强制你交付一份**研究包（research package）**：说明书 + 主结果 + 成本敏感性 + 偏差自检 + EXP 链，而不是一张漂亮曲线。

## 学习目标

做完本清单，你要能做到：

1. 选定一个冻结版本策略作为作业主策略。  
2. 完成样本划分与一次测试集主结果。  
3. 做至少两档成本敏感性。  
4. 用清单自检前视、过拟合、规则缺口。  
5. 归档到 [[实验日志目录]]，并可用 `summarize_report.py` 出风控摘要。

## 作业主策略怎么选

| 路径 | 命令入口 | 适合 |
|---|---|---|
| A 双均线 | `run_dual_ma.py` / Backtrader | 最稳妥交作业 |
| B 时序动量 | `run_ts_momentum.py` | 练信号窗口 |
| C 等权组合 | `run_equal_weight_rebalance.py` | 练组合层（信号弱） |
| D 因子打分 | `run_factor_score.py` | 仅当能写清 PIT 边界 |

**推荐**：A 或 B 作为主交卷；C/D 作附录对照。

## 交付物清单（全部要有）

| # | 交付物 | 最低标准 |
|---:|---|---|
| 1 | 策略说明书 | 信号、池、成交、成本、区间、失效条件 |
| 2 | 数据 meta | 源、复权、日期、路径 |
| 3 | 训练/测试划分 | 写进说明书与 EXP |
| 4 | 测试集主结果 JSON | `reports/*.json` |
| 5 | 成本敏感性 | ≥2 档 cost_bps 或 commission |
| 6 | 偏差自检表 | 见下文，逐项打勾 |
| 7 | EXP 笔记 | 预注册 + 结论 |
| 8 | 风控摘要 | `summarize_report.py` 输出或手抄 |

## Step-by-step

### 1. 冻结版本（30 分钟）

```text
策略名：____________  v____
禁止再改参直到作业写完。
```

### 2. 写说明书（1 小时）

用 [[第一个可回测策略]] 模板，至少含：

- 经济/行为逻辑一句话  
- 明确成交时点  
- A 股未建模项声明（[[A股交易规则实操要点]]）  

### 3. 跑主结果（1 小时）

```powershell
cd ...\quant-lab
# 例：双均线
python scripts/run_dual_ma.py --symbol 000001 --fast 5 --slow 20 --cost-bps 10
python scripts/summarize_report.py reports/dual_ma_000001_5_20.json --max-dd-limit 0.25
```

### 6. EXP 归档

复制 [[EXP模板]]，链到 JSON 路径。结论只能：保留 / 放弃 / 迭代（作业交卷可用「保留为阶段三样本」）。

### 7. （可选）第二引擎对照

双均线：`run_backtrader_dual_ma.py`，说明 stake 不同则收益不可横比。见 [[Backtrader对照实操]]。

## 完成定义

- [ ] 八项交付物齐全  
- [ ] 自检表无「空白」  
- [ ] 能用 5 分钟讲清：逻辑、主结果、主要风险、下一步  
- [ ] 不把作业收益当实盘预期
