# 毕业项目实操模板

> [!note] 核心问题
> [[毕业项目]] 定义了「要交付什么」；本页给你**可勾选的组装清单**，把阶段零产物与五阶段笔记装进一个文件夹式交付包，避免毕业时从零拼凑。

## 学习目标

1. 建立毕业交付目录结构。  
2. 用已有作业填充 IPS / 公司分析 / 策略包 / 风控卡 / 日志。  
3. 完成最小可行体系（MVP）自检。  
4. 写清迭代循环中你卡在哪一环。  
5. 会指向 quant-lab 路径作为可复现附件。  

## 交付包目录（建议）

**可直接填写的总装表：** [[毕业交付包-填空稿]] · 交前必做 [[作业一致性自检表]] · 材料索引 [[作业填空/目录]]

```text
毕业交付-你的名字-日期/
  00_README.md                 # 5 分钟导读
  01_IPS投资策略说明书.md
  02_公司分析_v0.1.md          # 来自阶段二作业
  03_策略研究包/               # 来自阶段三作业
    说明书.md
    EXP主结果.md
    reports/                   # 可复制 json 摘要
  04_风控卡.md                 # 来自阶段四风控卡
  05_模拟与执行日志.md         # 链接或摘录
  06_复盘与迭代计划.md
  07_毕业自测勾选.md
```

Vault 内也可分散用双向链接；**必须有一份 00_README 总索引**。

## MVP 定义（建议默认）

个人学习者的 MVP **不是**全自动高频，而是：

1. 书面 IPS；  
2. 一个低频可解释策略 + 诚实回测；  
3. 一张风控卡；  
4. 可执行的复盘节奏。  

复杂期权、ML、多空可进阶段五专题后再升级。

## 毕业自测（精简版）

完整版见 [[毕业项目]] 正文；此处为阶段零组装自检：

| # | 问题 | 是/否 |

| 2 | 有一份真实公司分析？ |  |
| 3 | 有策略说明书且费用>0？ |  |
| 4 | 测试集主结果只汇报一次主表？ |  |
| 5 | 有风控卡且含熔断动作？ |  |
| 6 | 有模拟日志证据？ |  |
| 7 | 知道体系哪一环最弱？ |  |
| 8 | 明确不做什么？ |  |

8 项全「是」可视为**入门毕业可讨论**；否则先补最短板。
