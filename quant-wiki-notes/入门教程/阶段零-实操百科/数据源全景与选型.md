# 数据源全景与选型

> [!note] 核心问题
> 量化是数据行业。选错数据源，后面所有回测都是沙上建塔。本篇按**场景**帮你选型：免费能做什么、积分墙在哪、如何做数据质量检查，以及和本库进阶文档如何衔接。

## 学习目标

读完这篇，你要能做到：

1. 按「市场 + 频率 + 预算」选出主数据源与备用源。
2. 解释复权、点-in-time、幸存者偏差对研究的影响。
3. 对拉取结果做最小质量检查。
4. 知道个人学习、严肃研究、准实盘三阶段数据标准不同。
5. 在笔记中正确记录数据出处与版本。

## 先问需求，再问接口

| 问题 | 你的答案决定 |
|---|---|
| 哪个市场？ | A 股 / 港股 / 美股 / 期货 / 基金 |
| 什么频率？ | 日线足够？还是分钟、tick？ |
| 要不要财务与复权？ | 选股因子 vs 纯价格 |
| 是否必须点-in-time？ | 防未来函数 |
| 预算？ | 纯免费 / 低价会员 / 机构 |
| 用途？ | 学习 / 论文级 / 准实盘 |

> [!important]
> 学习阶段用免费源完全够。把时间花在**研究规范**上，比过早买贵数据更重要。

## 五个比「免费」更重要的概念

### 1. 复权

| 口径 | 含义 | 误用后果 |
|---|---|---|
| 不复权 | 原始成交价 | 均线、涨跌幅被除权跳空扭曲 |
| 前复权 | 以当前价为基准回推 | 常用技术/趋势研究 |
| 后复权 | 以历史为基准向后 | 长期收益研究常见 |

**规则**：同一研究内口径一致；笔记写明 `adjust=...`。

### 2. 幸存者偏差

只拿「现在还在市的股票」回测历史选股，会**高估**收益。严肃研究需要包含退市、用当时可投资股票池。免费源在这一点上往往弱于商业/平台数据——学习阶段先**知道问题存在**，进阶再升级数据。

### 3. 点-in-time（PIT）

财报、一致预期必须用「当时可知」的数据。用最终修正后的财报回测历史，是隐性未来函数。见 [[回测方法论]]。

### 4. 停牌、涨跌停、ST

不可交易时段若按可买可卖撮合，会美化结果。平台回测通常有处理；本地自写引擎必须显式处理。

### 5. 时区与交易日历

多市场组合时，对齐日历错误会导致虚假同期信号。

## 数据质量最小检查表

每次新接口或新时间段，跑一遍：

| 检查 | 方法 | 不合格时 |
|---|---|---|
| 行数是否合理 | 交易日数量级 | 重拉或换源 |
| 缺失值 | `isna` 比例 | 填补策略或剔除 |
| 价格跳变 | 日收益绝对值阈值 | 查除权、错误 tick |
| 成交量负值/零 | 统计 | 标记停牌 |
| 时间单调 | 排序与重复索引 | 去重 |
| 与第二来源抽查 | 同一日收盘价对比 | 记录差异 |
| 复权一致性 | 事件日前后 | 统一口径 |

```python
# 假设示例：日线质量快检
import pandas as pd

def quick_check(df, date_col="日期", close_col="收盘"):
    d = df.copy()
    d[date_col] = pd.to_datetime(d[date_col])
    d = d.sort_values(date_col)
    ret = d[close_col].pct_change()
    report = {
        "rows": len(d),
        "dup_dates": int(d[date_col].duplicated().sum()),
        "na_close": int(d[close_col].isna().sum()),
        "max_abs_ret": float(ret.abs().max()) if len(ret) else None,
    }
    return report
```

## 如何使用各网站（操作层）

### AKShare

1. 文档：[akshare.akfamily.xyz](https://akshare.akfamily.xyz/)  
2. `pip install akshare --upgrade`  
3. 数据字典 → 选接口 → 复制示例  
4. 固定 `symbol`、日期、`adjust`  
5. 存 `data/raw/` 并写元数据  

深读：[[AKShare概览]]、[[AKShare安装配置]]、[[AKShare-股票数据]]。

### Tushare

1. 打开 [tushare.pro](https://www.tushare.pro/) 注册。  
2. 在个人主页获取 token。  
3. 阅读接口文档的**积分/权限**要求。  
4. 本地：

```python
import tushare as ts
pro = ts.pro_api("YOUR_TOKEN")  # 不要把 token 提交到公开仓库
# 具体接口名与字段以官网文档为准
```

5. 用环境变量存 token：`TUSHARE_TOKEN`。  

深读：[[TuShare数据接口]]。

### 平台内数据（聚宽等）

1. 注册后在研究环境直接调用平台 API。  
2. 优点：与回测对齐，少清洗。  
3. 缺点：迁移本地成本；注意用户协议与导出限制。  

见 [[国内量化平台对比与上手]]。

## 费用与合规心态

| 原则 | 说明 |
|---|---|
| 免费源 ≠ 可任意商用 | 读许可与免责声明 |
| 爬虫与频率 | 控制频率，尊重源站；优先官方 API |
| 实盘行情 | 常用券商/交易所授权源 |
| 内幕与未公开数据 | 违法红线，不在讨论范围 |

## 升级路径

| 阶段 | 数据标准 |
|---|---|
| 学习 | 免费日线 + 明确复权 + 质量检查 |
| 作品集 | 样本内外划分、基准、成本；尽量处理 ST/停牌 |
| 准实盘 | 许可清晰、延迟可控、运维监控、备份 |
| 机构 | PIT、退市、公司行动、审计轨迹 |

## 常见误区

| 误区 | 更好的理解 |
|---|---|
| 数据越贵 alpha 越多 | 贵的是完整与许可，不是印钞 |
| 一个源包打天下 | 主源 + 抽查源 |
| 不记录拉取参数 | 无法复现等于没研究 |
| 实时 tick 才是量化 | 日频已足够建立正确方法论 |
| 忽略公司行动 | 拆分合并分红会毁信号 |
