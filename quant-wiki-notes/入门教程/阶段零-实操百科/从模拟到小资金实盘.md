# 从模拟到小资金实盘

> [!note] 核心问题
> 回测曲线好看，不等于你能在真实世界执行。本篇定义从**研究 → 模拟 → 小资金灰度 → 是否扩容**的门槛，以及每一阶段必须有的日志、风控与熔断，避免「第一周实盘归零」或「无限模拟自我安慰」。

## 学习目标

读完这篇，你要能做到：

1. 说出回测、模拟、实盘三者的本质差异。
2. 用清单判断策略是否达到模拟资格、小资金资格。
3. 设计 20–60 个交易日的模拟方案与记录表。
4. 为小资金设定亏损上限、仓位上限与一键停止条件。
5. 在扩容前做归因：偏差来自数据、执行、还是心理。

**可填写门槛表：** [[模拟到小资金-门槛填空稿]] · 交毕业前并入 [[毕业交付包-填空稿]]。

## 三层现实

| 层级 | 钱 | 成交 | 主要教你什么 |
|---|---|---|---|
| 回测 | 虚拟历史 | 引擎假设 | 逻辑是否自洽、历史是否敏感 |
| 模拟 / 纸交易 | 虚拟实时 | 近似市价/延迟 | 信号纪律、运维、情绪 |
| 实盘 | 真钱 | 真实微观结构 | 冲击、部分成交、事故 |

## 回测 ≠ 模拟 ≠ 实盘：差异清单

| 差异点 | 回测常见简化 | 实盘真实情况 |
|---|---|---|
| 成交价 | 收盘或开盘一价 | 滑点、排队、部分成交 |
| 可交易性 | 常忽略涨跌停 | 买不进卖不出 |
| 数据延迟 | 完美对齐 | 推送延迟、断线 |
| 心理 | 无 | 恐惧、报复交易 |
| 运维 | 无 | 断电、密钥、重复下单 |
| 成本 | 常数 bps | 佣金阶梯、冲击随量变 |
| 监管 | 无 | 异常交易、报备（程序化） |

因此：模拟的价值是**练流程与纪律**，不是证明 alpha 永恒。

## 门槛 1：能否进入模拟

同时满足再开模拟：

- [ ] 有书面策略说明书（见 [[第一个可回测策略]]）  
- [ ] 测试集结果已记录，且费用 > 0  
- [ ] 明确基准与最大可接受回撤  
- [ ] 代码/平台策略版本固定（冻结一版）  
- [ ] 知道策略逻辑失效的客观条件  
- [ ] 有每日或每周固定复盘时间  

未满足就继续研究，不进入「假装实盘」。

## 模拟盘怎么做（20–60 交易日）

### 模式选择

| 模式 | 做法 | 优点 | 缺点 |
|---|---|---|---|
| 平台仿真 | 聚宽等仿真/比赛资金 | 省事 | 撮合仍可能偏理想 |
| 手工纸交易 | 按收盘信号次日记录虚拟单 | 强迫纪律 | 费时 |
| 本地纸交易 | 脚本读行情写日志不下真单 | 可自动化 | 要自己写可靠日志 |

阶段零推荐：**平台仿真 + 手写决策日志** 双轨，防止「全自动但从未看过单」。

### 模拟规则（写进笔记）

| 项目 | 建议 |
|---|---|
| 时长 | 至少 20 个交易日；趋势策略可 40–60 日 |
| 资金 | 虚拟 10 万等，与回测同量级便于对比 |
| 信号时点 | 与说明书完全一致 |
| 禁止 | 模拟中途改参数「救火」 |
| 允许 | 发现 bug 则停止，修完重开计数 |
| 对照 | 每周对比：信号应达仓位 vs 实际记录 |

## 门槛 2：小资金实盘资格

模拟结束后做诚实评估：

| 问题 | 合格倾向 |
|---|---|
| 执行率 | 计划内信号执行率高，漏单有记录原因 |
| 规则外交易 | 接近 0 |
| 运维事故 | 无未发现的重复下单/数据错 |
| 回撤体感 | 模拟回撤下仍按规则 |
| 期望 | 接受「可能不赚钱，只验证流程」 |

### 小资金定义（原则，非建议金额）

| 原则 | 说明 |
|---|---|
| 亏光不影响生活 | 房租、医疗、应急金之外 |
| 小于等于「学习预算」 | 像交学费的上限 |
| 单策略上限 | 不要 All-in 一个实验 |
| 与风控卡一致 | 接 [[风险管理框架]] |

> [!important]
> 本库不给出具体投资建议或保证收益。小资金的唯一正当目的是**验证执行与心理**，不是快速翻倍。

### 上线前检查（实盘）

- [ ] 账户权限、身份证与风险测评完成  
- [ ] 交易密码 / API 密钥不进公开仓库  
- [ ] 研究环境与实盘账户隔离  
- [ ] 单笔、单日、总回撤限额写入系统或检查表  
- [ ] 有「今日停止交易」条件  
- [ ] 有人工接管步骤（断网、锁账户）  
- [ ] 程序化交易若涉及监管要求，已了解并合规  
- [ ] 首次仅最小可交易单位试单

## 小资金运行规则

| 规则 | 例（结构） |
|---|---|
| 首周 | 半仓或最小仓，只验证下单链路 |
| 扩容 | 连续 N 日无事故且执行合格再评估 |
| 减仓 / 停止 | 触达日亏损、总回撤、数据异常、接口异常 |
| 改参 | 实盘中禁止；回到研究与模拟 |
| 复盘 | 每日 10 分钟，每周 30 分钟 |

### 事故分级

| 级别 | 例子 | 动作 |
|---|---|---|
| P0 | 重复下单、错标的、密钥泄漏 | 立即停，人工处理，复盘报告 |
| P1 | 数据延迟导致错信号 | 停开仓，修数据，再模拟 |
| P2 | 轻微滑点超预期 | 记录，收紧下单方式 |
| P3 | 情绪想破例 | 记日志，不交易 |

## 门槛 3：是否扩容

扩容前用数据说话，而不是用「最近运气」：

| 证据 | 含义 |
|---|---|
| 实盘与模拟的跟踪误差 | 执行是否可靠 |
| 成本是否接近假设 | 是否该提高滑点模型 |
| 最大回撤是否在预算内 | 能否承受 |
| 生活与睡眠是否被破坏 | 规模可能过大 |
| 逻辑是否仍成立 | 市场结构是否变 |

若证据不足：**保持小资金或退回模拟**，这是专业行为。

## 心理与纪律

常见实盘心理坑：

| 坑 | 表现 | 对策 |
|---|---|---|
| 报复交易 | 亏损后加仓 | 日损失熔断 |
| 怕踏空 | 无信号开仓 | 规则外 = 违规计数 |
| 完美主义瘫痪 | 迟迟不下第一单 | 最小试单 |
| 曲线崇拜 | 回测截图当信仰 | 强制读失效条件 |

## 明确不建议新手碰的

| 项目 | 原因 |
|---|---|
| 高杠杆合约全仓 | 路径依赖爆仓 |
| 无日志 API 刷单 | 事故不可逆 |
| 不明来源「实盘跟单」 | 利益冲突与过拟合 |
| 未理解的期权卖方 | 尾部风险 |
| 规避监管的灰产接口 | 合规与资金风险 |

## 常见误区

| 误区 | 更好的理解 |
|---|---|
| 模拟赚钱 = 实盘必赚 | 只说明执行练习过关的一部分 |
| 小亏就能无限加仓 | 扩容是独立决策 |
| 实盘可边跑边调参 | 那是另一版本策略，需重新验证 |
| 自动化就无情绪 | 关停按钮前的犹豫也是情绪 |
| 不记录也记得住 | 不写等于没发生 |
