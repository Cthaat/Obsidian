# 因子打分实操

> [!note] 核心问题
> 第三策略用**多因子打分**把「截面排序」说清楚，但阶段零**只用价量因子**，并明确写出滞后与 PIT 边界，避免财务脚本假装可交易。对接 `run_factor_score.py` 与 [[因子投资体系]]。

## 学习目标

读完这篇，你要能做到：

1. 描述动量 + 低波两个价量因子如何合成。  
2. 跑通 `run_factor_score.py` 并读 last_scores。  
3. 说明权重 `shift(1)` 如何降低前视。  
4. 写出为何本脚本**不包含**财务因子。  
5. 把结果当成流程练习，而非可投资模型。  

## 因子定义（v0.1）

| 因子 | 原始量 | 方向 | 截面处理 |
|---|---|---|---|
| momentum | `P_t / P_{t-L} - 1` | 高更好 | 横截面 z-score |
| low_vol | 滚动波动 | 低更好（取负） | 横截面 z-score |
| 合成 | 两 z 等权平均 | 高更好 | 再取 top_k 等权 |

再平衡默认周频 `W`，权重次日生效。

## 为什么不做财务因子（本阶段）

| 原因 | 说明 |
|---|---|
| 公告日/PIT | 免费财务接口常缺严格 point-in-time |
| 单位与口径 | 需大量清洗与勾稽 |
| 教学目标 | 先掌握打分与组合接口 |

财务作业仍走 [[阶段二作业打通清单]] / [[财务数据实操]]；因子化基本面留给进阶 `因子投资/`。

## 命令

```powershell
cd ...\quant-lab
python scripts/pull_watchlist.py
python scripts/run_factor_score.py --top-k 2 --mom-lookback 20 --vol-lookback 20 --cost-bps 15
```

输出：

- `reports/factor_momvol_k2_m20_v20.json`  
- `reports/factor_momvol_*_last_scores.json`  

## 与单一动量的差别

| 项目 | 截面动量 | 因子打分 v0.1 |
|---|---|---|
| 输入 | 单一动量排序 | 动量 + 低波合成 |
| 教学点 | 排序与 top_k | 多因子合成与冲突 |
| 代码 | `run_cross_section_momentum.py` | `run_factor_score.py` |

当动量与低波冲突时，合成分数会「折中」——打开 last_scores 观察。

## 实验纪律

| 版本 | 唯一变化 |
|---|---|
| v0.1 | 默认 20/20，k=2 |
| v0.2 | 只改 mom_lookback=60 |
| v0.3 | 只改 top_k=3 |
| v0.4 | 只改 cost_bps |

禁止一次改五个超参只报最好曲线。见 [[研究笔记与实验工作流]]。

## 失效与红旗

| 红旗 | 处理 |
|---|---|
| 池子 < 5 | 只作流程，不写业绩故事 |
| 无成本很好看 | 提高 cost_bps |
| 与等权组合几乎一样 | 因子可能没增量 |
| 想加 ROE 却无公告日 | 停止并读 PIT 文档 |

## 常见误区

| 误区 | 更好的理解 |
|---|---|
| z-score 很高级=有 alpha | 只是无量纲化 |
| 因子越多越好 | 噪声与过拟合 |
| 价量因子无前视风险 | 仍有复权/停牌/未来函数实现 bug |
| 教学脚本可直连实盘 | 否 |
