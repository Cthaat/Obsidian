# 回测框架选型与最小示例

> [!note] 核心问题
> 回测框架不是宗教：事件驱动、向量化、AI 研究平台、交易系统框架解决的问题不同。本篇帮你**选一个**跑通最小示例，并连回 [[回测方法论]] 的正确姿势。

## 学习目标

读完这篇，你要能做到：

1. 区分事件驱动 vs 向量化回测的适用场景。
2. 为「学习双均线 / 多因子研究 / 实盘交易」匹配框架。
3. 在本地或文档级理解最小运行骨架。
4. 给回测加上不可省略的费用与基准意识。
5. 知道框架不能自动消灭的偏差（前视、过拟合等）。

## 框架地图

| 类型 | 代表 | 核心思想 | 适合 |
|---|---|---|---|
| 事件驱动 | Backtrader, zipline-relatives, vn.py 回测 | 按 tick/bar 推送事件 | 路径依赖、订单逻辑、教学 |
| 向量化 | vectorbt, 自写 pandas | 整列运算 | 快速参数扫描、组合研究 |
| 研究平台 | 聚宽等云回测, QuantConnect LEAN | 数据+引擎托管 | 少运维、标准撮合 |
| AI 量化研究 | [Qlib](https://qlib.readthedocs.io/) | 数据层-模型-组合-工作流 | 因子/ML 研究管线 |
| 交易系统 | vn.py | 网关+策略 app | 仿真与实盘工程 |

本库延伸：[[Backtrader实战入门]]、[[开源回测框架全景对比]]、[[VnPy框架详解]]、[[quant_learn]]。

## 怎么选（决策表）

| 你的目标 | 首选 | 次选 | 不要先碰 |
|---|---|---|---|
| 第一周学会回测概念 | 云平台 或 Backtrader | backtesting.py | 自写完整撮合 |
| 大量参数/组合扫描 | vectorbt / pandas | — | 纯事件驱动扫万级参数 |
| 多因子 + ML 工作流 | Qlib | 自建 pipeline | 无验证的深度学习 |
| 期货 CTA 仿真到实盘 | vn.py | 平台仿真 | 无日志的脚本下单 |
| 论文复现 | 原作者代码 + 自建 | Qlib | 随意换框架丢细节 |

## 事件驱动 vs 向量化

| 维度 | 事件驱动 | 向量化 |
|---|---|---|
| 速度 | 相对慢 | 快 |
| 订单与路径 | 自然 | 需小心实现 |
| 学习曲线 | 中 | 低（若懂 pandas） |
| 典型坑 | 写复杂 | 偷看未来一行、忽略停牌 |
| 示例问题 | 「这根 K 线收盘后下单，下一根开盘成交」 | 「全样本 shift 是否正确」 |

两者可组合：向量化做研究筛选，事件驱动做最终验证。

## 最小示例 A：Backtrader 骨架

安装：

```powershell
pip install backtrader pandas
```

教学骨架（需自备 CSV，列含 datetime/open/high/low/close/volume）：

```python
import backtrader as bt
import datetime

class DualMA(bt.Strategy):
    params = dict(fast=5, slow=20)

    def __init__(self):
        self.fast_ma = bt.ind.SMA(period=self.p.fast)
        self.slow_ma = bt.ind.SMA(period=self.p.slow)
        self.crossover = bt.ind.CrossOver(self.fast_ma, self.slow_ma)

    def next(self):
        if not self.position and self.crossover[0] > 0:
            self.buy()
        elif self.position and self.crossover[0] < 0:
            self.close()

if __name__ == "__main__":
    cerebro = bt.Cerebro()
    data = bt.feeds.GenericCSVData(
        dataname="data/processed/sample.csv",
        dtformat="%Y-%m-%d",
        datetime=0, open=1, high=2, low=3, close=4, volume=5,
        openinterest=-1,
    )
    cerebro.adddata(data)
    cerebro.addstrategy(DualMA)
    cerebro.broker.setcash(100000.0)       # 假设初始资金
    cerebro.broker.setcommission(0.0003)  # 假设佣金，不完整建模印花税
    print("Start", cerebro.broker.getvalue())
    cerebro.run()
    print("End", cerebro.broker.getvalue())
```

> [!warning]
> 上例**没有**完整建模 A 股印花税、最小手数、涨跌停、滑点。教学用。严肃研究请补全或使用更完整的平台撮合。

## 最小示例 B：pandas 向量化双均线（思想）

```python
import pandas as pd

def dual_ma_signal(close: pd.Series, fast=5, slow=20) -> pd.Series:
    fast_ma = close.rolling(fast).mean()
    slow_ma = close.rolling(slow).mean()
    # 1 多头，0 空仓；用 shift(1) 避免用到当根未来
    signal = (fast_ma > slow_ma).astype(int).shift(1).fillna(0)
    return signal

def backtest_long_only(close: pd.Series, signal: pd.Series, cost_bps=10):
    ret = close.pct_change().fillna(0)
    pos = signal
    # 换仓成本：假设示例
    turn = pos.diff().abs().fillna(0)
    strat_ret = pos * ret - turn * (cost_bps / 10000.0)
    equity = (1 + strat_ret).cumprod()
    return strat_ret, equity
```

关键：`shift(1)` 表达「信号在收盘后产生，下一期才持仓」。漏了就是经典未来函数。

## 最小示例 C：云平台 / QuantConnect 思路

| 平台 | 你要找的文档关键词 |
|---|---|
| 聚宽 | 策略程序框架、initialize、handle_data / 定时、order、set_benchmark、set_order_cost |
| QuantConnect | QCAlgorithm, Initialize, OnData, SetCash, SetBrokerageModel |

流程统一为：

```text
Initialize → 订阅数据 → 每个 bar 更新指标 → 下单 → 引擎撮合 → 统计
```

## Qlib 适合什么

[Qlib](https://qlib.readthedocs.io/) 面向 **AI/量化研究工作流**：数据准备、模型训练、组合生成、回测分析、实验记录。  

适合：你已会 pandas，想做系统化因子/ML，而不是第一周学买卖函数。  

新手路径：官方 Quick Start → 准备数据 → 跑通工作流 → 再换自定义模型。A 股数据需按文档自行准备或使用社区数据方案。

## 回测里必须显式设定的项

| 项目 | 坏默认 | 更好做法 |
|---|---|---|
| 初始资金 | 随意 | 写进报告 |
| 手续费 | 0 | 保守偏高 |
| 滑点 | 0 | 按品种流动性 |
| 成交价 | 当根收盘无约束 | 规定下一根开盘/均价 |
| 基准 | 无 | 指数或 60/40 |
| 股票池 | 全市场无过滤 | 可投资集合 |
| 涨跌停 | 忽略 | 不可买/不可卖规则 |

## 框架不能替你做的事

即使引擎再强，下列仍靠你：

1. 经济逻辑是否成立；  
2. 样本外是否预留；  
3. 是否在全历史调参；  
4. 数据是否 PIT；  
5. 心理上能否执行。  

对照 [[回测方法论]]、[[回测七大陷阱系统解析]]、[[过拟合识别与防御]]。

## 建议学习顺序（框架）

```text
1. 云平台跑通官方策略模板
2. pandas 手写向量化，吃透 shift 与成本
3. Backtrader 事件驱动，理解订单
4. （可选）vectorbt 做参数扫描
5. （可选）Qlib 做研究管线
6. （可选）vn.py 做仿真交易
```

## 常见误区

| 误区 | 更好的理解 |
|---|---|
| 换框架能提高夏普 | 换的是工具，不是 alpha |
| 向量化一定错 | 正确 shift 与可交易性处理即可 |
| 事件驱动一定对 | 仍可能写错成交时点 |
| 先写完美引擎 | 先用现成引擎学方法论 |
| 忽略基准 | 牛市满仓策略都会「好看」 |
